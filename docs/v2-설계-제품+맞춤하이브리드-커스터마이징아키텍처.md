# v2 설계 — 제품 + 맞춤(커스텀) 하이브리드 모델 & 커스터마이징 아키텍처

> 성격: 방향 합의용 **설계도**(구현 스펙 아님). 두 가지를 한 그림으로: ① 셀프서브 SaaS + 유료 맞춤(컨설팅) 사업 모델 ② 그 맞춤을 **싸게** 굴리는 커스터마이징 아키텍처(entitlements + account_settings).
> 전제: B1(구독 잠금)·B2(프리미엄 게이트) 완료. `account`/`plan` 위에 얹힘.
> 한 줄 규율: **"설정으로, 예외만 코드로, 되풀이되면 제품으로."**

---

## 0. 요약

- **기본** = 셀프서브 SaaS(basic/premium · 좌석 정액). 지금 그대로 배포.
- **상위** = **맞춤(엔터프라이즈)** — 대표가 연락 → 컨설팅 + 센터 맞춤 제작, **유료.**
- 이걸 외주 함정 없이 굴리는 열쇠 = **커스텀을 코드 분기가 아니라 계정별 데이터(설정·플래그)로** 처리 → 대부분 설정 편집, 예외만 플래그 모듈.
- **선투자 최소:** 지금 다 안 만들어도 됨. 첫 요청 올 때 그 요청 기준으로 substrate(설정·플래그 뼈대)를 작게 깐다.

---

## 1. 티어 구조 & 가격

| 티어 | 대상 | 내용 | 과금 |
|---|---|---|---|
| **basic** (셀프) | 솔로·소형 | 트레이너 도구(OT·재등록·음성일지·급여·라이브러리) | 좌석 월정액(₩5~7만) |
| **premium** (셀프) | 회원앱 원하는 곳 | + 회원앱(링크·성과 그래프) + (후속) 회원 AI | basic + α |
| **Enterprise/맞춤** (연락) | 센터·특수 니즈 | premium + **센터 맞춤 설정/기능** + 온보딩 컨설팅 | **셋업비 + 상위 월정액** 또는 프로젝트 견적 |

- **CTA**(인바운드 창구): 앱·랜딩·온보딩에 **"우리 센터에 맞춰 쓰고 싶으세요? 연락주세요"** 한 줄 + 폼/카톡. **먼저 영업하지 말고 오는 걸 받는다.**
- 맞춤은 **먼저 팔지 않음** — 셀프 제품이 1순위, 맞춤은 지불의향 높은 센터를 잡는 상위 티어.

## 2. 커스텀 요청 파이프라인 (요청 들어왔을 때)

```
[대표 문의] → 스코프 분류
   │
   ├─ (a) 설정형   → account_settings/entitlements 편집 (코드 0 · 분 단위)
   │                 예: 센터명·로고·accent 색 · KPI 목표 · 급여 파라미터 · 대본/프롬프트 톤 · 필드 라벨
   │
   ├─ (b) 기능개발형 → 일반화 모듈 + 플래그 게이트 (웹Claude 스펙 → Claude Code 구현)
   │                 → 해당 계정만 켬 → 학습을 제품 백로그로 (제품화)
   │
   └─ (c) 진짜 1센터 전용 → 최후 수단, 격리 (되도록 피함)
[견적·계약] → 작업 → 배포(플래그 on) → 되먹임
```

**규칙:** **"두 센터가 같은 걸 요청하면 = 제품 기능"** → 커스텀에서 entitlements 플래그로 승격(재판매 가능). 커스텀 포크는 진짜 한 센터만의 니즈일 때만.

---

## 3. 커스터마이징 아키텍처 (시스템을 어떻게 갖추나)

핵심 원리: **커스텀 = 계정별 데이터.** 한 센터 변경이 다른 센터에 영향 0, 되돌리기 쉬움.

### 3.1 `account_settings` — 설정 계층 (설정형 커스텀)

- 계정당 1행(또는 jsonb). UI·프롬프트·집계가 **하드코딩 대신 이 설정을 읽음.**
- 담는 것(예): 브랜딩(센터명·로고 URL·accent 토큰), 커스텀 필드 라벨, KPI/목표 매출, 급여 정책 파라미터, 대본·프롬프트 톤 노브, 표시 옵션(탭 노출 등).
- 폴백: 설정 없으면 **제품 기본값**. (안전)
- RLS: 계정 멤버가 자기 것만 읽기(`auth_account_id()` 스코프).

### 3.2 `entitlements` — 기능 플래그 (기능형 커스텀 · premium 세분화 공용)

```
entitlements ( account_id, feature_key, enabled bool, config jsonb, primary key(account_id, feature_key) )
```

- 계정별로 **어떤 모듈/기능이 켜졌나** + 그 기능의 세부 config(jsonb).
- basic/premium 기본 세트를 넘어 **맞춤 추가 기능**을 계정별로 on. B2의 `plan` 게이트와 **같은 층**에 얹힘(더 잘게).
- 헬퍼(정의):
  ```
  has_entitlement(p_key text) returns boolean  -- auth_account_id() 계정에 그 feature가 enabled(+구독 활성)인지
  ```
  → RLS·서버·UI에서 재사용(auth_account_plan()과 동일 틀).

### 3.3 기능 모듈 규율

- **새 기능은 항상 "플래그로 켜고 끄는 모듈"로** 짓는다 → `has_entitlement()` + `account_settings` 읽어 동작.
- 색/문구/숫자는 **설정에서**(퍼지세이프 토큰 맵 유지 · 동적 클래스 조립 금지 규칙 준수).
- 하드코딩 제거를 점진 확대(지금 데모 하드코딩들도 이 계층으로 흡수 가능).

### 3.4 B계열과의 정합

- `plan`(basic/premium) = 굵은 등급 게이트(B2). `entitlements` = 그 위 **잔 게이트**.
- `auth_account_id()`·`auth_account_plan()`가 이미 active+구독+만료를 봄 → `has_entitlement()`도 같은 조건 위에서만 true(일관).
- **회원 AI 게이트(남은>0)·프리미엄 세분화**도 이 entitlements 위에 자연히 얹힘 → 한 번 깔면 수익모델 전체의 스위치판.

## 4. Claude(Code)로 커스텀 제작 (실제 방식)

- 흐름 동일: **요청 → 웹Claude 스펙(일반화·플래그 게이트로 설계) → Claude Code 구현 → 검토 → 배포(해당 계정 플래그 on).**
- substrate가 config/flag-driven이면 **대부분 커스텀 = 설정 편집 또는 작은 플래그 모듈** → 빠르고 저비용.
- 즉 **너 혼자 + Claude Code로 맞춤을 감당 가능**해지는 조건 = "커스텀을 코드가 아니라 데이터로" 설계해두는 것.

---

## 5. 선투자 vs 지연 (YAGNI)

- **지금 풀 스캐폴딩 불필요.** 요청 패턴을 모른 채 미리 다 만들면 과설계.
- **최소 착수(첫 요청 때):** `account_settings` 뼈대 1 + `entitlements` 뼈대 1 + `has_entitlement()` 헬퍼 = 작게. 그 첫 요청이 필요로 하는 필드/플래그부터 채운다.
- **권장:** 베타·초기 인바운드로 **실제 반복되는 요청**을 관찰 → 반복되는 것부터 설정화(제품화). 시장이 스키마를 정하게.

---

## 6. 리스크·규율 (재확인)

- **맞춤이 제품 시간을 잡아먹지 않게** — 커스텀 프로젝트 동시 진행 상한. SaaS(레버리지)가 본체, 컨설팅(선형)은 보조.
- **무료 커스텀 금지** — 셋업비/프로젝트비 명시.
- **포크 최소** — 설정/플래그 우선, 1센터 전용 코드는 최후.
- **되먹임 의무** — 커스텀 학습은 제품 백로그로. 커스텀은 "유료 사용자 리서치"다.

---

## 7. 지금 할 것 / 나중 할 것

- **지금:** CTA 문구 + 인바운드 창구(폼/카톡) + 이 문서. (셀프 제품은 그대로 배포.)
- **첫 맞춤 요청 때:** `account_settings`·`entitlements` 최소 스캐폴딩 + `has_entitlement()` → 그 요청부터 처리.
- **로드맵 연결:** B계열(구독) 위에 얹힘. entitlements는 **회원 AI 게이트·프리미엄 세분화**와 공용이라, 그쪽 착수할 때 같이 깔면 효율.

> 결론: 아이디어 채택. 기술적으로 **완전히 가능**하고, 네가 이미 만든 account/plan/RLS 위에 **entitlements + account_settings 얇은 계층**만 얹으면 커스텀이 "포크"가 아니라 "토글"이 된다. 그 상태면 너 + Claude Code로 맞춤을 감당하면서도 본체 제품은 계속 큰다.
