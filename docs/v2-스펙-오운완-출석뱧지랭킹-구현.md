> 성격: 클로드코드 **구현 스펙**. 웹Claude(설계) → 트레이너 → 클로드코드(diff) → 트레이너(git diff) → 웹Claude(검토) → 폰/Supabase 검증.
> 커밋은 트레이너 직접(파일지정 `git add` · `npm.cmd run lint` 통과 후). SQL=Supabase 대시보드 수동(git엔 이 기록본).

## 0. 개요 — "운동해야 오운완"

**오운완(오늘 운동 완료) = 파생, 버튼 아님.** 실제 기록이 있으면 자동 획득(부정 원천 차단).

**획득 규칙(정본 · SQL `ounwan_days`에만 구현):** 한 회원의 **한 날짜(KST)** 에 아래 중 **하나라도** 있으면 그날 오운완 1개(날짜 distinct):
1. `cardio_log.performed_on` = 그날(유산소), 또는
2. `schedule_check.on_date` = 그날 이고 `kind='personal'`(개인운동), 또는
3. `daily_workout_log`: `(coalesce(session_at, created_at) AT TIME ZONE 'Asia/Seoul')::date` = 그날 **이고 유효**(`voided=false` and `source<>'noshow'`). ★`session_at`이 nullable이라 **클라 `buildActivityMap`의 `session_at ?? created_at`과 동일하게 `coalesce`** — 안 그러면 null 행이 달력엔 뜨는데 오운완엔 안 잡히는 드리프트.

**신규 저장 테이블 = 포상용 `trainer_reward` 하나뿐.** 오운완/뱃지/연속·랭킹은 전부 **RPC 서버 집계**(기존 데이터 파생 · 신규 테이블 0). 클라 `buildActivityMap`은 **달력 표시 전용**(집계에 안 씀).

## 1. 데이터 계약 · 마이그레이션

### 1.1 (선커밋 · 독립) `member_workout_log` 뷰 보정 — 노쇼/보이드 누출 차단
현재 뷰 정의가 `select id, created_at, ai_summary from daily_workout_log where user_id = auth_member_id()`로 **`voided`/`source` 필터가 없다** → 노쇼 차감 로그가 회원 달력·수업일지에 이미 새는 중(오운완과 무관한 실제 버그).
```sql
create or replace view member_workout_log with (security_invoker = false) as
  select id, created_at, ai_summary, session_at, sets_structured   -- 라이브 실제 5컬럼 그대로 유지
  from daily_workout_log
  where user_id = auth_member_id()
    and coalesce(voided, false) = false          -- 이미 라이브에 존재
    and coalesce(source, '') <> 'noshow';        -- ★유일한 신규(노쇼 누출 차단)
```
- ★**라이브 뷰가 git 기록과 다름(반드시 `pg_get_viewdef`로 실제 컬럼 확인 후 작성)**: `member-workout-structured` 마이그레이션이 뷰를 이미 5컬럼(**+session_at, +sets_structured**)·`voided=false` 필터로 갱신해둠. `sets_structured`는 회원앱 **종목별 무게 변화(`buildExerciseSeries`)가 쓰는 필수 컬럼** — 절대 드롭 금지. `create or replace`는 컬럼을 못 빼므로(빼면 `ERROR: cannot drop columns from view`), **5컬럼 그대로 두고 `source<>'noshow'` 한 줄만 추가**.
- ★**이번 변화는 노쇼 제거 하나뿐**: `session_at`·`voided` 필터는 이미 라이브라 달력 날짜·보이드는 안 바뀜. **노쇼가 달력·수업일지에서 사라지는 것만** 회원에게 보임(트레이너 사전 인지).
- ★**오운완과 분리해 이 뷰 보정만 먼저 단독 커밋.**
- 멱등: 컬럼 동일 + WHERE만 변경이라 `create or replace` 통과·재실행 무해.

### 1.2 오운완 집계 RPC (★정의 단일 출처 · ①②③ 동시 해결)
```sql
-- 내부 헬퍼: 한 회원의 오운완일(distinct date · KST). §0 규칙의 유일한 구현.
create or replace function ounwan_days(p_user_id uuid)
returns setof date language sql stable security definer set search_path = public as $$
  select distinct d from (
    select performed_on as d from cardio_log where user_id = p_user_id
    union
    select on_date from schedule_check where user_id = p_user_id and kind = 'personal'
    union
    select (coalesce(session_at, created_at) at time zone 'Asia/Seoul')::date  -- ★클라 `session_at ?? created_at`와 동일(null 드리프트 방지)
      from daily_workout_log
      where user_id = p_user_id and coalesce(voided,false)=false and coalesce(source,'')<>'noshow'
  ) x where d is not null;
$$;
-- ══════════════════════════════════════════════════════════════════════════
-- ★★ 보안 경계 (anon-open 금지와 동급 · 절대 완화 금지) ★★
--   ounwan_days / ounwan_stats_for 는 SECURITY DEFINER = 밑 3테이블 RLS를 우회한다.
--   정확성이 오직 p_user_id 인자에만 달려 있음 → 이 함수에 grant execute를 붙이면
--   ★아무 회원이나 임의 user_id를 넣어 남의 운동 이력을 읽는다(회원 격리 붕괴).
--   ⛔ authenticated/anon 에 grant execute 절대 금지.
--      노출은 래퍼만: 회원=ounwan_stats()(auth_member_id 스코프) · 트레이너=ounwan_ranking()(auth_account_id 스코프).
-- ══════════════════════════════════════════════════════════════════════════
revoke all on function ounwan_days(uuid) from public, anon, authenticated;  -- 내부용 · ⛔grant 금지(RLS 우회)

-- 코어: 한 회원의 {total, month_count, streak}. ounwan_days 단일 소스.
create or replace function ounwan_stats_for(p_user_id uuid)
returns table(total int, month_count int, streak int)
language plpgsql stable security definer set search_path = public as $$
declare v_days date[]; v_today date := (now() at time zone 'Asia/Seoul')::date; v_cur date;
begin
  select coalesce(array_agg(d order by d), '{}'::date[]) into v_days from ounwan_days(p_user_id) d;
  total := coalesce(array_length(v_days,1), 0);
  select count(*) into month_count from unnest(v_days) d
    where to_char(d,'YYYY-MM') = to_char(v_today,'YYYY-MM');
  -- 연속일: 오늘 없으면 어제부터(오늘 아직이어도 어제까지 연속 유지). 없으면 0.
  streak := 0; v_cur := v_today;
  if not (v_cur = any(v_days)) then v_cur := v_today - 1; end if;
  while v_cur = any(v_days) loop streak := streak + 1; v_cur := v_cur - 1; end loop;
  return next;
end $$;
revoke all on function ounwan_stats_for(uuid) from public, anon, authenticated;  -- 내부용 · ⛔grant 금지(위 보안경계 — RLS 우회)

-- 회원 self: auth_member_id() 기준(본인만).
create or replace function ounwan_stats()
returns table(total int, month_count int, streak int)
language plpgsql stable security definer set search_path = public as $$
declare v uuid := auth_member_id();
begin
  if v is null then return; end if;
  return query select * from ounwan_stats_for(v);
end $$;
grant execute on function ounwan_stats() to authenticated;

-- 트레이너 랭킹: auth_account_id() 회원 전체(서버 집계 → max-rows 무관 · 코어 재사용).
create or replace function ounwan_ranking()
returns table(user_id uuid, total int, month_count int, streak int)
language plpgsql stable security definer set search_path = public as $$
declare v_acc uuid := auth_account_id(); r record; s record;
begin
  if v_acc is null then return; end if;
  for r in select u.id from user_table u where u.account_id = v_acc and coalesce(u.hidden,false)=false loop
    select * into s from ounwan_stats_for(r.id);
    user_id := r.id; total := s.total; month_count := s.month_count; streak := s.streak;
    return next;
  end loop;
end $$;
grant execute on function ounwan_ranking() to authenticated;
```
- **①해결**: `ounwan_days`가 원본 3테이블 **전체**를 집계 → limit 창 무관·누적 불변(감소 없음).
- **②해결**: 회원·트레이너 둘 다 `ounwan_stats_for` 한 함수 → 정의 물리적 단일화(드리프트 불가). schedule_check는 `kind='personal'`만(정본).
- **③해결**: 랭킹이 raw를 안 받고 서버에서 집계행만 반환 → 1000행 잘림 무관.
- **보안**: `ounwan_days`/`_for`는 authenticated 실행 회수(내부용) → 회원/트레이너는 self·account 스코프 함수만 호출. `auth_member_id()`/`auth_account_id()`는 SECURITY DEFINER 안에서도 caller JWT 기준(정상). `set search_path=public` 고정.
- **성능 노트**: `ounwan_ranking`은 회원 N명 루프 × 3테이블 스캔. `cardio_log`·`schedule_check`는 (user_id,날짜) 인덱스 보유. `daily_workout_log`는 **인덱스 확인·추가 필요(§1.2b)** — 제일 크고 빨리 자라는 테이블이라 없으면 N번 순차 스캔. 후속 set-based 재작성 여지.

### 1.3 `trainer_reward` — 트레이너별 포상 정의 (신규 테이블 · trainer_goal 패턴)
```sql
create table if not exists trainer_reward (
  id          uuid primary key default gen_random_uuid(),
  account_id  uuid not null default auth_account_id() references account(id),
  trainer_id  uuid not null default auth.uid(),
  milestone   int  not null,                       -- 오운완 누적 N회 달성 시
  reward_text text not null,
  active      boolean not null default true,
  created_at  timestamptz default now(),
  constraint trainer_reward_milestone_chk check (milestone > 0)
);
create index if not exists trainer_reward_acc_idx on trainer_reward (account_id, trainer_id);
alter table trainer_reward enable row level security;

drop policy if exists "tr_reward_read" on trainer_reward;
create policy "tr_reward_read" on trainer_reward for select to authenticated
  using (account_id = auth_account_id() and (auth_is_owner() or trainer_id = auth.uid()));
drop policy if exists "tr_reward_insert" on trainer_reward;
create policy "tr_reward_insert" on trainer_reward for insert to authenticated
  with check (account_id = auth_account_id());
drop policy if exists "tr_reward_update" on trainer_reward;
create policy "tr_reward_update" on trainer_reward for update to authenticated
  using (account_id = auth_account_id() and (auth_is_owner() or trainer_id = auth.uid()))
  with check (account_id = auth_account_id());
drop policy if exists "tr_reward_delete" on trainer_reward;
create policy "tr_reward_delete" on trainer_reward for delete to authenticated
  using (account_id = auth_account_id() and (auth_is_owner() or trainer_id = auth.uid()));

-- ★회원 read — user_table 서브쿼리 금지(회원은 user_table을 못 읽음 · member-views가 회원 select 정책 제거).
--   정책 안 서브쿼리도 RLS를 받으므로 exists(select from user_table)는 회원 세션에서 항상 0행 → 정책 영구 false.
--   → 회원 account/trainer를 SECURITY DEFINER 헬퍼로 가져와 직접 비교(헬퍼가 RLS 우회). 인자 없음 = 자기 것만 → grant 안전.
create or replace function auth_member_account_id() returns uuid
language sql stable security definer set search_path = public as $$
  select account_id from user_table where id = auth_member_id();
$$;
create or replace function auth_member_trainer_id() returns uuid
language sql stable security definer set search_path = public as $$
  select trainer_id from user_table where id = auth_member_id();
$$;
grant execute on function auth_member_account_id() to authenticated;
grant execute on function auth_member_trainer_id() to authenticated;

-- 자기 트레이너 활성 포상. solo(trainer_id null)면 계정 내 전체(트레이너 1인).
drop policy if exists "tr_reward_member_read" on trainer_reward;
create policy "tr_reward_member_read" on trainer_reward for select to authenticated
  using (
    active
    and account_id = auth_member_account_id()
    and (trainer_id = auth_member_trainer_id() or auth_member_trainer_id() is null)
  );
```

## 3. 트레이너앱 — 오운완 랭킹 (`components/views/OunwanRanking.jsx` 신규 · 읽기)

### 3.1 데이터 — RPC 1콜
- `supabase.rpc("ounwan_ranking")` → `[{ user_id, total, month_count, streak }]`(account 스코프·서버 집계). `ChurnRiskToday` 골격(useEffect 1회·데모/0건 숨김·try/catch/finally).
- **raw 테이블 무필터 조회 금지**(초안 ③버그) — RPC가 집계행만 반환. `members`(prop)로 이름·뷰 매핑.

### 3.2 표시 (트레이너 전용)
- **뷰 토글**: `[이번달 횟수(month_count)]` / `[연속일(streak)]` — 선택 기준 내림차순. (달성률 뷰는 주간목표 도입 후 후속.)
- 순위·회원명(`MemberBadge`)·지표·클릭 시 `onSelect(id)`. **회원 비노출**(회원앱엔 자기 진행률만).
- **이탈위험 시너지**: 오운완 저조 = `ChurnRiskToday` 대상 → 같은 '오늘' 스택에 나란히.

### 3.3 마운트 위치 — ★'내 실적'(tab 8 · MyStats) (개정)
- `components/views/MyStats.jsx` 안에 `<OunwanRanking members={members} onSelect={onSelect} />` 렌더.
- `app/page.jsx`: `<MyStats members={members} isSolo={isSolo} onSelect={(id)=>{setSelectedId(id); setTab(0);}} />`로 콜백 1개 전달 → MyStats가 OunwanRanking에 포워드.
- **왜 '오늘' 아님**: '오늘'은 앱 열 때마다 로드(이미 ChurnRiskToday 조회 2개) + N명 루프 RPC 부담 + 랭킹은 주1~2회 보는 정보 + '오늘' 잡음 최소화. 랭킹은 '내 실적' 성격에 맞음.

## 6. 주의 · 엣지
- **집계 = RPC만**: 클라 `Object.keys(activityMap)`로 누적/연속 세지 말 것(limit 창·감소·드리프트). `buildActivityMap`은 **달력·"오늘 여부"** 전용.
- **정의 단일화**(②): `ounwan_days` 한 함수가 유일한 정의. 규칙 바꾸면 여기만 수정 → 회원·트레이너 자동 일치.
- **★SECURITY DEFINER grant 금지(anon-open 급 경고)**: `ounwan_days`/`ounwan_stats_for`는 RLS 우회 → `grant execute`(authenticated/anon) 붙이면 임의 user_id로 타 회원 이력 유출. SQL에 배너 주석 있음(§1.2). **CLAUDE.md에 anon-open 금지와 같은 급으로 1줄 추가 권장**: "RLS 우회 SECURITY DEFINER 집계 함수(`ounwan_days`·`ounwan_stats_for`)에 grant execute 금지 — self/account 래퍼(`ounwan_stats`·`ounwan_ranking`)만 노출."
- **★교훈 — RLS 정책 안 서브쿼리도 RLS를 받는다**: 회원 대상 정책에서 `exists(select from user_table …)`는 **금지** — member-views가 회원의 user_table select 정책을 제거해 회원 세션은 user_table 0행 → 정책 영구 false(포상이 필드 다 맞아도 안 보였던 원인). 회원 스코프가 필요하면 `auth_member_id()`처럼 **SECURITY DEFINER 헬퍼**로 값을 뽑아 직접 비교(`auth_member_account_id()`·`auth_member_trainer_id()`). cardio/schedule 회원 정책이 멀쩡했던 건 서브쿼리 없이 `user_id = auth_member_id()` 직접 비교였기 때문.
- **뷰 보정 회원 가시 변화**(④): 노쇼 사라짐·날짜 이동 → 트레이너 사전 인지(문의 대비). 그래서 §1.1 단독 선커밋.
- **KST**: RPC가 `session_at at time zone 'Asia/Seoul'`로 통일. cardio/schedule은 `date`라 그대로.
- **solo trainer_id null**: 회원 포상 read가 null 허용(§1.3) — solo 포상 노출 보장.
- **honor-system**: 유산소·개인운동 자기신고는 완벽 검증 아님 — PT 검증·포상 재량으로 충분(설계 합의).
- **랭킹 공평성**: 순수 횟수는 고빈도 유리 → 연속일 뷰 병행 · 회원 비노출(하위권 사기저하 방지).
