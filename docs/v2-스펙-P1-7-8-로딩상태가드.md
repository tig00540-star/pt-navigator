## A. P1-7 — `useAccount` try/catch + 관리자 버튼 fail-closed

### A-0. 문제 (확인)
- `lib/useAccount.js:11,15` 두 await가 try/catch 밖 → 네트워크 reject 시 `setState` 미실행 → `loading` 영구 `true`. **[확인]**
- `app/page.jsx:698` `{!isSolo && (<a href="/admin">원장 대시보드</a>)}` — `isSolo`는 `accountType==="solo"`라 **조회 전/실패 시 `false` → `!isSolo` 참 → 1인 트레이너에게 admin 버튼 노출(fail-open)** + 매 로드 깜빡임. **[확인]**

## B. P1-8 — 로딩/저장 가드 스윕

### B-0. 두 캐논 패턴 (레포 기존 규율)
- **로더(useEffect fetch):** `supabase` 데모 가드 뒤의 await 구간을 `try`로 감싸고, 끝의 `setLoading(false)`를 **`finally { if (!cancelled) setLoading(false); }`**로 이동. 실패해도 스피너 해제 → 무한 로딩 방지.
- **저장(핸들러):** DB 호출 구간을 `try`로 감싸고 **`finally { setSaving(false); }`**(또는 `setPosting(false)`), inner 조기 `setX(false)` 제거. 보조 뮤테이션(toggle/remove·busy 없음)은 **`catch { setErr(...) }`**만.

> ⚠️ `if (cancelled) return;` / 조기 `return`이 try 안에 있어도 **finally는 실행**되므로 안전. 데모 가드(`if (!supabase) …`)는 try **밖**에 유지.

        setAnns((p) => [data[0], ...p]);
      }
      resetForm();
    } catch {
      setErr("게시 실패 — 네트워크 확인");
    } finally {
      setPosting(false);
    }
  };
