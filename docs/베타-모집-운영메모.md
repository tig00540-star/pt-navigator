# 베타 모집·운영 메모 (트레이너 10명 · 7일 무료)

> 목적: 핸드픽 트레이너 10명에게 7일 무료 체험 → 피드백 수집.
> 전제: **B1(구독 잠금) 완료·배포됨.** 신규 가입 = 자동 잠김(결제벽) → 내가 활성해야 사용.
> 핵심 원칙: **접근 통제는 "가입"이 아니라 "내가 활성화".** 그래서 가입 링크가 뿌려져도 무해.

---

## 1. 원칙 (왜 이렇게)

- **비번 공유 금지.** 각자 본인 이메일로 가입(데이터 격리·보안이 여기서 나옴).
- **가입 = 잠김.** 활성 안 하면 결제벽만 보임 → 아무나 가입해도 안전.
- **공개 배포 X, 개별 초대.** 고른 10명에게만 1:1로 가입 링크 전달.

---

## 2. 모집 (사전 준비)

- **후보 10명 리스트업** — 잘 아는·피드백 성실할·유형 다양한(경력·솔로/센터·연령대) 트레이너로.
- **개별 연락**(카톡/DM). 아래 초대 문구 복붙.

**초대 문구 템플릿:**

> 안녕하세요 OO님! 제가 PT 트레이너용 세일즈·회원관리 앱(오직 트레이너)을 만들어서 7일 무료 베타를 열었어요. OT·재등록 클로징 컨닝페이퍼, 음성으로 수업일지 자동작성, 회원앱(회원이 자기 인바디·운동 성과 봄) 같은 기능이에요. 편하게 써보시고 솔직한 피드백만 주시면 큰 도움이 됩니다.
> 가입 링크: https://pt-navigator.vercel.app/signup
> 가입하시고 쓰신 이메일만 알려주시면 제가 바로 열어드릴게요. (가입 직후엔 "구독 필요" 화면이 떠도 정상이에요 — 제가 활성화하면 열립니다.)

## 3. 활성화 절차 (트레이너 1명당)

1. 트레이너가 `https://pt-navigator.vercel.app/signup`에서 **가입** → "구독이 필요해요" 결제벽(정상).
2. **가입한 이메일 받기.**
3. **account_id 조회**(§4-A).
4. **활성 SQL**(§4-B) 실행 → +7일 premium.
5. "이제 됩니다, 앱 새로고침(하드리프레시)하세요" 안내.

---

## 4. 자주 쓰는 SQL (복붙 모음 · Supabase SQL Editor)

**A. 이메일로 account_id 찾기**
```sql
select a.id as account_id, u.email, a.subscription_status, a.current_period_end
from account a
join trainer t on t.account_id = a.id
join auth.users u on u.id = t.id
where u.email = '트레이너이메일@example.com';
```

**B. 활성 + 7일 체험**
```sql
update account
  set subscription_status = 'active', plan = 'premium',
      current_period_end = now() + interval '7 days'
where id = '<account_id>';
```

**C. 체험 연장(+7일 더)**
```sql
update account set current_period_end = now() + interval '7 days' where id = '<account_id>';
```

**D. 종료/비활성(체험 끝, 전환 안 함)**
```sql
update account set subscription_status = 'inactive' where id = '<account_id>';
```

**E. 유료 전환(장기 활성)** — 결제 자동화(B4) 전까지 수동
```sql
update account set subscription_status = 'active', plan = 'premium',
       current_period_end = now() + interval '30 days'   -- 결제 주기에 맞춰
where id = '<account_id>';
```

**F. 베타 현황 한눈에** (내 계정 제외 · 만료 임박순)
```sql
select u.email, a.subscription_status, a.plan, a.current_period_end,
       (a.current_period_end - now()) as time_left
from account a
join trainer t on t.account_id = a.id
join auth.users u on u.id = t.id
where a.id <> '11111111-1111-1111-1111-111111111111'   -- 파일럿 센터(내 계정) 제외
order by a.current_period_end;
```

> 7일 만료는 **자동**(로그인 시점에 `current_period_end`가 지났으면 자동 잠금 · 크론 불필요). 별도로 끌 필요 없음.

## 5. 온보딩 (전환율 핵심 — 10명이라 1:1이 먹힘)

활성 직후 짧게(전화/영상/직접) 핵심 4개만 같이 눌러보기:

1. **음성일지** — 수업 끝나고 말로 → 자동 정리 → 카톡 복사.
2. **회원 등록 → 링크 발급** → 회원이 자기 앱에서 인바디·운동 성과 그래프 봄.
3. **OT 사전무장** — 수업 전 3분 클로징 컨닝페이퍼.
4. **재등록 서포트** — 재등록 대상 회원 브리핑.

> "첫날 실제 수업 하나만 음성일지로 남겨보기"를 숙제로 주면 계속 씀(습관화).

## 9. 안전·운영 체크

- **활성/비활성 SQL은 `account_id`로만** 실행(이메일 오타로 엉뚱한 계정 건드리지 않게 §4-A로 먼저 조회).
- **내 계정(파일럿 센터 `11111111-…`)은 절대 건드리지 않기.** §4-F에서 제외해둠.
- **Anthropic 크레딧 Auto-reload 켜두기** — 베타 중 크레딧 소진되면 음성일지·OT가 데모로 떨어짐.
- **베타 종료 후** 안 쓰는 계정은 §4-D로 비활성(또는 정리) — 나중에 실데이터와 안 섞이게.
- 결제벽·활성이 이상하면: 트레이너에게 **하드리프레시** 먼저 안내(옛 번들 캐시).
