## 0. 목표 · 범위 · 대원칙

**목표:** 할일/파생 위젯 5개(ReapproachToday · RegisterDueToday · RegisterReapproachToday · UnclosedClosingToday · PastDueAppointments)에 **인라인 중복**된 카드/헤더/행 뼈대를 `components/ui` 프리미티브 **3종(Card · SectionHeader · ListRow)** 으로 추출. 앞으로 기능4 리포트·회원카드·공지가 같은 부품을 상속하게 만드는 첫 단추.

**대원칙 = 픽셀 동일(pixel-identical).** 이번 라운드는 **겉모습을 바꾸지 않는다.** 구조만 추출하고, 색·간격·타이포는 **각 위젯의 현재 클래스 문자열을 그대로 복제**한다. 톤별 미세 불일치(예: 미처리예약 테두리 `zinc-400`)도 **지금은 보존**하고, 통일 후보는 §6에 따로 모아 트레이너가 나중에 결정.

**범위(이번):** 신규 파일 4개(`components/ui/tone.js` · `Card.jsx` · `SectionHeader.jsx` · `ListRow.jsx`) + 위 5개 위젯의 **표현 JSX만** 교체. 각 위젯의 **데이터 로직(useEffect·판정·정렬·nameOf 등)·props 시그니처·default export는 무수정** → 호출자(`TodoTab`) 변경 0.

**작업 방식 = 부품 하나씩(part-by-part).** 3부품을 한 번에 갈아끼우지 않고, **부품 1개 → 5개 위젯 일괄 적용 → diff·폰 픽셀 동일 검증 → 서브커밋**을 3라운드. 각 라운드 diff가 작고, "이 부품만" 바뀌었는지 확인이 쉽다(§4).

## 2. 신규 파일 4개 (전체 소스)

> **purge 주의:** 아래 톤 문자열(특히 arbitrary value `bg-amber-500/[0.06]`)은 지금 위젯 파일에 있어서 Tailwind가 살려두는 것. 추출 후엔 이 리터럴이 **`tone.js` 안에만** 존재하게 된다. Tailwind v4는 프로젝트의 `.js`도 스캔하므로 유지되지만, **purge 실패는 lint에 안 잡히고 폰에서 배경/테두리가 사라지는 형태로만 보인다** → §5 검증에서 반드시 눈으로 확인.

### 2-1. `components/ui/tone.js` (신규)

```js
/* =========================================================================
   할일/파생 위젯 시맨틱 톤 — Card · SectionHeader · ListRow 공용 단일 출처.
   purge-safe 정적 리터럴만(동적 클래스 조립 금지 · MemberBadge VIEW_META 컨벤션).
   값 = 마이그레이션 전 각 위젯의 기존 클래스를 '픽셀 동일'하게 복제한 것(겉모습 불변).
   톤: amber(재접근) · zinc(미처리/중립) · rose(위험/미마감) · emerald(재등록/성공).
   ========================================================================= */
export const WIDGET_TONE = {
  amber: {
    card:         "border-amber-500/25 bg-amber-500/[0.06]",
    icon:         "text-amber-600",
    title:        "text-amber-700",
    countBadge:   "bg-amber-500/15 text-amber-700",
    rowBorder:    "border-amber-500/20",
    rowHover:     "hover:border-amber-400/50",
    chevron:      "text-amber-500/50",
    chevronHover: "group-hover:text-amber-600",
  },
  zinc: {
    card:         "border-zinc-400/25 bg-zinc-500/[0.06]",
    icon:         "text-sub",
    title:        "text-ink",
    countBadge:   "bg-elevate text-sub",
    rowBorder:    "border-line",
    rowHover:     "hover:border-primary",
    chevron:      "text-muted",
    chevronHover: "group-hover:text-primary-strong",
  },
  rose: {
    card:         "border-rose-500/25 bg-rose-500/[0.06]",
    icon:         "text-rose-600",
    title:        "text-rose-700",
    countBadge:   "bg-rose-500/15 text-rose-700",
    rowBorder:    "border-rose-500/20",
    rowHover:     "hover:border-rose-400/50",
    chevron:      "text-rose-500/50",
    chevronHover: "group-hover:text-rose-600",
  },
  emerald: {
    card:         "border-emerald-500/25 bg-emerald-500/[0.06]",
    icon:         "text-primary-strong",
    title:        "text-primary-strong",
    countBadge:   "bg-primary-soft text-primary-strong",

    chevron:      "text-emerald-500/50",

  return WIDGET_TONE[tone] || WIDGET_TONE.zinc;
}
```

### 2-2. `components/ui/Card.jsx` (신규)

```jsx
/* 파생 위젯 표준 컨테이너 — tone별 테두리+틴트 배경. 구조만 추출(픽셀 동일).
   기본 형태: mb-4 rounded-2xl border p-4 + 톤 색. shadow는 기본 없음(틴트 카드 결). */
import { widgetTone } from "@/components/ui/tone";

export default function Card({ tone = "zinc", className = "", children }) {
  const t = widgetTone(tone);
  return (
    <section className={`mb-4 rounded-2xl border p-4 ${t.card} ${className}`}>
      {children}
    </section>
  );
}
```

### 2-3. `components/ui/SectionHeader.jsx` (신규)

```jsx
/* 위젯 섹션 헤더 — 아이콘 + 제목 + (선택)카운트 배지 + (선택)힌트 한 줄. 픽셀 동일.
   icon = lucide 컴포넌트(소비처가 import해 넘김). count는 0도 표시(null/undefined만 생략). */
import { widgetTone } from "@/components/ui/tone";

export default function SectionHeader({ tone = "zinc", icon: Icon, title, count, hint }) {
  const t = widgetTone(tone);
  return (
    <div className="mb-3 flex items-center gap-2">
      {Icon && <Icon className={`h-4 w-4 ${t.icon}`} />}
      <h3 className={`text-sm font-semibold ${t.title}`}>{title}</h3>
      {count != null && (
        <span className={`rounded-full px-2 py-0.5 text-[10px] font-semibold ${t.countBadge}`}>
          {count}
        </span>
      )}
      {hint && <span className="text-[11px] text-muted">{hint}</span>}
    </div>
  );
}
```

### 2-4. `components/ui/ListRow.jsx` (신규)

```jsx
/* 회원명 + 컨텍스트 + Chevron 탭 행(재접근·미처리예약·재등록 공통 꼴). 픽셀 동일.
   name = 노드 허용(문자열 또는 '이름 미상' muted span). onClick 필수.
   children = 이름 아래 메타 라인 '전체'를 소비처가 자기 wrapper째로 넘긴다
             (메타 wrapper가 위젯마다 미세하게 달라 여기서 고정하지 않음 — §6-c). */
import { ChevronRight } from "lucide-react";
import { widgetTone } from "@/components/ui/tone";

export default function ListRow({ tone = "zinc", name, onClick, children }) {
  const t = widgetTone(tone);
  return (
    <button
      onClick={onClick}
      className={`group flex items-center justify-between gap-3 rounded-xl border bg-card px-3 py-2.5 text-left shadow-sm transition active:scale-[0.99] ${t.rowBorder} ${t.rowHover}`}
    >
      <div className="min-w-0">
        <div className="text-sm font-semibold text-ink">{name}</div>
        {children}
      </div>
      <ChevronRight className={`h-4 w-4 shrink-0 ${t.chevron} ${t.chevronHover}`} />
    </button>
  );
}
```

> **클래스 순서 안내:** 원본은 `border border-amber-500/25 ...` 처럼 base와 색이 붙어 있고, 추출본은 base를 앞에, 톤 색을 뒤 보간에 둔다. **CSS는 클래스 '속성 순서'가 아니라 스타일시트 규칙 순서로 적용**되므로, 클래스 나열 순서가 달라도 렌더 결과는 동일하다(픽셀 동일 유지). `border`(1px 실선 base) + `border-{tone}`(색) 둘 다 존재하는지만 확인.

## 4. 서브커밋 순서 (부품 하나씩 · 각 라운드 폰 픽셀 동일 검증)

> 각 라운드는 **부품 1개 파일 추가 + 5개 위젯의 해당 부분만 교체**. `tone.js`는 B-1a에서 함께 추가(세 부품 공용이라 먼저 만들어도 무해).

### ▷ B-1a — Card (컨테이너)

**신규:** `components/ui/tone.js` · `components/ui/Card.jsx`.
**교체(5파일):** 각 위젯 최상단 `<section className="mb-4 rounded-2xl border border-… bg-… p-4">` → `<Card tone="…">`, 닫는 `</section>` → `</Card>`. 상단에 `import Card from "@/components/ui/Card";` 추가.

BEFORE/AFTER 예 (ReapproachToday):
```jsx
// BEFORE
<section className="mb-4 rounded-2xl border border-amber-500/25 bg-amber-500/[0.06] p-4">
  {/* ...헤더+그리드... */}
</section>

// AFTER
<Card tone="amber">
  {/* ...헤더+그리드 그대로... */}
</Card>
```
톤 매핑: Reapproach=`amber` · RegisterDue=`emerald` · RegisterReapproach=`emerald` · Unclosed=`rose` · PastDue=`zinc`.
**검증 포인트:** 5개 카드의 배경 틴트·테두리 색이 이전과 동일(특히 PastDue의 `zinc-400/25` 테두리 + `zinc-500/[0.06]` 배경 조합 보존).
**커밋:** `refactor(ui): Card 프리미티브 추출 + 파생 위젯 5개 컨테이너 이관`

### ▷ B-1b — SectionHeader (헤더)

**신규:** `components/ui/SectionHeader.jsx`.
**교체(5파일):** 각 위젯의 `<div className="mb-3 flex items-center gap-2"> …아이콘/h3/카운트/힌트… </div>` 블록 → `<SectionHeader … />` 한 줄. `import SectionHeader from "@/components/ui/SectionHeader";` 추가.

BEFORE/AFTER 예 (ReapproachToday):
```jsx
// BEFORE
<div className="mb-3 flex items-center gap-2">
  <CalendarClock className="h-4 w-4 text-amber-600" />
  <h3 className="text-sm font-semibold text-amber-700">오늘 재접근</h3>
  <span className="rounded-full bg-amber-500/15 px-2 py-0.5 text-[10px] font-semibold text-amber-700">{list.length}</span>
```

### ▷ B-1c — ListRow (탭 행)

**신규:** `components/ui/ListRow.jsx`.
**교체(5파일):** `<div className="grid gap-2">`(그리드 래퍼) 안의 `<button …> …이름/메타/Chevron… </button>` → `<ListRow …> {메타} </ListRow>`. `import ListRow from "@/components/ui/ListRow";` 추가. **그리드 래퍼 `<div className="grid gap-2">`·`.map()`·`key`·`over`/`ago` 계산은 유지**(단 `key`는 ListRow로 이동).

BEFORE/AFTER 예 (ReapproachToday, `.map` 콜백 내부):

**위젯별 주의 (메타 라인은 소비처가 wrapper째 그대로 children으로 — 픽셀 동일 위해 건드리지 말 것):**
- **PastDueAppointments:** 메타 wrapper가 `mt-0.5 flex flex-wrap items-center gap-x-2 text-[11px] text-sub`로 **`gap-y-0.5`가 없다** — 원본대로 유지(통일 후보 §6-c). `name`엔 `nameOfEl(a.user_id)`(노드) 그대로 넘김. `ChevronRight` import는 이제 ListRow가 쓰므로 위젯에서 **미사용이 되면 제거**(lint `no-unused-vars` 대비 — §5).
- **UnclosedClosingToday:** 메타가 flex가 아니라 단일 줄 `<div className="mt-0.5 text-[11px] text-sub">2차 클로징 결과 입력</div>` — 그대로 children.
- **RegisterDueToday:** children = `<div className="mt-0.5 flex flex-wrap items-center gap-x-2 gap-y-0.5 text-[11px] text-sub"><span>잔여 유료 <b className="text-primary-strong">{r.paid}</b></span><span className="text-muted">· 서비스 {r.service}</span></div>`.
- **RegisterReapproachToday:** Reapproach와 동형(`reg_reapproach_at` · `over` · `· 재등록 보류`), tone `emerald`.
- 5파일 모두 최상단 `import { ChevronRight } ...`가 **ListRow로 이동**하며 위젯에서 미사용이 되므로 **해당 import 정리**(개별 확인). CalendarClock 등 SectionHeader용 아이콘 import는 **유지**(icon prop으로 여전히 씀).

**검증 포인트:** 행 테두리·hover 테두리·chevron 색/hover·active 눌림·이름/메타 타이포가 톤별로 이전과 동일. amber/rose 행은 hover가 동색 계열, zinc·emerald 행은 hover가 `border-primary`(초록)인 차이 유지.

## 6. 미세 통일 후보 (이번 X · 트레이너 결정용 백로그)

픽셀 동일 원칙상 **이번엔 건드리지 않음.** 나중에 "미세 통일" 서브커밋으로 한 개씩 결정:

- **(a) 제목 색 규칙:** amber/rose=`-700`, emerald=`text-primary-strong`, zinc=`text-ink`로 제각각. 감사 §2-6(시맨틱 색 토큰화)에서 "제목은 톤 강조색 통일" 같은 규칙으로 정리 가능.
- **(b) 카운트 배지 배경식:** amber/rose=`bg-{tone}-500/15`, emerald=`bg-primary-soft`, zinc=`bg-elevate`로 공식이 다름 → 하나로.
- **(c) 메타 wrapper `gap-y-0.5`:** 미처리예약만 빠져 있음(2줄 wrap 시에만 차이). ListRow에 `meta` 편의 슬롯을 두고 `flex flex-wrap … gap-y-0.5` 기본 제공하는 방향.
- **(d) PastDue 카드 테두리 `zinc-400` vs 배경 `zinc-500`:** 톤 내 명도 불일치 — `zinc-500/25`로 맞출지 검토.
- **(e) 확장:** TodoManual(중립 카드=`border-line bg-card shadow-sm`, 힌트 없음) · MyStats StatTile · 회원 카드도 같은 Card/SectionHeader 소비처. `neutral` 톤(shadow 포함) 변주 추가로 다음 라운드 흡수 → 기능4 리포트가 상속.
