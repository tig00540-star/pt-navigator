## 0. 배경 — 왜 "즉효 묶음"만인가

디자인 감사(`v2-디자인-감사-폴리시.md`)의 **P0는 프리미티브 추출(B) 스프린트에서 이미 대부분 반영됨**을 코드 확인으로 검증했다:

- P0-① 트레이너 이메일→이름: ✅ `ScheduleBoard`가 셀·범례·필터 전부 `personName()` 적용.
- P0-② 이름 폴백 "회원"→muted "이름 미상": ✅ `ScheduleBoard.memberNameEl`·`PastDueAppointments.nameOfEl`.
- P0-③ 빈값 대시 남발: ✅ `MemberListTab`이 `hasVal()`로 빈 라인 숨김, 목표는 muted `Chip`.
- P0-⑤ 금액 자간: ✅ `MyStats` 전부 `tabular-nums`(과한 tracking 없음 · 폰 육안만 재확인).

**따라서 이 스펙 = 아직 안 된 즉효 4건**(작고 회귀 낮고 바로 티나는 것). 나머지 P1(⑦ Button 프리미티브 · ⑧ 타이포 스케일 · ⑩ 섹션헤더 통일)은 **후속 스펙에서 순차**.

이 스펙의 4건:

| # | 항목 | 파일 | 성격 |
|---|---|---|---|
| A | AA 대비 — `muted` 토큰 한 톤 진하게 (P1-⑪) | `app/globals.css` | 1줄 |
| B | 죽은 `C` 토큰 객체 삭제 | `app/page.jsx` | 삭제 |
| C | 헤더 회원 셀렉트에 보이는 라벨 (P1-⑨) | `app/page.jsx` | 소규모 |
| D | 스케줄 모바일 = 좁은 폭 기본 '오늘' 뷰 (P0-④ 잔여) | `components/views/ScheduleBoard.jsx` | 소규모 |

**커밋 단위(권장):** A → B → C → D 순, **항목별 서브커밋**(A·B·C는 근접 파일이라 원하면 A / B+C / D 로 3커밋도 OK). 각 커밋 후 `npm.cmd run lint` → `git show --stat HEAD`.

## A. AA 대비 — `muted` 토큰 한 톤 진하게 (P1-⑪)

### 현재
`app/globals.css`:
```css
  --color-muted: #94a3b8;         /* 흐린 텍스트(부차 정보만) */
```

### 문제 (실측)
흰 카드(`#ffffff`) 위 `text-muted` 대비 = **2.57:1** → WCAG AA(본문 4.5:1) **미달**. 페이지 배경(`#eef1f5`) 위에선 더 낮다. `muted`는 급여 카드의 `이달 예상 급여` 라벨(uppercase), 날짜, 힌트, 10px 배지 등 **실제 정보 텍스트**에 광범위하게 쓰여 장식이 아니다. 현장(밝은 헬스장 조명)에서 폰 가독성에 직접 영향.

### 목표
`#94a3b8` → **`#64748b`**(slate-500). 흰 카드 위 대비 = **4.77:1**(AA 통과), 배경 `#eef1f5` 위 ≈ **4.5:1**. 색상은 슬레이트 계열 그대로라 톤 인상 불변(딱 한 단계 진해질 뿐).

### diff 지침
`app/globals.css` 해당 한 줄만:
```css
  --color-muted: #64748b;         /* 흐린 텍스트(부차 정보만) · AA 4.5:1 확보(흰 카드 위 4.77:1) */
```

- `--color-sub`(#475569, 흰 위 7.58:1)·`--color-line`·`--color-ink` 등 **다른 토큰은 건드리지 않는다.** `sub`은 이미 AA 통과라 유지 → `sub`(더 진함) > `muted`(한 톤 연함)의 위계도 그대로 유지된다.
- `placeholder-muted`(입력창)·`text-line group-hover:text-muted`(스케줄 빈칸 Plus 아이콘) 등은 **더 진해지는 방향**이라 안전(가독성만 개선).

### 검증
- lint 무관(CSS 토큰).
- 폰 육안: 급여 카드 부제·날짜·힌트·비활성 탭 라벨이 이전보다 또렷한지. 회귀(너무 진해 강조처럼 보임)는 없어야 함 — slate-500은 여전히 명백한 보조 톤.

## B. 죽은 `C` 토큰 객체 삭제

### 현재
`app/page.jsx` 약 104–145줄: **다크테마 잔재** 색 토큰 객체.
```jsx
/* =========================================================================
   PURGE-SAFE COLOR TOKENS
   ========================================================================= */

const C = {
  sky:    { text: "text-sky-400", soft: "bg-sky-500/10", ... },
  lime:   { ... },
  orange: { ... },
  emerald:{ ... },
};
```

### 문제
`text-sky-400`·`bg-sky-500/10` 등 **다크테마 시절 클래스**로, 라이트테마 이관 후 **참조 0건**(검증: `page.jsx` 내 `C.sky`/`C.lime`/`C.orange`/`C.emerald` 사용 없음, `C` 심볼은 정의 1곳뿐). 남아있으면 후속 색 작업 때 혼동·오사용 위험.

### diff 지침
- `app/page.jsx`에서 **`const C = {...}` 블록 전체 + 바로 위 `PURGE-SAFE COLOR TOKENS` 주석 헤더**를 삭제한다.
- (선택) 그 아래 내용 없는 스캐폴딩 주석 블록(`SMALL PIECES`, `TAB 2 — 2차 OT 내비게이터`, `신규 회원 사전 정보 등록 폼` 위의 빈 구분 주석 등)도 정리 가능 — 단 **실제 코드가 붙은 주석은 남길 것**. 애매하면 `C` 블록만 지우고 나머지는 손대지 않는다(안전 우선).
- `GROUP_TAB`·`viewMeta`·`MemberBadge` 등 실제 쓰이는 상수/컴포넌트는 **건드리지 않는다**.

### 검증
- `npm.cmd run lint` — no-unused-vars 등 통과(오히려 죽은 코드 제거).
- `Select-String -Path app\page.jsx -Pattern "\bC\."` → 매치 0건 확인(삭제 누락 없음).
- 화면 회귀 없음(미사용이었으므로 렌더 무변).

## C. 헤더 회원 셀렉트에 보이는 라벨 (P1-⑨)

### 현재
`app/page.jsx` 헤더 우측(약 684–697줄). 선택된 회원 드롭다운에 `aria-label="회원 선택"`만 있고 **눈에 보이는 라벨이 없어** 무엇을 고르는지 즉시 안 읽힘(계정/트레이너 선택과 혼동 소지).
```jsx
{members.length > 0 && (
  <select
    value={selectedId || ""}
    onChange={(e) => setSelectedId(e.target.value)}
    className="max-w-[110px] rounded-lg border border-line bg-elevate px-2 py-1.5 text-xs text-sub outline-none focus:border-primary sm:max-w-[130px]"
    aria-label="회원 선택"
  >
    {members.map((m) => (
      <option key={m.id} value={m.id}>
        {m.name}
      </option>
    ))}
  </select>
)}
```

### 목표
앞에 **User 아이콘 + "회원" 라벨**(좁은 폭에선 아이콘만)을 붙여 "이건 회원 선택"임이 즉시 읽히게. 헤더 공간이 빡빡하므로 라벨 텍스트는 `sm` 이상에서만 노출. `User` 아이콘은 이미 import되어 있음(page.jsx 상단).

### diff 지침
위 블록을 아래로 교체:
```jsx
{members.length > 0 && (
  <label className="flex items-center gap-1 rounded-lg border border-line bg-elevate pl-2 pr-1 focus-within:border-primary">
    <User className="h-3.5 w-3.5 shrink-0 text-muted" aria-hidden />
    <span className="hidden text-[10px] font-medium text-muted sm:inline">회원</span>
    <select
      value={selectedId || ""}
      onChange={(e) => setSelectedId(e.target.value)}
      className="max-w-[88px] bg-transparent py-1.5 pr-1 text-xs text-sub outline-none sm:max-w-[120px]"
      aria-label="회원 선택"
    >
      {members.map((m) => (
        <option key={m.id} value={m.id}>
          {m.name}
        </option>
      ))}
    </select>
  </label>
)}
```

- 테두리·배경·포커스 링을 바깥 `<label>`로 옮기고(`focus-within:border-primary`), `<select>`는 `bg-transparent`·보더 제거. 겉모습은 아이콘+라벨이 추가된 것 외 동일 계열.
- `aria-label`은 유지(중복이라도 무해). `<label>`이 select를 감싸므로 접근성도 향상.
- 아이콘/라벨 추가로 폭이 늘므로 `max-w`를 110→88(sm 130→120)로 살짝 줄여 헤더 우측(신규등록·공지벨·관리자 버튼)과 안 부딪히게. **폰 실기기에서 헤더 한 줄 유지 여부 확인**(줄바꿈/오버플로우 없어야 함).

### 검증
- `npm.cmd run lint` 통과(`User` 이미 사용 중).
- 폰: 헤더 우측이 한 줄에 들어오는지, "회원" 드롭다운이 다른 버튼과 안 겹치는지. 데스크톱: `👤 회원 [○○ ▾]`로 읽히는지.

## D. 스케줄 모바일 = 좁은 폭 기본 '오늘' 뷰 (P0-④ 잔여)

### 현재
`components/views/ScheduleBoard.jsx`:
```jsx
const [mode, setMode] = useState("week"); // 'week' | 'today'
```
주간 7열 그리드는 이미 **가로스크롤(`overflow-x-auto`) + 시간열 sticky(`sticky left-0`)**로 감사 대안 일부를 충족한다. 하지만 **항상 주간으로 시작**해, 폰(~390px)에선 첫 화면이 7열 압박 상태다. `오늘`(일 뷰) 토글은 이미 존재.

### 목표
**좁은 폭(모바일)에서 첫 진입 기본을 '오늘' 뷰로.** 넓은 폭은 주간 유지. 토글은 그대로 두어 사용자가 언제든 전환 가능(자동 전환은 **마운트 1회만**, 이후 사용자 선택 존중).

### diff 지침
`useState` 초기값은 `"week"` 그대로 두고(SSR 하이드레이션 안전), **마운트 1회 effect**로 좁은 폭이면 `today`로 전환. 다른 `useEffect`들 근처(예: 트레이너 목록 fetch effect 아래)에 추가:
```jsx
// 좁은 폭(모바일)에선 기본을 '오늘(일)' 뷰로 — 주간 7열이 폰에서 압박(감사 P0-④).
// 마운트 1회만(빈 deps) → 이후 사용자 토글 존중. window는 클라 전용이라 lazy init(하이드레이션 불일치) 대신 effect.
useEffect(() => {
  // eslint-disable-next-line react-hooks/set-state-in-effect
  if (typeof window !== "undefined" && window.innerWidth < 640) setMode("today");
}, []);
```

- 브레이크포인트 `640`(Tailwind `sm`)로 통일. 태블릿/데스크톱(≥640)은 주간 유지.
- **주의(하이드레이션):** `useState(() => window... )` 식 lazy init은 서버="week"/클라="today"로 마운트 mismatch를 유발하므로 **쓰지 말 것.** effect 방식이라 모바일에서 주간→오늘 순간 전환(1프레임 깜빡임) 가능 — 허용 범위.
- `react-hooks/set-state-in-effect` 룰은 저장소 컨벤션대로 인라인 `eslint-disable-next-line`으로 처리(page.jsx `loadMembers`/뷰 보정 effect와 동일 패턴).
- 토글 버튼·주간 그리드 로직은 **불변**(초기 모드만 바뀜).

### 검증
- `npm.cmd run lint` 통과.
- 폰: 스케줄 첫 진입이 '오늘' 뷰인지, `주간` 눌러 주간 전환·가로스크롤·시간열 sticky 정상인지. 데스크톱: 여전히 주간으로 시작하는지.
- (선택) 브라우저 좁혀서(<640) 새로고침 시 오늘 뷰로 뜨는지.
