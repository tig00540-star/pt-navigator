## 0. 이번 세션 성과

### A. 기능1 앱내 공지 — A~D 전부 배포 (origin/main)
**한 줄:** 원장→트레이너 단방향 앱내 공지. **공지마다 원장이 강도 선택**(`must_ack`): **일반**=헤더 벨 배지만, **필수확인**=앱 열 때 전체 덮는 게이트 모달(확인 눌러야 넘어감). 읽음은 서버 기록이라 기기 무관 다시 안 뜸.

- **설계 결정(합의):** 원장 작성만(RLS `auth_is_owner()`) · 타겟팅(`target_trainer_ids uuid[]`, null=전체) · 읽음 테이블 신설 · 필수확인 게이트는 `must_ack=true`만 발동 · 벨 열람은 **일반 공지만** 자동읽음(필수확인은 게이트 확인으로만).
- **마이그레이션:** `docs/migrations/2026-07-12-announcement.sql` — announcement(+must_ack) · announcement_read(unique(ann,trainer)) · RLS 4정책(ann_select 대상+원장 · ann_write 원장만 · annread_insert 본인 · annread_select 본인+원장). **Supabase 실행 완료**(auth_is_owner() 존재 확인함).
- **lib:** `lib/announce.js` — `isTargeted` · `unreadAnnouncements`(벨 배지) · `mustAckUnread`(게이트) · `visibleAnnouncements`(재열람). 순수·검증 완료.
- **UI:** `components/AdminAnnouncements.jsx`(원장 작성·목록·N/M 읽음, admin 배선) · `components/AnnouncementGate.jsx`(게이트 강제 + 벨 재열람 2모드, page.jsx 마운트 + 헤더 벨 배지).
- **커밋:** `1e29a57`(A 마이그) · `ba5cf4a`(B lib) · `a8c4f01`(C 원장 UI) · `43b9104`(D 게이트+벨). **배포·동작 확인**(/admin에 공지 UI·필수확인/핀 공지 라이브 확인).
- **남은 것 = 폰 왕복 테스트만:** ① 필수확인 게시→대상 폰 게이트(배경/ESC 안 닫힘)→확인→새로고침 안 뜸 ② 일반 게시→게이트 없이 벨 배지+1→벨 열면 배지 0 ③ 비대상·원장 본인 안 뜸 ④ 원장 "N/M 읽음" 증가.

### B. 디자인 감사 (A라운드 · `docs/v2-디자인-감사-폴리시.md`)
Claude in Chrome으로 배포본 5화면 라이브 캡처(스케줄·할일·회원·내실적·admin/공지). **미커버(2라운드 대기): OT/PT AI 탭 + 정밀 모바일**(도구 resize가 캡처에 반영 안 됨 → 폰 실기기 스크린샷 필요).
- **P0 즉효:** ①트레이너가 이메일로 표시(→이름; 근본=trainer.name 이메일 시드) ②할일 미처리예약 이름 폴백 "회원"(→실명 조인/‘이름 미상’ muted) ③회원카드 빈값 `-` 남발(→숨김) ④스케줄 7열 그리드 폰 압박(→좁은 폭 일뷰 기본/가로스크롤+시간열 고정) ⑤금액 자간 과다(→tabular-nums).
- **P1 일관성:** 시맨틱 색 역할 토큰화 · 버튼 변주 정리 · 타이포 스케일 · 헤더 회원셀렉트 라벨 · 섹션헤더 통일 · 대비 AA.
- **P2 근본(=B 디자인시스템):** 반복 인라인 중복(Card·SectionHeader·Badge·Chip·StatTile·ListRow·EmptyState·Modal·Button) → `components/ui` 추출. 추출 후 남은 기능이 폴리시 상속.

### C. IA(정보구조) 재정리 — 확정 (`pt-navigator-IA.html` 시안)
**진단:** 탭 한 줄에 "나 중심"(스케줄·할일·회원·내실적)과 "회원 중심"(OT/PT 워크플로우)이 섞여 최대 7탭 + 상태별 탭이 나타났다 사라짐 = 흩어짐.
**확정 방향:**
- **① "오늘" 통합** — 스케줄 + 오늘 할일을 한 화면으로(시작점 하나).
- **② Phase 1만 먼저** — 평평한 탭 유지하되 재정렬·묶음. Phase 2(회원 워크스페이스 구조)는 폰 체감 후 판단.
- **목표형(참고):** Level 1 글로벌 3탭(오늘·회원·내실적)+헤더(공지🔔·신규등록·관리자) ↔ Level 2 회원 워크스페이스(OT: 1차지원→관찰→2차 / PT: 운동일지·인바디·재등록).

## 1. 즉시 다음 액션 (다음 채팅)

1. **디자인 Phase 1(IA 재정렬) 스펙 → 클로드코드.** 평평한 탭 유지, TABS 배열·라벨·구분선만:
   - 상단 왼쪽 고정: **오늘 · 회원 · 내 실적**. "오늘"=스케줄+할일 통합(스케줄 보드 아래 오늘 재접근·미처리예약·메모).
   - 회원 선택 시 회원 탭을 **여정 순서**로: OT = 1차 지원 → 관찰 → 2차 / PT = 운동일지 · 인바디 · 재등록.
   - OT묶음(amber)·PT묶음(sky) 구분선+색으로 시각 그룹. 구조 변경 없음(회귀 낮음).
2. **디자인 폴리시 P0(즉효 5개) → 클로드코드.** 트레이너 이름 표기 → 이름폴백 → 빈값 → 금액자간 → 스케줄 모바일 순. 작은 서브커밋.
3. **감사 2라운드** — 폰 실기기 스크린샷(스케줄·2차 OT·회원·운동일지)로 OT/PT 탭 + 모바일 정밀 감사.
4. **기능 4 월간 리포트** — IA/폴리시 정리 뒤 착수(정돈된 시스템 위에 = 폴리시 상속). 그다음 3 FC → 5 회원앱.
5. **디자인시스템 B(ui 추출)** — Phase 1/폴리시 뒤, 별도 스펙. 이후 C(claude.ai/design) 단일 소스.
6. **페이롤 `time_slot` 자동**(짬) · **파일럿 D-3 케이스 5건**(코드 아님·축적).
