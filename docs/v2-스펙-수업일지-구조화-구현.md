## 0. 무엇을 만드나 (한 줄)

음성일지 AI가 이미 뽑는 종목·수치를 **구조화(숫자) 데이터로 저장까지 관통**시키고, 트레이너가 **저장 전 세트 그리드로 보정**한 뒤, PT 운동일지 탭에 **종목별 무게 추이**(인바디 추이 패턴 재사용)를 보여준다. → (다음 스프린트) 회원앱 종목별 무게 그래프.

**핵심 배경(왜 배선이 필요한가):** 지금 `voice-log`는 `machines:[{name, detail, method}]`를 뽑지만 `detail`이 **자유텍스트**("20kg×12·30kg×12")이고, 저장 시 `VoiceLogTab → handleVoiceResult(raw, summaryText)`가 **평문 요약만** 넘겨 `daily_workout_log.ai_summary`(텍스트)로 들어간다. 즉 **구조화 배열은 화면 렌더 후 소실**된다. ③은 이 데이터를 (a)숫자로 강화하고 (b)저장까지 관통시키는 게 본질.

## 2. 데이터 모양 (계약)

### 2.1 `daily_workout_log.sets_structured` (jsonb, nullable)

```json
[
  { "exercise": "벤치프레스", "sets": [ {"weight":20,"reps":12}, {"weight":30,"reps":12} ] },
  { "exercise": "랫풀다운",  "sets": [ {"weight":40,"reps":15} ] }
]
```

- `exercise` (string, required·non-empty): **canonical 종목명**(집계 키). 그리드에서 편집 가능.
- `sets` (array): 각 `{weight, reps}`.
  - `weight`: kg 숫자 또는 **null**(맨몸운동·중량 미언급).
  - `reps`: 반복 횟수 숫자 또는 **null**(미언급).
  - weight·reps 둘 다 null인 세트는 저장 전 제거(무의미).
- 전체가 `null` 또는 `[]` = 구조 데이터 없음(노쇼·손입력 미기재). **그래프 원천에서 제외.**

> `ai_summary`(텍스트)와 **공존**: `ai_summary`=카톡·회원 타임라인용 사람 텍스트(종전 불변), `sets_structured`=그래프용 숫자. 서로 대체하지 않음. voice-log의 `detail`(사람용 텍스트)도 종전대로 유지.

### 3-C. `lib/workout.js` — 순수 헬퍼 (신규 파일)

```js
// 수업 구조화 데이터 헬퍼(순수 함수 · UI 무의존). PtWorkoutTab·(Phase2)회원앱 공용.

// voice-log report.machines → sets_structured 배열. exercise·세트 없는 항목 제거.
export function machinesToStructured(machines) {
  if (!Array.isArray(machines)) return [];
  return machines
    .map((m) => ({
      exercise: (m?.canonical || m?.name || "").trim(),
      sets: (Array.isArray(m?.sets) ? m.sets : []).filter(
        (s) => s?.weight != null || s?.reps != null
      ),
    }))
    .filter((m) => m.exercise && m.sets.length);
}

// 저장 직전 정리 — 빈 exercise·빈 세트 제거, 숫자화. 그리드 편집 결과에도 적용.
export function cleanStructured(list) {
  if (!Array.isArray(list)) return [];
  const num = (v) => (v === "" || v == null || !Number.isFinite(Number(v)) ? null : Number(v));
  return list
    .map((ex) => ({
      exercise: (ex?.exercise || "").trim(),
      sets: (Array.isArray(ex?.sets) ? ex.sets : [])
        .map((s) => ({ weight: num(s?.weight), reps: num(s?.reps) }))
        .filter((s) => s.weight != null || s.reps != null),
    }))
    .filter((ex) => ex.exercise && ex.sets.length);
}

// 한 세션·한 종목의 대표 무게 = 최고 중량 세트(진행 지표). 없으면 null.
export function topSetWeight(sets) {
  const w = (Array.isArray(sets) ? sets : []).map((s) => s?.weight).filter((v) => v != null);
  return w.length ? Math.max(...w) : null;
}

// logs(daily_workout_log 행들) → 종목별 날짜순 추이.
// [{ exercise, points:[{ date, topWeight, sets }] }] · 오래된→최신. voided·빈 것 제외.
export function buildExerciseSeries(logs) {
  const byEx = new Map();
  const sorted = [...(logs || [])]
    .filter((l) => !l.voided && Array.isArray(l.sets_structured) && l.sets_structured.length)
    .sort(
      (a, b) =>
        new Date(a.session_at ?? a.created_at ?? 0) - new Date(b.session_at ?? b.created_at ?? 0)
    );
  for (const l of sorted) {
    const date = l.session_at ?? l.created_at ?? null;
    for (const ex of l.sets_structured) {
      const name = (ex?.exercise || "").trim();
      if (!name) continue;
      if (!byEx.has(name)) byEx.set(name, []);
      byEx.get(name).push({ date, topWeight: topSetWeight(ex?.sets), sets: ex?.sets || [] });
    }
  }
  // 최근 활동순(마지막 date 내림차순)으로 종목 정렬.
  return [...byEx.entries()]
    .map(([exercise, points]) => ({ exercise, points }))
    .sort((a, b) => new Date(b.points.at(-1)?.date ?? 0) - new Date(a.points.at(-1)?.date ?? 0));
}
```

### 3-D. `components/ui/Sparkline.jsx` — 공용 스파크라인 (신규 파일)

`PtInbodyTab`·`app/m/[token]/page.jsx`에 있는 스파크라인과 **동일 구현**을 공용화(신규 종목 추이 + Phase2 회원앱 재사용). 기존 두 파일은 이번엔 **건드리지 않음**(회귀 위험 회피 · 추후 통합은 별도).

```jsx
// 미니 스파크라인 — 값 추이(오래된→최신, 좌→우). 값 2개 미만이면 null.
// PtInbodyTab 패턴 공용화. 축·격자·호버 없음(수치는 리스트가 담당).
export default function Sparkline({ values }) {
  const w = 100, h = 28, pad = 3;
  const pts = (values || []).filter((v) => v != null);
  if (pts.length < 2) return null;
  const min = Math.min(...pts), max = Math.max(...pts), span = max - min || 1;
  const stepX = w / (pts.length - 1);
  const d = pts
    .map((v, i) => {
      const x = i * stepX;
      const y = h - pad - ((v - min) / span) * (h - pad * 2);
      return `${i ? "L" : "M"}${x.toFixed(1)},${y.toFixed(1)}`;
    })
    .join(" ");
  return (
    <svg viewBox={`0 0 ${w} ${h}`} width="100%" height={h} preserveAspectRatio="none" className="mt-1 text-primary/70">
      <path d={d} fill="none" stroke="currentColor" strokeWidth="2" strokeLinecap="round" strokeLinejoin="round" vectorEffect="non-scaling-stroke" />
    </svg>
  );
}
```

> delta 톤: 인바디의 `deltaTone`은 `goodDir` 기반이라 여기 안 씀. 무게는 **증가=good** 단순 규칙(로컬 헬퍼): `d>0→good, d<0→bad, 0→flat`. `DELTA_TONE` 맵(`{good:"text-primary-strong", bad:"text-rose-600", flat:"text-muted"}`)은 공유 import가 아니므로 **PtWorkoutTab 상단에 동일 값으로 const 선언**(purge-safe · 동적 조립 금지).

## 7. 주의·엣지

- **`ai_summary` 불변식:** 카톡 전송·회원 타임라인·회차 계산이 `ai_summary`/로그수에 의존 → 텍스트 경로는 절대 건드리지 말 것. `sets_structured`는 **부가 칸**.
- **UPDATE 정책 없음(의도):** 저장 전 보정만 하므로 `daily_workout_log` UPDATE 정책 추가 안 함(교훈1). 과거 기록 세트 수정 UI는 이번 범위 밖(원하면 별도 스펙 — UPDATE 정책 선행).
- **jsonb 반환 타입:** Supabase는 jsonb를 파싱된 JS 배열로 반환 → `Array.isArray(log.sets_structured)` 가드로 충분. 문자열로 오면(구버전) 방어적으로 무시.
- **진행 지표 = 최고 중량 세트(topWeight):** 가장 직관적("20→30kg"). 볼륨(Σwt×reps)·1RM 추정은 후속 옵션.
- **canonical 드리프트:** AI가 가끔 다르게 부르면 트레이너가 그리드에서 이름 통일 → 다음부터 같은 키. 사전 강제 매핑은 과설계라 보류.
