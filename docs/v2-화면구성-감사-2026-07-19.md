## 0. 한 줄 요약

기능이 어지러운 게 아니라, **① 바닥에서 고정 바 2개가 충돌하고 ② 브랜드 레드가 6색으로 갈라졌고 ③ 봐야 할 정보가 제일 작은 글씨이고 ④ 긴 한글 라벨 2열 폼이 세로로 어긋난다** — 이 네 뿌리 원인이 화면마다 반복돼서 "군데군데 많다"로 체감된다. 큰 리팩터 없이 배치·표시만 정리하면 해소된다.

## 1. 반복되는 뿌리 원인 (이게 "군데군데"의 정체)

### 뿌리① — 바닥에서 고정 요소 2개가 하단탭바와 충돌 **[H]**
- **`FirstOTTab.jsx` 현장메모 입력바** = `fixed inset-x-0 bottom-0 z-30`. 전역 **`BottomNav`** = `fixed inset-x-0 bottom-0 z-40`. **둘 다 bottom-0** → 메모바가 하단탭바 뒤로 겹쳐 눌린다(z-30<z-40). 하단탭바(앱느낌 2단계)를 나중에 붙이면서 이 탭의 옛 고정 메모바를 안 올린 **회귀**. → 7:18 스크린샷 "뭉개짐"의 직접 원인.
- **`AuthGate.jsx` 로그아웃 버튼** = `fixed right-4 bottom-[calc(5rem+…)] z-40`. **로그인된 모든 화면**에 상시 떠서 콘텐츠(스케줄 그리드·할일 카드·태도태그 칩)를 가리고 다닌다. 로그아웃은 드문 동작인데 전 화면을 따라다님.
- **조치 방향:** 로그아웃 플로팅 제거 → **설정 탭(계정)** 으로 이관. FirstOTTab 메모바는 하단탭바 위로(`bottom-[calc(하단바높이+safe)]`) 올리거나 고정 해제 후 카드 하단 인플로우로.

### 뿌리② — 레드 브랜드가 6색으로 분열 **[M · 광범위]**
`v2-스펙-레드톤통일-초록제거.md`로 맞춰둔 통일이 위젯·브리핑에서 다시 깨졌다. 지금 크롬에서 동시에 경쟁하는 색:
- **amber** = OT 서브탭 · "재등록 타이밍" 배지(PtWorkoutTab·PtReRegTab) · SecondOTTab/RegBrief 헤더 · Observation!canEdit 안내
- **sky** = PT 서브탭 · ObservationTab "수정" 배지 · Second/Reg 브리핑 헤더 · MemberSchedule "개인운동" 배지
- **emerald** = RegisterDue/RegisterReapproach 카드 · MonthlyReport 급여 헤드라인(같은 지표를 MyStats는 레드로!)
- **rose** = ChurnRisk 섹션 · UnclosedClosing · Refund danger
- **fuchsia** = 관리자 버튼(헤더)
- **violet/teal/zinc** = ScheduleBoard 트레이너 팔레트
→ 특히 **오늘 대시보드**는 위젯이 amber→emerald→emerald→rose→zinc→red **무지개 카드 스택**(TodoTab). **조치:** 카드 크롬은 중립(line/card)으로, 색은 아이콘/카운트 배지에만 국소로. 의미 없는 sky/amber/emerald 헤더는 레드 또는 sub로 강등. 같은 지표(급여)는 한 색으로.

### 뿌리③ — 봐야 할 정보가 제일 작은 글씨 **[M · 광범위]**
`text-[9px]`/`[10px]`/`[11px]`에 **핵심 콘텐츠**가 들어가 있다:
- 스케줄 예약 칩(회원/트레이너 이름) `text-[10px]`·서브 `text-[9px]`(ScheduleBoard)
- 종목별 세트 데이터 `text-[10px]`(PtWorkoutTab 타임라인) · 인바디 수치 라인 `text-[11px]`(PtInbodyTab)
- 클로징 **멘트/근거**(so_what·if_weak) `text-[9~12px]`(SecondOTTab·RegBrief) — "바로 외워 말할 대사"가 제일 흐림
- 폼 라벨 `text-[11px]` · 도움말 `text-[10px]`(모든 폼)
→ **조치:** 의미 정보는 최소 `text-xs`(12px)로. 장식/보조만 11px 이하 유지.

### 뿌리④ — 긴 한글 라벨 2열 그리드 → 라벨 줄바꿈 → 입력칸 세로 어긋남 **[H]**
"회원정보 뭉개짐"의 폼 버전. `grid grid-cols-2`에 라벨이 한 셀은 2줄("휴대폰 번호 (회원앱 로그인용)"), 옆 셀은 1줄("나이")이면 두 입력칸 시작 높이가 어긋나 계단처럼 밀린다.
- **`MemberForm`(신규 등록, 17필드)** · **`MemberEditForm`(동일 17필드)** — 가장 심함
- **`ContractAmountFields`**(PT확정 모달, max-w-sm 안에서 4필드 2열)
- **`PtPricingSettings`**(가격 4필드 · 체크박스+입력 높이 불일치)
- **`TrainerProfileSettings`** — MBTI 1개짜리를 `grid-cols-2`로 감싸 오른쪽 절반이 텅 빔
→ **조치:** 이 폼들 **단일 컬럼**(`grid-cols-1`)로, 또는 라벨에 `min-h`로 높이 예약. 자유서술 placeholder는 축약.

### 뿌리⑤ — `flex-wrap` + `ml-auto` 버튼이 좁은 폭에서 홀로 다음 줄로 튐 **[M]**
텍스트+배지들이 한 줄을 채우면 오른쪽 CTA가 빈 공간 아래로 떨어져 "붕 뜬" 배치가 됨. 반복처:
- 재등록 버튼(PtWorkoutTab 잔여카드) · 재생성 버튼(SecondOTTab meta 행) · 링크복사/PT종료(MemberAppLink) · PT등록확정 CTA(PtConfirmBanner) · 할일 추가행(TodoManual) · 스케줄 헤더 시간 select(ScheduleBoard)
→ **조치:** 좁은 화면에서 CTA를 **자기 전용 행**(`flex-col`/전폭)으로 내리거나 배지 클러스터를 함께 wrap.

### 뿌리⑥ — 상단 밴드 적층 + 정보 순서 역전 **[M]**
- **1차 OT 준비:** 회원 identity 카드(`text-3xl/4xl` 이름)가 AI 블록 **아래**. 정작 "누구 수업인지"가 맨 밑 + 고정 메모바에 눌림.
- **1차 피드백/OT 화면 상단:** PtConfirmBanner(빨강) → 정보수정 버튼 → Eyebrow → 이름+수정배지 → 섹션 = 실제 입력 전 **4겹**.
- **헤더 높이 점프:** 서브탭 줄이 오늘·회원목록·내실적엔 없고 워크플로우/설정엔 있어 탭 이동마다 헤더가 커졌다 작아짐.
→ **조치:** identity를 상단으로(또는 헤더에 회원 컨텍스트 고정), 정보수정/수정배지 중복 진입점 정리, PtConfirmBanner를 콤팩트하게.

### 뿌리⑦ — 고정폭/overflow로 뷰포트 밖 삐져나감 **[M]**
- **ScheduleBoard** 주간 `min-w-[720px]` → 390pt 폰에서 가로스크롤 표. 게다가 6~24시 18행 빈 그리드가 화면을 지배(예약 0일 때 800px 공백).
- **CenterMachineSettings** 장비 pill의 이름 span에 `min-w-0/truncate` 없어 긴 이름("테크노짐 레그프레스 · 덤벨 2.5~40kg")이 pill을 뷰포트 밖으로.
- **PtWorkoutTab** 종목명 `truncate text-[11px]` → "인클라인 덤벨프레스" 잘림(폰은 hover title 못 봄).
→ **조치:** min-width 제거/축소, `min-w-0 truncate` 또는 `line-clamp-2`, 빈 시간대 접기.

### 뿌리⑧ — 아이콘 체계 불일치(이모지 ↔ lucide) **[L]**
2차/재등록 브리핑이 이모지 헤더(↩️💪🎯💬🔥📈🔑🛡)와 lucide를 한 화면에서 혼용 → `lucide-react only` 규칙 위반 + 톤 들쭉날쭉. → lucide로 통일(Undo2·Dumbbell·Target·MessageSquareQuote·Flame 등).

## 3. 권장 수정 배치 (스펙화 순서 · 전부 표시/additive)
