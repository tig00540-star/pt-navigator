  - **저장(V1):** `setX(true)` → `try{본문}` → `catch{그 사이트 기존 에러표기}` → `finally{setX(false)}`. try는 **데모 가드 뒤·첫 await 앞**. 기존 `setX(false)` 그대로(이중 무해). 성공콜백(onSaved/onDone)은 **try 밖 + catch에 return**(실패 오인 방지).
  - **조회(L1):** effect IIFE에서 `finally{if(!cancelled)setLoading(false)}`(언마운트 안전). 로딩 플래그 없으면 catch에서 setErr/setDbNote만.
  - **V2:** boolean 반환·자체 상태 없는 함수(confirmPtActive류)는 finally 없이 catch에서 `return false`.
  - 빈 catch엔 주석 1줄(`no-empty` 회피).
