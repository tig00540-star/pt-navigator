## 0. 대원칙 (프리미티브·토큰 작업 시 항상)

1. **이름은 앱 것 유지, 값만 DS로 교체.** 토큰(`--color-*`·`--radius-*`·`--shadow-*`) 이름을 안 바꾸고 값만 DS 규격으로 덮었다 → 기존 유틸 **1,962곳이 한 줄도 안 바뀌고 자동 반영**. DS 이름(`--red`·`rounded-card` 등)을 새로 만들면 같은 뜻에 이름이 둘이 되어 드리프트한다(`bg-red`는 Tailwind 기본 `bg-red-500`과도 혼동).
2. **purge-safe 정적 리터럴만.** 모든 색·형태 클래스는 완성된 문자열 리터럴. **동적 조립 금지**(Tailwind purge가 못 지킴). 변형은 `variant×accent` → 완성 클래스 **정적 맵**으로.
3. **인라인 카드/폼/모달을 새로 손수 만들지 않는다.** 공용 프리미티브(§3)를 쓴다. 같은 문자열이 82곳(카드)·21파일(입력)로 흩어져 "한 곳 고쳐도 안 따라오던" 드리프트를 없앤 게 이번 개편의 핵심.
4. **역할색은 '뜻'을 코드에 박는다.** `text-amber-700`을 화면에서 직접 고르면 "이 amber가 OT라서인지 그냥 노랑인지"가 코드에 안 남는다. 역할 토큰(`--color-ot/pt/admin/danger`)과 역할 이름(tone)으로 수렴. ⚠️ **역할색 채움값은 글자 단독 사용 금지**(밝은 배경 대비 미달 · amber #f59e0b 흰 배경 2.15) → 반드시 `-soft` 배경 + `-text` 글자 세트.
5. **한글 라벨 자간 예외.** 라틴 라벨 규격(`+.16~.2em uppercase`)은 한글에서 무효(uppercase)이고 자소가 벌어져("내 프 로 필") 안 읽힌다. 한글 섞이면 `tracking-label-ko`(0.02em)·대문자 변환 없음. `Eyebrow`가 렌더 시 정규식 1회로 자동 판별.

## 2. 폰트 — `app/layout.js`

**Pretendard Variable** 한 벌(`localFont`, `src: public/fonts/PretendardVariable.woff2`, `weight "45 920"`, `--font-pretendard`, `display:swap`, 가변 woff2 2.0MB). **Geist 제거 이유:** Geist엔 한글 글리프가 없어 스택이 `Geist, system-ui`였을 때 라틴·숫자만 Geist, **한글은 전부 기기 기본폰트로 폴백** → 한국어 앱인데 한글 조판이 기기마다 달랐다. Pretendard가 한글·라틴을 한 벌로 덮어 통제권 회복. `body` `font-weight:450`·`letter-spacing:-0.011em`. `font-mono`(Geist Mono)만 숫자·코드용 유지(47곳). ⚠️ DS 본문 행간 1.7은 **전역 적용 안 함**(Tailwind text-* 유틸이 이미 크기별 행간 보유 → 유틸 없는 요소만 1.7이면 세로 크기 예측 불가). 행간은 컴포넌트 단위로.

## 3. 프리미티브 카탈로그 (`components/ui/`)

각 항목: 파일 · 목적 · API · 규율.

### 셸/레이아웃
- **`Card` / `CardRow`** (`Card.jsx`) — 기본 흰 카드 셸(`rounded-2xl`·`shadow-sm`·`border-line`). props: `as`·`padding`(md `p-5`/sm `p-4`/lg `p-6`/none `p-0`)·`interactive`·`selected`(→`border-primary bg-primary-soft`)·`elevated`(기본 true). `padding="none"`은 분할 리스트(overflow-hidden 자동). `CardRow`는 분할 리스트 한 행(`first`면 상단 구분선 없음). ⚠️ **위계는 그림자가 아니라 배경 명도**(page<card)로 만든다 — 카드에 `shadow-lg` 얹지 않는다.
- **`ToneCard`** (`ToneCard.jsx`) — 파생 위젯 표준 컨테이너(tone별 테두리+틴트). `mb-4 rounded-2xl border p-4` + `widgetTone(tone).card`. shadow 없음(틴트 카드 결). ⚠️ **구 `Card`(톤 카드)가 이걸로 개명됨** — "카드"라는 말이 기대되는 흰 셸(82곳)이 다수라 이름을 가져갔고, 톤 위젯(11곳)이 `ToneCard`로.

### 위젯 톤 — `tone.js` (`WIDGET_TONE` / `widgetTone`)
**색 이름 → 역할 이름 체계**로 전환. `ROLE_TONE` 키: `reapproach`(재접근·OT흐름·amber) · `renewal`(재등록·PT흐름·sky) · `brand`(이 화면 최우선 하나·red) · `unclosed`(미확정 클로징·유일한 red 우선순위 신호) · `neutral`(중립·zinc) · `danger`(환불·손실·rose · 할일 위젯엔 안 씀). 각 tone은 `{card, icon, title, countBadge, rowBorder, rowHover, chevron, chevronHover}` 세트. **색 이름 별칭 유지(하위호환):** `amber→reapproach` · `zinc→neutral` · `rose→danger` · `emerald→renewal`(⚠️ emerald는 레거시라 정의가 red였음 → 재등록 위젯 2곳의 원래 의도대로 renewal로 매핑). **새 코드는 역할 이름을 쓸 것.** 소비처: `ToneCard`·`SectionHeader`·`ListRow`. ⚠️ 배정 교정: 과거 미확정 클로징=rose·재등록=red였는데 DS 규칙은 반대(unclosed만 red, renewal은 sky).

### 폼 — `Field.jsx` (+ `NumberInput.jsx`)
- **4종:** `Input` · `Textarea` · `Select` · `Checkbox`. 공통 `Shell`(라벨 `text-[12.5px] text-sub` + hint/error). props: `label`·`hint`·`error`·`id`·`size`(md `px-3.5 py-[11px] text-[15px]` / sm)·`accent`(trainer/owner=admin fuchsia)·그 외 네이티브 pass-through. `Input`은 `suffix`·`align="right"`·`mono`, `Select`는 `appearance-none`+직접 그린 chevron, `Checkbox`는 `accent-primary`.
- ⚠️ **테두리는 `border`(ring 아님)** — DS 명세는 "focus 1px→2px 밀림 방지로 ring"이었으나 **현장 제보로 되돌림**: ①흰 카드 위 `bg-card`면 배경만으론 필드 경계 안 보임 ②`ring`=box-shadow인데 **iOS Safari가 네이티브 `<select>`에 box-shadow 미표시** → 필드가 통째로 사라졌다(2차 클로징 드롭다운이 글자만 떠 있던 현상). 그래서 배경 `bg-elevate`(눌린 면) + `border`(모든 브라우저 확실). focus는 색만 변경(두께 유지·밀림 없음), ring은 보너스로만.
- **과도기 공유 클래스:** 아직 컴포넌트로 안 옮긴 raw 필드용 `inputCls`(md·trainer) / `inputClsOwner`(admin fuchsia) / `inputClsSm`(조밀 표 입력 · SetsEditor·PayrollConfirm). 21개 파일의 복사본을 여기로 수렴(라벨 마크업이 8종이라 JSX 재구조화는 위험 대비 이득 낮음 → '드리프트'부터 제거).
- **`NumberInput`** — 천단위 콤마 정수 입력. ★ `value`는 **콤마 없는 숫자 문자열**(부모 state 그대로 · 화면에만 콤마), `onValueChange=(raw)=>void`. 저장·검증은 부모가 기존과 동일. 캐럿 복원(콤마 삽입 시 커서 튐 방지). 셸은 `Field`의 `Input`에 위임(17곳 일괄 규격 · React 19라 `ref`가 일반 prop). 정수 전용(소수·%엔 금지).

### 피드백/오버레이
- **`Modal`** (`Modal.jsx`) — **포털(`createPortal`→body)** 다이얼로그. props: `variant`(center/sheet) · `size`(sm/md/lg · 생략 시 center=md·sheet=lg) · `title`/`subtitle`/`footer` · `dismissable` · `blocking`(필수공지 · 닫기·배경클릭·ESC 전부 무시) · `onClose`. 기능: 포커스 트랩·ESC 닫기·배경 스크롤 잠금·트리거 포커스 복원·`--shadow-pop`. `title` 없이 쓰면 **하위호환**(children이 곧 카드 내용·`p-5`), `title` 주면 DS 구조(헤더/본문/푸터). ⚠️ **포털 이전은 과거 버그의 구조적 해결** — `fixed`가 조상 transform 기준으로 앵커돼 화면 밖으로 밀리던 문제를, body 직속으로 붙여 조상을 없애 재발 불가로 만듦.
- **`AIBriefBlock`** (`AIBriefBlock.jsx`) — AI 사전무장/브리핑 단일 출처(1차 `FirstOTAssist`·2차 `SecondOTTab`·재등록 `PtReRegTab`). props: `status`(idle/loading/ready/stale/demo)·`title`·`waitingHint`·`idleDescription`·`meta`·`notice`·`onGenerate`/`onRegenerate`·`generateLabel`·children(본문). ⚠️ **`loading`은 반드시 `waitingHint`와 함께** — "AI 45초~1분인데 빈 화면"이 PRD 리스크 1순위이자 이 컴포넌트의 **존재 이유**. loading에서 진행바(`ot-indeterminate`)+힌트를 구조적으로 강제. `stale`=amber "기록 변경됨·재생성 권장" 뱃지, `demo`="데모" 뱃지. 좌측 3px 레드 강조선은 상태 무관 항상 유지.

### 컨트롤/라벨
- **`Button`** (`Button.jsx`) — 전역 버튼 단일 출처. props: `variant`(primary=레드 그라데 채움 / solid=soft+ring / ghost=아웃라인 / danger=rose 아웃라인)·`accent`(trainer/owner=admin fuchsia)·`size`(sm/md · **lg는 규격엔 있으나 미도입**·앱 쓰임 0)·`subtle`(danger 저강도)·`fullWidth`·`as`(a=링크). 아이콘은 children. ⚠️ md 모서리 `rounded-xl`(12px)→`rounded-lg`(10px) 65곳(브랜드가이드 §06 "10px 아니면 14~16px, 사이 금지"). 초록 없음(primary/danger를 fill/outline로 구분).

- **`FilterChip`** (`FilterChip.jsx`) — 필터·선택 토글 칩. `selected`(→`bg-primary-soft ring-primary`, `aria-pressed`). ⚠️ **이름이 `FilterChip`인 이유:** 앱에 이미 다른 뜻의 `Chip`(정적 메타 태그·MBTI/통증·상호작용 없음)이 10곳 있어 덮으면 뜻이 조용히 바뀐다(앱 Chip→MetaTag 개명은 별개 작업으로 보류). **선택색은 브랜드 레드**(OT/PT 필터여도) — 역할색은 뱃지가, 선택상태는 레드가. 테두리 `ring`(border는 상태별 두께 변화로 1px 밀림).
- **`StatTile`** (`StatTile.jsx`) — 라벨+큰 숫자 타일(내 실적·리포트 6곳). `icon`·`label`·`value`·children(보조 라인). 셸은 `Card`에 위임.

## 4. 재발방지 함정 (핵심 규율 요약)
- **iOS `<select>`엔 border**(ring=box-shadow 미표시로 필드 사라짐). 배경은 `bg-elevate`.
- **모달은 `Modal`(포털)** — 손수 `fixed` 모달 금지(조상 transform에 갇힘). `.tab-anim` fill-mode=backwards도 같은 문제의 다른 층 방어.
- **`Wordmark`는 `whitespace-nowrap`** — 글자 갈라짐 방지.
- **`AIBriefBlock` loading엔 `waitingHint` 필수** — 빈 화면 금지.
- **역할색 글자는 `-text`만**(채움값은 밝은 배경 대비 미달).
- **모든 색·형태 클래스는 정적 리터럴**(동적 조립 금지).
