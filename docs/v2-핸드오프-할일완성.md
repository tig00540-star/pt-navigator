## 0. 이번 세션 성과 — 기능2 할일 (전부 배포됨 · origin/main)

**한 줄:** 흩어져 있던 "해야 할 것" 신호(재접근·재등록·미마감)를 **새 상시 탭 "할일"** 하나로 통합 + 수동 메모. 회원 탭의 위젯 3개를 흡수(중복 제거). 트레이너 본인 담당 스코프.

### 설계 (스펙: `docs/v2-할일-스펙-통합액션.md`)
- **자동 5소스 + 수동 메모, 한 화면.** 자동=기존 데이터 파생(테이블 없음), 수동=신설 `trainer_todo`.
- **흡수(재구현 아님):** 기존 `ReapproachToday`·`RegisterDueToday`·`RegisterReapproachToday`를 **할일 탭으로 옮겨 렌더**하고 회원 탭에선 제거. 컴포넌트 파일 자체는 유지(재사용).
- **스코프 = 트레이너 개인.** 원장은 계정 전체가 RLS로 넘어오므로 할일 탭에서 `uid`로 본인 담당만 필터(개인 뷰). 센터 전체 감시는 /admin 몫.

### 자동 5소스 (할일 탭)
1. **오늘 재접근(OT)** — `ReapproachToday`(그대로). ot_log hold + closing_reapproach_at 도래. → 1차 OT.
2. **재등록 타이밍** — `RegisterDueToday`(그대로). PT 활성계약 잔여 유료 <10. → 재등록.
3. **재등록 재접근** — `RegisterReapproachToday`(그대로). session_log 최신계약 reg hold + reg_reapproach_at 도래. → 재등록.
4. **클로징 미마감**(신규 `UnclosedClosingToday`) — OT 회원 중 ot_log round=2 있는데 closing_result **빈값**(hold/success/fail 제외 → 재접근과 안 겹침). → 2차 OT.
5. **미처리 예약**(신규 `PastDueAppointments`) — appointment status='booked' + start_at 과거 + `trainer_id=내 uid`. → 스케줄.

### 수동 메모 (`TodoManual` · trainer_todo)
- 추가·체크(done 토글)·삭제(인라인 2단 확인·브라우저 다이얼로그 X). due_date 선택. 정렬=미완료 먼저·최신순.
- 전부 `.select()` 하드닝(정책 없으면 조용한 실패 → "실패" 표기). 데모(키 없음)=로컬 상태만.

### 마이그레이션 (docs/migrations, 이번 세션)
- **`2026-07-12-trainer-todo.sql`** — trainer_todo(id·account_id DEFAULT auth_account_id()·**trainer_id DEFAULT auth.uid()**·body·done·due_date·member_id(→user_table on delete set null)·created_at·done_at). **프로젝트 최초 trainer_id 스코프 RLS**(`for all` · 본인 것만 · UPDATE/DELETE 포함 = 토글·삭제 보장). 인덱스 2(owner·open 부분). - **자동 정확성:** 5개 섹션이 "지금 진짜 할 일"만 뜨나 / 원장 뷰에서 본인 담당만 / 각 항목 탭→알맞은 회원·탭.
- **미확정 클로징(자동4) watch:** 실데이터에서 데이터 있는데 안 뜨면 `ot_log.closing_result` 실제 빈값 표현 확인(null vs "" — 순수함수는 둘 다 처리하나 저장경로 재확인). 반대로 과하게 뜨면 round 게이트(현재 round=2만) 조정.
- **미처리 예약(자동5):** 완료/취소하면 사라지나 / 미래 예약 안 뜨나 / 아주 오래된 booked 누적 시 상한 필요할지(현재 limit 없음).
- **수동:** 추가·토글·삭제 왕복 + 새로고침 유지 / 마감일 표기 / 데모 크래시 없음.
