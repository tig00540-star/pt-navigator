# v2 스펙 — 콤마 입력(천단위) · 재사용 숫자입력 컴포넌트 (#3)

> 백로그 #3. 가격·세션·급여·환불 등 숫자 입력칸에 **천단위 콤마**를 실시간 표시.
> 역할: 이 문서=스펙(웹Claude) → 트레이너 → 클로드코드(diff). 커밋은 트레이너 직접(파일지정 add · lint 통과 후 · `git show --stat HEAD`).

---

## 0. 핵심 원리 (회귀 위험 낮은 이유 — 먼저 읽기)

- 지금 모든 돈/수량 입력은 `type="number"` + **문자열 state**(`e.target.value`) + 저장 시 `Number(x)` + 표시는 `won()`(콤마).
- **`type="number"`는 콤마를 못 보여준다**(브라우저가 `1,200,000` 같은 포맷 값을 거부·무시). 그래서 재사용 컴포넌트는 **`type="text"` + `inputMode="numeric"`**(모바일 숫자 키패드 유지)로 간다.
- ★ **저장값(부모 state)은 "콤마 없는 숫자 문자열" 그대로 유지 — 화면에만 콤마를 입힌다.**
  → 부모의 `Number(x)`, `x === ""` 검증, `String(value)` 프리필, 자동총액(`sessions*price`) 계산이 **전부 그대로** 작동. 다운스트림 0 변경 = 회귀면이 작다.
- 즉 이건 "표시 레이어만 교체". 값 흐름·DB·검증은 안 건드린다.

## 2. 신규 컴포넌트 — `components/ui/NumberInput.jsx`

```jsx
"use client";
import { useRef } from "react";

// 천단위 콤마 정수 입력. ★ value = '콤마 없는 숫자 문자열'(부모 state 그대로) — 화면에만 콤마.
// 저장·검증(Number()/=== "")은 부모가 기존과 동일. 정수 전용(소수·퍼센트엔 쓰지 말 것 — §5).
// 부모 계약: value=문자열, onValueChange=(raw문자열)=>void. 나머지(className·placeholder·disabled)는 그대로 전달.
const group = (digits) => digits.replace(/\B(?=(\d{3})+(?!\d))/g, ",");
const toRaw = (s) => String(s ?? "").replace(/[^\d]/g, "").replace(/^0+(?=\d)/, ""); // 숫자만 + 선행 0 제거(단, 단독 "0"은 보존)

export default function NumberInput({ value, onValueChange, ...rest }) {
  const ref = useRef(null);
  const display = group(toRaw(value));

  const handle = (e) => {
    const el = e.target;
    const caret = el.selectionStart ?? el.value.length;
    const digitsBefore = el.value.slice(0, caret).replace(/[^\d]/g, "").length; // 캐럿 앞 '숫자' 개수
    onValueChange(toRaw(el.value));
    // 리렌더 후 캐럿 복원 — 콤마 삽입으로 커서가 튀는 것 방지
    requestAnimationFrame(() => {
      const node = ref.current;
      if (!node) return;
      let pos = 0, seen = 0;
      while (pos < node.value.length && seen < digitsBefore) {
        if (/\d/.test(node.value[pos])) seen++;
        pos++;
      }
      node.setSelectionRange(pos, pos);
    });
  };

  return (
    <input
      {...rest}
      ref={ref}
      type="text"
      inputMode="numeric"
      value={display}
      onChange={handle}
    />
  );
}
```

- `{...rest}`를 **먼저** 펼치고 뒤에 `type/value/onChange`를 둬서, 호출부의 `className`·`placeholder`·`disabled`는 통과하되 제어 속성은 컴포넌트가 소유.
- 간단판(캐럿 미보존)을 쓸 경우: `handle`을 `onValueChange(toRaw(e.target.value))` 한 줄로, `ref`/`requestAnimationFrame` 삭제.

## 4. 호출부 치환 규칙 (onChange 모양 3종)

```jsx
// (1) 단순 setter  — RefundMember·PayrollConfirm·TrainerGoalSetter·PtPricingSettings
<input type="number" inputMode="numeric" value={amount} onChange={(e) => setAmount(e.target.value)} ... />
→ <NumberInput value={amount} onValueChange={setAmount} ... />

// (2) 디스패처(key,value) — ContractAmountFields
<input type="number" value={price} onChange={(e) => onChange("price", e.target.value)} ... />
→ <NumberInput value={price} onValueChange={(v) => onChange("price", v)} ... />

// (3) 패치 업데이트 — AdminPayrollSettings 밴드
<input type="number" inputMode="numeric" value={b.min} onChange={(e) => updateBand(i, { min: e.target.value })} ... />
→ <NumberInput value={b.min} onValueChange={(v) => updateBand(i, { min: v })} ... />

// (4) 커링 setter(set(k)=(e)=>...) — MemberForm 이월 2칸 (app/page.jsx L138 `const set`)
<input type="number" value={form.carryPrice} onChange={set("carryPrice")} ... />
→ <NumberInput value={form.carryPrice} onValueChange={(v) => setForm((f) => ({ ...f, carryPrice: v }))} ... />
//   (carrySessions도 동일. set()은 안 쓰고 setForm 인라인으로 raw 값 반영.)
```

- `className`·`placeholder`·`disabled`는 **그대로 유지**(컴포넌트가 통과시킴). `type="number"`·`inputMode`·`onChange`만 제거.
- import 추가: `import NumberInput from "@/components/ui/NumberInput";`

## 5. 제외 대상 (그대로 `type="number"` 유지) — 중요

콤마·정수화가 값을 깨는 필드는 **건드리지 말 것**:

- **퍼센트/소수 가능:** `AdminPayrollSettings` 지급값(payout_value, %일 수 있음)·인센값(incentive_value, %). `toRaw`가 소수점을 지워 `12.5%` → `125`로 망가짐.
- **인바디(소수 kg·%):** `components/views/PtInbodyTab.jsx` 전체(체중·골격근·체지방 등).
- **나이·시간·순서:** `MemberForm`(app/page.jsx) **나이**(이월 세션수·회당단가는 §3에서 포함), 스케줄 시간, sort 인덱스 — 콤마 무의미.

> 원칙: **정수 원화·정수 수량만** `NumberInput`. 소수/퍼센트/식별용 숫자는 제외.

## 7. 검증 (폰 = Vercel 배포본 하드리프레시)

1. `npm.cmd run lint` 통과(신규 파일 포함).
2. **각 전환 사이트 타이핑 테스트:** 회당단가 `60000` 입력 → `60,000` 실시간 표시. 백스페이스·중간 삽입 시 커서 안 튐(보존판).
3. **저장 왕복 정확성:** 콤마 보이는 상태로 저장 → DB엔 `60000`(콤마 없는 정수). 재진입 프리필도 정확.
4. **자동총액:** ContractAmountFields에서 세션수·단가 입력 → 총액 자동값이 콤마로 정확히 뜨는지(계산 로직 무변 확인).
5. **데모 모드**(키 없이)도 렌더·입력 정상.
6. 제외 필드(지급%·인바디kg) 그대로 소수 입력 가능한지 회귀 확인.

## 8. 리스크 / 회귀면

- `type="number"` 제거 영향: 네이티브 스피너·소수검증 상실 → 이 앱은 스피너 안 쓰고 검증은 수동 `Number()`라 **영향 없음**. `inputMode="numeric"`로 모바일 숫자 키패드 유지.
- 저장값 스키마 무변(여전히 콤마 없는 정수 문자열) → DB·급여계산·매출집계 안전.
- 유일 주의: **§5 제외 대상에 잘못 적용하면 소수/％가 깨진다** — grep 대조로 방지.
