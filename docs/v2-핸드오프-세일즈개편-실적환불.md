REQUIRED_FIRST·FIELD_TERMS 갱신.
- **B-2(클라)** `FirstOTAssist.jsx` — 마운트 시 본인 active 패키지 로드→요청 주입. **pick_ref로 내 목록에서 실가격 조회 렌더**(AI는 번호만=환각 방지). 핵심3줄(💳추천/🎯승부처/📍클로징진입)+펼치기(추천상세·클로징4단계·거절4종·arc·치트키·관찰).

### S. 세일즈 전면 실전화 (핵심 철학 전환)
> 트레이너 요청: 운동은 스파링(성장)이 맞지만 **세일즈는 별개 스킬 → 결과가 나오게 실전 도구여야**. 관리자도 매출이 나와야 앱 쓸 이유.
- **S-1** `PREAMBLE` **도메인 분리** — 운동(방법·방향·자각·큐잉)=스파링 파트너(대본 금지·방향까지만) / 세일즈(추천·클로징·거절·비유)=**실전 클로저**(바로 말할 멘트·확신). 윤리 재프레이밍: "세일즈 강도=근거·책임의 세기". **조작 바닥선 유지**(허위 긴급성·FOMO·죄책감·소진강요·의료단정 금지 — 효과에도 역효과).
- **S-2** 1차 세일즈 실전 멘트화(firstPrompt closing enter/paint/land=실제 대사, objections sales_move·example 실전화) + FirstOTAssist 거절 예시 '흐림→"멘트" 박스'.
- **S-3** 2차(secondPrompt closing 발화규칙 + **objections에 sales_move 필드 신설** + SecondOTTab 렌더: 클로징 대사 따옴표·거절 sales_move·example 격상). = **2차 개편 본체**.
- **S-4** 재등록(reregisterPrompt + RegBriefView 동일 패턴). = **재등록 개편 본체**.
- 운동 파트(movement_cues·stimulus_response·arc example)는 흐림/발판 유지(도메인 분리).

## 1. 남은 큰 조각 = D (강점 활용 · 임계 5건 게이트)

- **뭐:** 1·2차 브리핑에 "이 트레이너의 과거 통한/막힌 클로징 케이스"를 근거로 피드백(D-3). 실패 복기(다른 벡터로)·결정권자 키트·반복 패턴=블라인드스팟·성공 접근 리딩.
- **게이트:** 성공 클로징 **5건 미만=꺼짐, 5+=자동 켜짐**(5~9 잠정경향, 10+ 확신). 지금 코딩해두고 5건 되면 자동 발동.
- **재료(이미 축적 중):** `ot_log` round1·2의 closing_result/approach/reason/closing_detail(3박자)/**closing_profile**(비슷한 회원 매칭용). C가 이걸 캡처 중.
- **선행:** 파일럿 며칠 굴려 케이스 쌓기(MASTERPLAN 제1법칙: 케이스 0이면 AI 헛돈다). 상세 설계 = `docs/v2-closing-cases-D3-design-note.md`.
- **스코프:** 본인 트레이너 케이스만(trainer_id=자기교정). 톤=정답 하달 아니라 거울(스파링) — 단 §4-B 리딩 톤·§5 강도 가드 준수.

## 2. 파일럿 관찰 포인트 (D 설계 전 실사용 피드백)

- **세일즈 톤:** 실전 멘트가 현장에서 자연스럽게 나가는지 / 압박·조작으로 안 새는지(선명 ≠ 압박) / 트레이너가 "그대로 읽는" 게 아니라 자기 톤으로 소화하는지.
- **추천 프로그램:** pick이 회원에 실제로 맞는지 / 대안이 유의미한지 / 가격 환각 0인지.
- **goal 렌즈:** 외형·체중 회원이 pain으로 안 새고 목적 중심으로 도는지 / 개인화 비유가 회원 직업에서 나오는지.
- **클로징 캡처 실사용:** 트레이너가 ㉠에서 사유·3박자를 실제로 입력하는지(D 재료 질). 안 하면 입력 마찰 줄일 방법.
- **환불·스코프:** 환불→매출 차감·회원 숨김 정상 / 원장 "내 회원/전체" 토글 체감.
