## 0. 배경 · 결정

로고 = **레드 크림슨 글로시 라운드 스퀘어 + 흰 "TO" 모노그램**("오직 트레이너"). 현재 앱 트레이너 accent는 **초록(emerald/lime)** 이라 로고와 정면 충돌.

**결정(트레이너 승인): 레드 브랜드 전환.**
- 트레이너 브랜드 accent: 초록 → **레드**.
- **성공/돈(급여·매출·목표달성) = 초록 유지**(돈=초록은 만국 공통 · 레드 CTA와도 잘 붙음).
- **danger(삭제·환불) = 레드 브랜드와 안 겹치게 재정리**(아래 §3).
- OT=amber · PT=sky · 원장=fuchsia는 **이번 스코프 밖(유지)**.

### 전환이 일어나는 두 지점 (맞물림)
1. **토큰 플립(§1)** — `--color-primary/*`를 레드로. 앱 전역 `primary` 토큰 사용처 **약 250곳**(선택·포커스·활성탭 바·소프트 배지·`text-primary-strong`·`border-primary`·`focus:border-primary`·`selection:bg-primary/20`)이 **한 커밋에 레드로** 수렴. 헤더 "오직 트레이너" eyebrow도 자동 로고색 일치.
2. **Button 프리미티브(§3)** — CTA 버튼 대부분은 **리터럴 emerald 그라데이션**(토큰 아님)이라 토큰 플립만으론 안 바뀜. Button의 `primary` 변주를 **레드**로 만들고, 기존 배치 1~4에서 CTA를 `<Button>`으로 이관하며 레드로 수렴.

> 전환 중(배치 진행 사이)에는 "포커스·선택은 레드인데 아직 이관 안 된 CTA는 초록"인 **일시 혼재**가 정상. 배치가 끝나면 균일. 급하면 배치 1을 토큰 직후 바로.

## 1. 색 토큰 재편 — `app/globals.css`

### 현재
```css
  --color-primary: #10b981;       /* 초록 포인트(채움·점·버튼) */
  --color-primary-strong: #059669;/* 초록 텍스트(흰 배경 가독) */
  --color-primary-soft: #ecfdf5;  /* 초록 옅은 배경(절제 사용) */
```

### 개정 (primary→레드, success(초록) 신설)
```css
  /* 브랜드 = 레드(로고). primary 토큰 = 채움·점·선택·포커스·활성·브랜드 텍스트. */
  --color-primary: #dc2626;        /* 레드-600 · 채움·점·선택·포커스 */
  --color-primary-strong: #b91c1c; /* 레드-700 · 텍스트(흰 배경 AA 6.5:1) */
  --color-primary-soft: #fef2f2;   /* 레드-50 · 옅은 배경(선택·소프트 배지) */
  /* 성공/돈 — 초록 유지(구 primary 값). 급여·매출·목표달성·긍정 상태 전용. */
  --color-success: #10b981;
  --color-success-strong: #059669;
  --color-success-soft: #ecfdf5;
```

- **대비 검증:** `primary-strong #b91c1c` 흰 배경 6.5:1(AA 통과). `success-strong #059669`는 기존값 그대로.
- 이 세 줄만 바꾸고 세 줄 추가. 나머지 토큰(bg/card/elevate/line/ink/sub/muted) 불변.
- 새 토큰 `success`/`success-strong`/`success-soft`는 Tailwind v4 `@theme`에 등록되므로 `bg-success-soft`·`text-success-strong`·`border-success`·`border-success/30` 등 클래스가 즉시 생성됨(purge-safe).

## 2. 돈/성공 = `success` 재지정 (초록 유지)

토큰 플립으로 `primary`를 쓰던 **모든** 자리가 레드가 된다. 이 중 **의미가 돈/성공/급여/매출/목표달성/긍정상태**인 곳은 초록이어야 하므로 `success`로 되돌린다.

### 재지정 규칙 (클로드코드 · 판단 + 검토 게이트)
`primary`/`primary-strong`/`primary-soft` 사용처를 훑어, **그 자리의 의미가 아래면 `success`로 교체**:
- 급여(예상/확정) 카드·숫자, 매출 금액·배지, 목표 달성 게이지·%, "성공" 통계, 긍정 상태 표식.

**그대로 primary(레드) 유지**할 것(브랜드·상호작용 의미):
- 선택됨(selected) 카드/칩, 활성 탭·바, 포커스 링(`focus:border-primary`), 검색창 포커스, 헤더 eyebrow, 일반 CTA(버튼은 §3에서 별도), 토글 선택 상태, 아이콘 포인트.

> 애매하면 **레드 유지 + git diff에 남겨 보고**(웹Claude 검토). "이게 돈/성공인가?"만 기준.

### 알려진 앵커 (반드시 success로)
- **`components/views/MyStats.jsx`** — 급여 헤드라인 카드: `border-primary/30 bg-primary-soft` → `border-success/30 bg-success-soft`, 급여 숫자 `text-primary-strong` → `text-success-strong`. (같은 파일 목표달성 게이지는 이미 **리터럴 emerald**라 자동 초록 유지 — 안 건드림.)
- **급여/매출 표시가 primary 토큰을 쓰는 다른 지점**(`PayrollConfirm`·`AdminPayrollSettings`·`admin/page.jsx`의 매출 KPI·`PtConfirmBanner`의 긍정 확정 표식 등) — 있으면 동일 규칙으로 success화. 없으면 스킵.

### 리터럴 emerald는?
- 이미 `from-emerald-400 to-emerald-600` 등 **리터럴 초록**인 돈/게이지(예: 목표 게이지, 급여 진행바)는 **그대로 두면 초록 유지** — 이번엔 건드리지 않는다(선택적 후속 토큰화).
- 단 리터럴 emerald가 **버튼(완료·저장 등 액션)**이면 그건 §3의 Button 이관 대상 → 레드로 수렴(게이지·바와 구분: 프로그레스바 채움은 절대 교체 금지).

## 3. Button 프리미티브 색 개정 (Button 스펙 §1 supersede)

`v2-스펙-Button프리미티브.md`의 컴포넌트에서 **`VARIANT` 맵과 danger만 아래로 교체.** BASE·SIZE·API(props)·`as`/`fullWidth`/`subtle`·배치 롤아웃·마이그레이션 규칙은 **그대로.**

```jsx
// variant → 색. accent 있는 variant는 {trainer, owner}, 없는 것은 문자열.
const VARIANT = {
  primary: {
    // 브랜드 레드 채움 + 흰 텍스트 — 로고(흰 모노그램 on 레드)를 버튼에서 에코.
    trainer: "bg-gradient-to-br from-red-500 to-red-600 text-white",
    owner:   "bg-gradient-to-br from-fuchsia-500 to-purple-600 text-white",
  },
  solid: {
    // 토큰 기반 → 자동 레드(중강조 소프트 채움).
    trainer: "bg-primary-soft text-primary-strong ring-1 ring-inset ring-primary/30",
    owner:   "bg-fuchsia-500/10 text-fuchsia-700 ring-1 ring-inset ring-fuchsia-500/30",
  },
  ghost: {
    // 토큰 기반 → hover가 자동 레드.
    trainer: "border border-line bg-elevate text-sub hover:border-primary hover:text-primary-strong",
    owner:   "border border-line bg-elevate text-sub hover:border-fuchsia-500/60 hover:text-fuchsia-700",
  },
  // 돈/긍정 확정 전용(초록 유지) — 예: 급여 관련 확정 액션. 대부분 버튼은 primary(레드).
  success: "bg-gradient-to-br from-emerald-400 to-emerald-600 text-zinc-950",
  // 파괴적 = 아웃라인 로즈(브랜드 레드 '채움'과 분리 · 채움 그라데이션은 브랜드 전용).
  danger: "border border-rose-500/40 bg-rose-500/5 text-rose-700 hover:border-rose-500/70 hover:bg-rose-500/10",
};

// 파괴적 저강도 인라인 트리거(보더 없이 텍스트만).
const DANGER_SUBTLE = "text-rose-600 hover:bg-rose-500/10";
```

- 컴포넌트 본문의 resolver는 기존과 동일(`variant === "danger" && subtle ? DANGER_SUBTLE : ...`). `success`는 문자열 variant라 accent 무관(danger와 같은 처리 경로).
- **핵심 원칙 — 채움 그라데이션은 이제 "브랜드 레드" 전용.** danger는 **절대 채움 레드로 만들지 않는다**(브랜드 CTA와 혼동 방지). 기존 `from-rose-500 to-rose-600 text-white`(환불 실행 등) 채움 확정 → **아웃라인 로즈**(`variant="danger"`)로 강등. 파괴적 액션이 브랜드 CTA보다 덜 튀는 게 오히려 옳음(UX).
- 고위험 확정(예: 환불 실행)을 조금 더 무겁게 하고 싶으면 `className="font-bold bg-rose-500/10"` 정도만 가산 — **채움 그라데이션 금지.**

### 변주 매핑 개정 (Button 스펙 표 교체)
| 기존 인라인 | Button |
|---|---|
| `from-lime-400 to-emerald-500` · `from-emerald-400 to-emerald-600` (트레이너 CTA) | `variant="primary"` → **레드 채움** |
| `from-fuchsia-500 to-purple-600` (원장 CTA) | `variant="primary" accent="owner"` |
| `bg-primary-soft text-primary-strong ring...` (중강조 · **토글 제외**) | `variant="solid"` → 레드 소프트 |
| `border border-line bg-elevate ... hover:border-primary` (보조) | `variant="ghost"` → hover 레드 |
| 급여/돈 관련 **긍정 확정 버튼**(있다면) | `variant="success"` → 초록 유지 |
| `from-red-500/rose-500 to-rose-600 text-white` (삭제·환불 확정) | `variant="danger"` → **아웃라인 로즈** |
| `text-red-600 hover:bg-red-500/10` (인라인 삭제 트리거) | `variant="danger" subtle` |

## 4. 시퀀싱 (커밋 순서)

1. **토큰 재편(§1) — 1커밋.** `git add app/globals.css` → `feat(theme): 브랜드 레드 전환 + success(초록) 토큰 분리`. 이 시점에 선택·포커스·활성·소프트배지가 레드로. (급여 카드 등은 아직 §2 전이라 잠깐 레드일 수 있음 → 바로 다음 커밋에서 success로.)
2. **돈/성공 success 재지정(§2) — 1커밋**(또는 파일별). `feat(theme): 급여·매출·성공 surface success(초록) 재지정`. 폰에서 급여=초록, 선택/포커스=레드 확인.
3. **Button 프리미티브(§3 반영) — 배치 0.** Button.jsx를 **레드 색 맵으로** 생성.
4. **Button 배치 1~4** — 기존 `v2-스펙-Button프리미티브.md §2` 그대로. CTA가 레드로, danger가 아웃라인 로즈로 수렴.

> 즉 **토큰(1·2) 먼저 → Button(3·4).** 토큰만 먼저 올려도 앱이 즉시 "레드 브랜드"로 보이기 시작(선택·포커스·헤더). CTA 색은 Button 배치를 따라 순차 수렴.

## 5. 유지(안 건드림) · 열린 노트

- **OT=amber · PT=sky** 그룹색 유지(레드와 다른 탭 그룹이라 충돌 낮음).
- **원장=fuchsia** 유지. ※ 열린 노트: 레드 브랜드 옆에서 fuchsia(마젠타)가 다소 근접해 보일 수 있음. '원장 모드' 구분 신호라 이번 스코프에선 유지하되, 나중에 원장 accent를 더 멀리(예: 인디고/슬레이트) 옮길지는 별도 판단.
- 리터럴 emerald 중 **돈/게이지/성공**은 초록 유지, **버튼**만 레드 이관.
- `zinc`/`slate` 중립 베이스·카드·라인 전부 불변.
