## 설계 원칙
네이티브 하단바는 **4~5개**가 적정. 네 탭은 글로벌 4(오늘·회원·내실적·설정) + 워크플로우 OT/PT 3이라 다 넣으면 과함. 그래서:
- **하단 고정바 = 글로벌 4개**(항상 보임) — 앱의 최상위 목적지.
- **상단 컨텍스트 탭 = 워크플로우(OT/PT) 서브탭** — 회원 워크플로우 탭(1차·관찰·2차 / 운동일지·인바디·재등록)에 들어갔을 때만 상단에 노출.

**핵심: `tab` 상태·`setTab`·view 보정 로직은 그대로.** 네비 렌더 위치만 재배치 → 회귀 위험 최소.

## 1. 신규 — `components/ui/BottomNav.jsx`

```jsx
"use client";
import { CalendarDays, Users, BarChart3, Settings } from "lucide-react";

// 하단바 = 글로벌 4탭(아이콘 포함). id는 기존 TABS와 동일.
const ITEMS = [
  { id: 9, label: "오늘",    Icon: CalendarDays },
  { id: 0, label: "회원",    Icon: Users },
  { id: 8, label: "내 실적", Icon: BarChart3 },
  { id: 7, label: "설정",    Icon: Settings },
];

// 현재 tab → 활성 섹션. 워크플로우 탭(1,5,2,10,11,12)·회원목록(0) 전부 '회원' 섹션(0)으로.
const sectionOf = (tab) => (tab === 9 ? 9 : tab === 8 ? 8 : tab === 7 ? 7 : 0);

export default function BottomNav({ tab, onTab }) {
  const active = sectionOf(tab);
  return (
    <nav
      className="fixed inset-x-0 bottom-0 z-40 border-t border-line bg-card/95 backdrop-blur-xl pb-[env(safe-area-inset-bottom)]"
      aria-label="주요 메뉴"
    >
      <div className="mx-auto grid max-w-5xl grid-cols-4">
        {ITEMS.map(({ id, label, Icon }) => {
          const on = active === id;
          return (
            <button
              key={id}
              onClick={() => onTab(id)}
              aria-current={on ? "page" : undefined}
              className={`flex flex-col items-center gap-0.5 py-2 text-[10px] font-semibold transition active:scale-95 ${
                on ? "text-primary-strong" : "text-muted hover:text-ink"
              }`}
            >
              <Icon className="h-5 w-5" strokeWidth={on ? 2.5 : 2} />
              {label}
            </button>
          );
        })}
      </div>
    </nav>
  );
}
```
- `pb-[env(safe-area-inset-bottom)]` = 홈바(제스처바) 영역 확보.
- 활성=레드(primary-strong)·굵은 아이콘, 비활성=muted. `active:scale-95` 눌림 반응.

## 2. `app/page.jsx` — 상단 워크플로우 탭만 남기고 + 하단바 장착

### 2-1. import
```jsx
import BottomNav from "@/components/ui/BottomNav";
```

### 2-2. 상단 `<nav>` — 글로벌 제거, 워크플로우 탭만·워크플로우 탭에서만 노출
현재(글로벌+워크플로우 섞이고 구분선 있는) `<nav>` 블록을 아래로 교체. **워크플로우 탭(group 있는 탭)일 때만** 렌더, 필터도 워크플로우만, 구분선 로직 삭제:

```jsx
{TABS.find((t) => t.id === tab)?.group && (
  <nav className="-mb-px flex items-stretch gap-1 overflow-x-auto whitespace-nowrap">
    {TABS
      .filter((t) => (t.ot && view === "ot") || (t.pt && view === "pt"))
      .map((t) => {
        const on = tab === t.id;
        const g = GROUP_TAB[t.group];
        return (
          <button
            key={t.id}
            onClick={() => setTab(t.id)}
            className={`relative px-3 py-2.5 text-xs font-semibold transition sm:px-4 ${on ? g.active : g.idle}`}
          >
            {t.label}
            {on && <span className={`absolute inset-x-2 bottom-0 h-0.5 rounded-full ${g.bar}`} />}
          </button>
        );
      })}
  </nav>
)}
```
> 결과: 오늘·회원목록·내실적·설정에선 상단 서브탭 없음(깔끔). 회원 워크플로우(1/5/2/10/12/11)에 들어가야 상단에 OT(amber)/PT(sky) 서브탭 표시. 서브탭 간 이동은 상단, 섹션 이동은 하단바.

### 2-3. 하단바 장착
루트 `<div className="min-h-screen bg-bg pb-28 ...">`의 **맨 끝**(닫는 `</div>` 직전, 모달들과 나란히)에:
```jsx
<BottomNav tab={tab} onTab={setTab} />
```
- 루트의 `pb-28`(112px)이 하단바(≈56px+safe-area)를 덮을 여백을 이미 확보 — 콘텐츠가 바 뒤로 안 숨음. 유지.

### 2-4. (정리) 이제 안 쓰는 것
- 상단 `<nav>`에서 `showSep`/구분선·`always` 글로벌 렌더는 사라짐. `TABS`의 `always` 플래그 자체는 남겨도 무해(BottomNav가 자체 ITEMS 사용). 삭제 안 해도 됨.

## 3. `components/AuthGate.jsx` — 로그아웃 버튼을 하단바 위로
플로팅 로그아웃(현재 `fixed bottom-4 right-4`)이 하단바에 겹침. 위로 띄우기:
```jsx
className="fixed bottom-20 right-4 z-40 ..."   // bottom-4 → bottom-20 (하단바 위)
```
> (후속 선택) 로그아웃을 아예 **설정(SettingsView) 화면 안으로** 옮기면 플로팅 버튼 자체가 없어져 더 깔끔 — 이번 스코프 밖.

## 검토 체크리스트
- [ ] 하단바 4개(오늘·회원·내실적·설정) 항상 노출, 활성 섹션 하이라이트(워크플로우/회원목록 → '회원' 활성).
