## 1. 관통 정책 (이걸 어기면 시스템이 무너진다)

### A. 토큰은 "이름 유지 · 값만 교체"
`globals.css`의 토큰 **이름은 앱 것을 유지하고 값만 DS 기준으로** 바꿨다. 유틸 사용처 1,962곳
(`muted` 489·`primary` 390·`line` 291·`ink` 233…)이 **한 줄도 안 바뀌고 자동 반영**되기 때문.
- DS 이름(`--red` 등)을 새로 만들면 같은 색에 이름이 둘이 되고, `bg-red`가 Tailwind 기본 `bg-red-500`과 혼동된다.
- 같은 정책이 라운드·그림자에도 적용된다 — Tailwind 기본값(`--radius-2xl`·`--shadow-sm`)을 **덮어써서** 기존 클래스가 그대로 새 규격을 따르게 했다. `rounded-card`·`shadow-card` 같은 새 유틸은 **앱에 없다**(만들면 드리프트).

### B. purge-safe 정적 리터럴만
색 클래스는 **완성된 문자열 리터럴**이어야 Tailwind purge가 남긴다. `` `text-${x}-500` `` 같은 **동적 조립 금지**.
재사용 톤은 전부 **정적 맵**에 둔다: `WIDGET_TONE`/`widgetTone`(tone.js), `BADGE_TONE`(Badge), `VIEW_META`(MemberBadge), `VARIANT`(Button), `GROUP_TAB`(page.jsx). 새 변형은 이 맵에 **리터럴 항목을 추가**한다. (현재 위반 0건 — 유지할 것.)

### C. 역할색 ≠ 브랜드색
- **브랜드색 = 레드**(`--color-primary`). 로고·CTA·"선택됨"·우선순위 신호.
- **역할색 = 업무 흐름 구분**(`--color-ot` amber / `--color-pt` sky / `--color-admin` fuchsia / `--color-danger` rose). **로고·CTA에 쓰지 않는다.**
- ⚠️ 역할색 **채움값**(`--color-ot` 등)은 아이콘·테두리·점에만. **밝은 배경 위 글자는 반드시 `-text`**(`--color-ot-text`) — 채움값은 흰 배경에서 대비 미달(amber `#f59e0b` = 2.15).
- 한 화면에서 색 하나가 두 뜻을 가지면 안 읽힌다. 예: OT/PT 필터 칩이라도 "선택됨"은 **레드**로, 흐름 구분은 뱃지에서.

### D. 한글 라벨 자간 예외
라틴 라벨(TODAY·OT)은 `+.16em uppercase`가 맞지만 **한글은 무효**(uppercase 효과 없음)이고 `.16em`이면 자소가 벌어져 "내 프로필"이 "내  프 로 필"로 읽힌다. 한글 라벨은 `--tracking-label-ko`(0.02em)를 쓴다.
- `Eyebrow`는 렌더 시 정규식으로 한글 유무를 판별해 자동 분기한다. `Badge`·`Eyebrow`는 한글에서 자간을 안 벌린다.

## 3. 프리미티브 카탈로그 (`components/ui/`)

**인라인으로 카드/폼/모달을 새로 만들지 말 것 — 아래를 쓴다.** 모든 색 클래스는 정적 리터럴.

### 컨테이너

**`Card` / `CardRow`** (`Card.jsx`) — 기본 흰 카드 셸.
- props: `as`(기본 div)·`interactive`·`selected`(레드 테두리+soft 배경)·`elevated`(기본 true=shadow-sm)·`padding`(`md`p-5 / `sm`p-4 / `lg`p-6 / `none`p-0).
- `padding="none"`은 **자동 `overflow-hidden`**(분할 리스트 — 행 배경이 라운드 밖으로 삐져나옴 방지). 그 안의 행은 `CardRow`(첫 행만 구분선 없음).
- ⚠️ 구 톤 카드가 아니다 — 톤 틴트는 `ToneCard`.

**`ToneCard`** (`ToneCard.jsx`) — 파생 위젯 틴트 컨테이너. `tone`별 테두리+틴트 배경(`widgetTone` 소비). 기본 `mb-4 rounded-2xl border p-4`, **shadow 없음**(틴트 카드 결).

### 폼 (`Field.jsx`)

**`Input` · `Textarea` · `Select` · `Checkbox`** — 라벨·hint·error·`accent`(trainer/owner) 축.
- 아직 컴포넌트로 안 옮긴 raw 필드는 공유 클래스 `inputCls`/`inputClsOwner`/`inputClsSm` 사용(21개 파일 복사본을 여기로 수렴).
- 크기: `md`(px-3.5 py-[11px] text-[15px]) / `sm`(px-2.5 py-1.5 text-sm).
- ⚠️ **테두리는 `border`(ring 아님), 배경은 `bg-elevate`.** ring(box-shadow)은 iOS Safari가 네이티브 `<select>`에 안 그려서 필드가 통째로 사라졌다(§5). focus는 색만 바꿔(두께 유지) 밀림 없음.
- `Select`는 `appearance-none` + 직접 그린 chevron(OS 기본 화살표는 플랫폼마다 달라 기기별로 다르게 보임).

**`NumberInput`** (`NumberInput.jsx`) — 천단위 콤마 정수 입력. `Field`의 `Input`에 위임(17곳 규격 통일). **부모 계약: `value`=콤마 없는 숫자 문자열, `onValueChange`=(raw)=>void** — 화면에만 콤마, 저장·검증은 부모가 기존대로. 캐럿 복원 로직 있음(React 19 ref=일반 prop). **정수 전용**(소수·% 금지).

### 모달

**`Modal`** (`Modal.jsx`) — **포털(`createPortal`→body)** 다이얼로그.
- props: `variant`(center/sheet)·`size`(sm/md/lg · 생략 시 center=md·sheet=lg)·`title`/`subtitle`/`footer`·`dismissable`·`blocking`(필수공지 — 닫기·배경클릭·ESC 전부 무시)·`onClose`.
- 포커스 트랩·ESC·배경 스크롤 잠금·`--shadow-pop`.
- **하위호환:** `title`·`footer` 없이 쓰면 기존 레이아웃(children이 곧 내용, p-5). `title`을 주면 DS 구조(헤더/본문/푸터).
- ⚠️ 포털인 이유는 조상 transform 캡처 버그의 **구조적** 해결(§5). 손수 만든 모달 대신 이걸 쓴다.

### AI 블록

**`AIBriefBlock`** (`AIBriefBlock.jsx`) — AI 사전무장/브리핑 단일 출처(1차 FirstOTAssist·2차 SecondOTTab·재등록 PtReRegTab 공유).
- props: `status`(idle/loading/ready/stale/demo)·`title`·`waitingHint`·`idleDescription`·`meta`·`notice`·`onGenerate`/`onRegenerate`·`generateLabel`·children(본문).
- ⚠️ **`loading`은 반드시 `waitingHint`와 함께** — "45초~1분 빈 화면 금지"가 존재 이유(PRD 리스크 1순위). 없이 쓰면 컴포넌트 목적 무력화.
- `stale`=amber 뱃지(재생성 권장), `demo`="데모" 뱃지. 좌측 3px 레드 강조선은 **상태 무관 항상 유지**(가장 오래 보이는 loading에서 브랜드 강조 유지). 재생성 버튼은 좁은 폭에서 아이콘만(aria-label 유지).

### 버튼 (`Button.jsx`)

앱 전역 버튼 단일 출처. `variant × accent × size` → 정적 리터럴.
- `variant`: `primary`(레드 채움 그라데이션) · `solid`(soft 배경+ring) · `ghost`(기본 · 테두리) · `danger`(아웃라인 rose · `subtle`=저강도 텍스트).
- `accent`: `trainer`(레드) / `owner`(admin fuchsia). danger엔 accent 없음.
- `size`: `sm`(text-[13px]) / `md`(text-[14.5px]). 모서리 **`rounded-lg`(10px)** — 12px는 금지된 중간값이라 65곳 변경됨. `lg` 사이즈는 규격엔 있으나 **미도입**(쓰임 0).
- `fullWidth`(폼 CTA) · `as="a"`(링크 버튼). 아이콘은 children으로. **초록 없음.**

## 4. 톤 시스템 (`tone.js`) — 역할 이름 체계

파생/할일 위젯의 시맨틱 톤. `ToneCard`·`SectionHeader`·`ListRow`가 공유. **역할 이름**을 쓴다(색 이름은 하위호환 별칭).

| 역할 이름 | 색 | 뜻 | 별칭(레거시) |
|---|---|---|---|
| `reapproach` | amber(OT) | 재접근 — OT 흐름 | `amber` |
| `renewal` | sky(PT) | 재등록 — PT 흐름 | `emerald`(레거시=red였음→renewal로 교정) |
| `unclosed` | **red** | 미확정 클로징 — 우선순위 신호(통계·자기진화 원천) | — |
| `brand` | red | 화면 최상 강조(급여 헤드라인 등) | — |
| `neutral` | zinc | 미처리 예약·중립 | `zinc` |
| `danger` | rose | 환불·손실 전용 | `rose` |

> ⚠️ 배정이 명세와 반대였던 것을 교정: 미확정 클로징=**red**(rose 아님), 재등록=**sky**(red 아님). 위험색(rose)은 환불·손실에 쓰는 색이지 "결과 미기록"에 쓸 색이 아니다.
> 새 코드는 **역할 이름**을 쓴다. 색 이름 별칭은 소비처를 한 번에 못 바꿔 남겨둔 것.

관련: `Badge`의 `BADGE_TONE`, `MemberBadge`의 `VIEW_META`, `page.jsx`의 `GROUP_TAB`(OT=amber·PT=sky, admin=fuchsia)도 같은 역할색 토큰에서 온다.

## 5. 현장 함정 모음 (되돌린 이유 — 다시 밟지 말 것)

이미 코드 주석·CLAUDE.md에 흩어진 것을 한곳에 집약.

- **입력칸이 안 보였다** — `bg-card`(흰 카드 위 흰 배경) + `ring`(iOS Safari가 네이티브 `<select>`에 box-shadow 미표시)의 조합. → `bg-elevate`(눌린 면) + `border`로 되돌림. focus는 색만 바꿈(두께 유지=밀림 없음). *(Field.jsx)*
- **`position:fixed` 모달이 화면 밖으로** — 조상에 transform이 있으면 `fixed`가 뷰포트가 아니라 그 조상 기준으로 앵커된다. `.tab-anim`의 `fill-mode:both`가 남긴 transform이 범인이었다. 2겹 해결: ① fill-mode를 `backwards`로 ② `Modal`을 **포털로 body에** 붙임(조상 소멸=재발 불가). *(globals.css / Modal.jsx)*
- **워드마크가 "오직 트레이/너"로 쪼개짐** — 390px 폰에서 단어 중간 잘림. → `Wordmark`에 `whitespace-nowrap`(브랜드 이름은 shrink 대상 아님, 헤더 다른 요소가 줄어들게). *(Wordmark.jsx)*
- **폼이 폰에서 2열로 깨짐** — `col-span-2`를 `grid-cols-1`에 쓰면 CSS Grid가 암묵적 2열을 만든다. → `sm:col-span-2`. *(page.jsx)*
- **한글 라벨 자간이 벌어짐** — `uppercase tracking-[0.2em]`(라틴 규격)을 한글에 쓰면 "내  프 로 필". → `Eyebrow` 정규식 분기 + `--tracking-label-ko`. *(Eyebrow.jsx / Badge.jsx)*
- **다크모드 검은 화면** — Next 스타터 `--background` 잔재. 제거. *(globals.css)*
- **AI 진행바가 저사양 폰에서 버벅** — `margin-left` 애니는 매 프레임 레이아웃 재계산. → `transform`. *(globals.css)*
- **컴포넌트명 `Symbol` 금지** — JS 내장 전역이라 import한 파일에서 내장 Symbol이 가려짐. → `BrandMark`. *(BrandMark.jsx)*
- **`.next` stale = 유령 버그** — 증상이 "됐다 안 됐다" 하거나 `npm run build`는 green인데 dev만 이상하면 코드 파기 전에 `.next` 삭제+재시작부터 의심.

## 6. 유지보수 규칙 (수정·확장할 때)

- **새 변형(색/톤) 추가** → 해당 정적 맵(`WIDGET_TONE`·`BADGE_TONE`·`VARIANT`·`VIEW_META`)에 **완성 클래스 리터럴** 한 줄 추가. 절대 동적 조립하지 않는다.
- **토큰 값 변경** → `globals.css`의 값만 바꾼다. 이름·유틸 클래스는 그대로(그래서 전 앱에 자동 반영). 새 토큰 이름을 만들면 드리프트(§1-A).
- **새 컴포넌트 추가** → 셸(카드/그림자/라운드)은 기존 프리미티브에 **위임**(StatTile→Card, NumberInput→Field처럼). 손수 문자열 복제 금지.
- **역할색을 글자에** → 반드시 `-text` 변형. 채움값은 대비 미달(§1-C).
- **CLAUDE.md 동기화** → 구조가 바뀌면 CLAUDE.md "디자인 시스템 프리미티브" 인덱스 한 줄과 이 문서를 함께 갱신. (코드-문서 어긋남이 회귀의 가장 그럴듯한 진입점.)

## 7. 미도입 · 보류 (의도적으로 안 넣은 것)

- **Button `lg` 사이즈** — 규격엔 있으나 앱에 쓰임 0. 필요한 화면 생길 때.
- **`success`(초록) 톤** — 이 브랜드는 레드·먹색 2색. 앱에 초록 0곳. 넣으면 색이 세 벌 → 클로드 디자인에 확인 요청 중.
- **전역 본문 행간 1.7** — Tailwind text-* 유틸이 이미 크기별 행간을 가져, 전역 적용 시 유틸 없는 요소만 늘어나 세로 크기 예측 불가. 행간은 컴포넌트 단위로.
- **`Chip`→`MetaTag` 개명** — 장기적으론 맞지만(FilterChip과 이름 혼동) 별개 작업.
- **`--shadow-sm` 얹힌 비카드 7곳** — DS상 그림자는 카드 전용. 정리 대상.
- **`Sparkline` 자기복사본**(PtInbodyTab) — 공용으로 수렴 대상.
