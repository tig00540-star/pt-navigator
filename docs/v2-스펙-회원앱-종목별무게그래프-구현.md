## 0. 무엇을 만드나 (한 줄)

Phase 1로 축적 중인 `daily_workout_log.sets_structured`를 **회원앱(`/m`)에 "종목별 무게 변화" 그래프**로 노출한다. 안전 definer 뷰(`member_workout_log`)에 숫자 컬럼만 얹고, **트레이너 화면에서 쓰던 `buildExerciseSeries`·`Sparkline` 공용 헬퍼를 그대로 재사용**한다. "벤치프레스 20 → 30kg" 성과 가시화 = **재등록 유도**.

**핵심:** 새 데이터·새 집계 로직은 없다. (a)뷰에 컬럼 노출 + (b)`/m` 페이지에 렌더 섹션 하나. 회원앱 규약(안전 뷰만·민감컬럼 제외)을 지킨다.

## 2. 데이터 계약

### 2.1 `member_workout_log` 뷰 (변경 후)

```
select id, created_at, ai_summary, session_at, sets_structured
from daily_workout_log
where user_id = auth_member_id() and coalesce(voided, false) = false;
```

- **기존 컬럼(`id, created_at, ai_summary`) 순서·타입 불변** — `create or replace view`는 기존 출력 컬럼을 삭제·재정렬·재타입 못 함. 새 컬럼은 **반드시 목록 끝에 append**(`session_at`·`sets_structured`). 순서 어기면 replace 에러 → `drop view` 필요해지므로 주의.
- **`WHERE`에 `voided` 필터 신규 추가.** 뷰 정의(쿼리)는 replace로 자유 변경 가능(컬럼 목록 제약과 무관).
- **노출 안전성:** `session_at`(시각)·`sets_structured`(종목명+무게/횟수 숫자)뿐 → 회원 노출 안전. **민감컬럼 계속 제외**(`raw_voice_text`·`note`·`member_note` 등은 애초에 select 안 함).
- jsonb는 Supabase가 **파싱된 JS 배열**로 반환. `Array.isArray` 가드로 충분(구버전 문자열로 오면 방어적으로 무시).

### 2.2 `sets_structured` 모양 (Phase 1과 동일 · 불변)

```json
[
  { "exercise": "벤치프레스", "sets": [ {"weight":20,"reps":12}, {"weight":30,"reps":12} ] },
  { "exercise": "랫풀다운",  "sets": [ {"weight":40,"reps":15} ] }
]
```

- `exercise` = canonical 집계 키. `weight`는 kg 숫자 또는 null(맨몸·미언급). `reps` 숫자 또는 null.
- 전체 null/`[]` = 구조 데이터 없음 → 그래프 원천 자동 제외.

### 2.3 `buildExerciseSeries` 입력 (매핑 불필요)

- 뷰가 `session_at`을 직접 노출하므로 헬퍼가 `session_at ?? created_at`로 날짜를 바로 씀 — **핸드오프 §1의 `l => ({...l, session_at: l.created_at})` 매핑은 이제 불필요**(뷰가 실제 세션 시각을 준다 · `session_at` null인 과거행은 헬퍼가 `created_at` fallback).
- `voided`는 뷰에서 이미 걸러져 뷰 행엔 없음 → 헬퍼의 `!l.voided`는 `undefined` 통과(무해).
- 페이지가 `order("created_at", desc)`로 받아도 헬퍼가 내부에서 오래된→최신 재정렬하므로 **수신 순서 무관**.

## 3. 파일별 변경

### 3-A. DB 마이그레이션 (신규 파일 · Supabase 수동 실행)

**`docs/migrations/2026-07-16-member-workout-structured.sql`**

```sql
-- =============================================================================
-- ③ Phase 2 — 회원앱 종목별 무게 그래프: member_workout_log 뷰에 구조화 컬럼 노출
-- 실행: Supabase SQL Editor(수동) · 이 파일 = git 기록본. 멱등(create or replace view).
--
-- ★ 하는 일:
--   1) member_workout_log 뷰 재생성 — 기존 컬럼 뒤에 session_at·sets_structured append.
--   2) WHERE에 voided 제외 추가(무른 세션 = 회원 타임라인·그래프 양쪽에서 사라짐 · 의도).
-- ★ 안전: 노출은 시각·숫자(종목명/무게/횟수)뿐. raw_voice_text·note 등 민감컬럼은 계속 미노출.
-- 전제: S2(member_workout_log 뷰) 존재 · S1 auth_member_id() 헬퍼.
-- 주의: create or replace view는 기존 출력 컬럼을 삭제/재정렬/재타입 불가 → 새 컬럼은 목록 끝.
-- =============================================================================
create or replace view member_workout_log
  with (security_invoker = false) as
  select id, created_at, ai_summary, session_at, sets_structured
  from daily_workout_log
  where user_id = auth_member_id()
    and coalesce(voided, false) = false;

grant select on member_workout_log to authenticated;
revoke select on member_workout_log from anon;

-- =============================================================================
-- 검증
--   (A) 컬럼 확인:
--     select column_name from information_schema.columns
--       where table_name='member_workout_log';   -- id·created_at·ai_summary·session_at·sets_structured
--   (B) ★회원 세션(S1 access_token)으로 REST:
--     GET /rest/v1/member_workout_log?select=*   → sets_structured 포함, raw_voice_text 없음, voided 세션 미포함.
-- ROLLBACK (컬럼·필터 원복):
--   create or replace view member_workout_log with (security_invoker=false) as
--     select id, created_at, ai_summary from daily_workout_log where user_id = auth_member_id();
-- =============================================================================
```

> 다른 뷰(`member_me`·`member_inbody`)는 **건드리지 않음**.

### 3-B. `app/m/[token]/page.jsx` — 그래프 섹션 + Sparkline 공용화

**(1) import 정리 (공용 Sparkline 통합)**

- 상단 로컬 `function Sparkline(...)`(현 31~49행)을 **삭제**하고 공용 컴포넌트로 교체:
  ```jsx
  import Sparkline from "@/components/ui/Sparkline";
  import { buildExerciseSeries } from "@/lib/workout";
  ```
- 인바디 섹션의 `<Sparkline .../>` 사용부는 **그대로**(공용 구현이 로컬과 동일 코드 = 무회귀). 통합으로 드리프트 감소.
- lucide import에 `Dumbbell` 추가(섹션 Eyebrow 아이콘). `TrendingUp/TrendingDown/Minus`는 이미 import됨(재사용).
- > 통합이 부담되면 **로컬 Sparkline 유지 + 그래프 섹션만 로컬 Sparkline 재사용**도 허용(둘이 동일 구현이라 무해). 단 이 경우 `buildExerciseSeries` import는 필수.

**(2) 파생 계산 (`HomeView` 내부, 인바디 파생 근처)**

```js
// 종목별 무게 추이 — 무게 point가 하나라도 있는 종목만(맨몸만 있는 종목 제외).
const exerciseSeries = buildExerciseSeries(logs).filter((s) =>
  s.points.some((p) => p.topWeight != null)
);
```

- `logs`는 뷰에서 `session_at`·`sets_structured` 포함해 이미 로드됨(추가 쿼리 0).

**(3) 무게 delta 톤 (파일에 이미 있는 `DELTA_TONE` 맵 재사용)**

- 무게는 **증가=good** 단순 규칙(인바디의 `deltaTone`은 `goodDir` 기반이라 여기 안 씀). 로컬 헬퍼:
  ```js
  // 무게 변화 톤 — 증가=good, 감소=bad, 동일=flat. DELTA_TONE 맵(파일 상단 기존)과 함께 씀.
  const weightTone = (d) => (d > 0 ? "good" : d < 0 ? "bad" : "flat");
  ```
- `DELTA_TONE = { good:"text-primary-strong", bad:"text-rose-600", flat:"text-muted" }`는 이미 파일 19행에 있음(purge-safe · 동적 조립 금지 규칙 유지).

**(4) 그래프 섹션 JSX** — "인바디 변화" `</section>` 아래, 로그아웃 위에 삽입. **무게 데이터 0이면 섹션 자체를 렌더 안 함**(빈 카드 없음):

```jsx
{exerciseSeries.length > 0 && (
  <section className="mb-8">
    <Eyebrow icon={Dumbbell}>종목별 무게 변화</Eyebrow>
    <ul className="space-y-3">
      {exerciseSeries.map((ex) => {
        const wpts = ex.points.filter((p) => p.topWeight != null); // 무게 있는 point만
        const latest = wpts[wpts.length - 1]?.topWeight ?? null;
        const first = wpts[0]?.topWeight ?? null;
        const prev = wpts.length > 1 ? wpts[wpts.length - 2].topWeight : null;
        const d = prev != null && latest != null ? latest - prev : 0; // 직전 대비
        const tone = weightTone(d);
        const Icon = d > 0 ? TrendingUp : d < 0 ? TrendingDown : Minus;
        // 격려 문구(성과만 · 의료/처방 없음): 첫→지금 상승이면 강조, 아니면 담백.
        const cheer =
          first != null && latest != null && wpts.length > 1 && latest > first
            ? `처음 ${first}kg에서 지금 ${latest}kg까지 올라왔어요.`
            : wpts.length > 1 && latest != null && first != null && latest === first
            ? `${latest}kg 꾸준히 유지하고 있어요.`
            : null;
        return (
          <li key={ex.exercise} className="rounded-2xl border border-line bg-card p-5 shadow-sm">
            <div className="flex items-baseline justify-between gap-3">
              <div className="text-lg font-bold text-ink">{ex.exercise}</div>
              <div className="font-mono text-2xl font-bold text-ink">
                {latest == null ? "–" : latest}
                <span className="ml-1 text-xs font-normal text-muted">kg</span>
              </div>
            </div>
            {prev != null && latest != null && (
              <div className={`mt-1 flex items-center justify-end gap-1 text-[13px] font-semibold ${DELTA_TONE[tone]}`}>
                <Icon className="h-3.5 w-3.5" /> 직전 대비 {d > 0 ? "+" : ""}{Math.round(d * 10) / 10}kg
              </div>
            )}
            <Sparkline values={ex.points.map((p) => p.topWeight)} />
            {cheer && <p className="mt-2 text-sm text-primary-strong">{cheer}</p>}
          </li>
        );
      })}
    </ul>
  </section>
)}
```

- 회원 톤: 카드 1열·글씨 크게(`text-lg`/`text-2xl`)·정보밀도 낮게. 인바디 카드와 톤 통일(`border-line bg-card rounded-2xl shadow-sm`).
- `Sparkline`은 null point를 내부에서 걸러내고 값 2개 미만이면 null 반환(한 번만 기록된 종목은 선 없이 숫자·문구만 — 정상).
- 데이터가 있어도 무게가 전부 같은 값이면 span=1로 평평한 선(헬퍼 기존 동작 · 문제 없음).

## 6. 주의·엣지

- **`create or replace view` 컬럼 제약(재확인):** 기존 `id·created_at·ai_summary` **뒤에만** 새 컬럼 추가. 앞에 끼우거나 순서 바꾸면 `cannot change name/type of view column` 에러. (본 스펙 SQL은 이미 끝에 append.)
- **Sparkline 통합 회귀:** 로컬 삭제 시 인바디 섹션이 같은 공용 컴포넌트를 쓰게 됨 — 구현이 동일해 무회귀지만, 검증에서 인바디 그래프 한 번 눈으로 확인. 부담되면 로컬 유지 + 그래프 섹션만 재사용(§3-B(1) 대안).
- **voided 뷰 필터의 부수효과(의도됨):** 회원 "내 수업일지" 타임라인에서도 voided 세션이 사라짐. 이게 싫고 **타임라인엔 남기되 그래프만 빼려면** → 뷰 WHERE 필터 대신 `voided`를 컬럼으로 노출하고 그래프 파생에서 `buildExerciseSeries`가 `!l.voided`로 거르게 전환(대안). 현 결정 = 뷰에서 제외.
- **회원앱은 실데이터 전용:** `memberSupabase` 없으면 애초에 로그인 불가(데모 경로 없음) → 그래프도 별도 데모 배선 불필요.
- **`session_at` null 과거행:** 헬퍼가 `created_at`로 fallback(무해). `created_at`은 뷰에 계속 노출(페이지 `order("created_at")` 키로 사용 중).
- **과거 미보정 데이터:** Phase 1 이전·구조화 미입력 세션은 `sets_structured=null` → 자동 제외. 회원 그래프는 Phase 1 배포 이후 축적분부터 자연 반영.
- **UPDATE 정책 없음(교훈1):** 이번 범위는 조회(뷰)·렌더뿐. 어떤 테이블도 UPDATE/INSERT 안 함 → 정책 변경 없음.
