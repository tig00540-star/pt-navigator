# v2 스펙 — 회원 앱 S3 (트레이너 링크 발급 UI) · 구현

> 성격: **구현 스펙**(클로드코드 실행용). 전제 = S1(인증·RLS)·S2(회원 홈·안전뷰) 완료·배포·검증. 설계 근거 = `docs/v2-설계-회원앱-MVP-아키텍처.md` §4.
> 스코프: 트레이너가 회원의 **회원앱 링크(`/m/<token>`)를 생성·복사·재발급**하는 UI. 이거 하면 손으로 SQL 안 만지고 회원앱 v1(인증·열람·발급) **완결**.
> 흐름: 이 스펙 → 트레이너 → 클로드코드(diff) → 트레이너(git diff) → 웹Claude(검토) → 폰 검증. 커밋은 트레이너 직접(파일지정 `git add` · `npm.cmd run lint` 통과 후 · 직후 `git show --stat HEAD`).
> ★**SQL 없음.** `user_table` UPDATE 정책이 이미 account 스코프로 존재(핸드오프 확인). `member_token`은 컬럼 값 변경이라 기존 정책 통과 → 마이그레이션 불필요.

## 0. 한눈에

```
[트레이너] PT 회원 선택 → PT 뷰 상단 '회원앱 링크' 바
   │
   ├─ (토큰 없음)  [링크 생성] → crypto.randomUUID() → user_table.member_token UPDATE(.select() 하드닝)
   │                            → https://<origin>/m/<token> 클립보드 복사 → 토스트
   │
   ├─ (토큰 있음)  [링크 복사] → 기존 토큰으로 URL 복사(발급 없음)  +  [재발급] → 2단계 확인 → 새 토큰 덮어쓰기(이전 링크 무효)
   │
   └─ 배지: member_token만 있으면 '발급됨' / member_auth_id도 있으면 '연결됨'(회원이 1회+ 로그인)
[트레이너] 복사된 링크를 기존 카톡 흐름으로 회원에게 전송 → 회원이 열고 끝4 입력 → 회원 홈(S2)
```

**핵심 원리:** 발급 = `user_table.member_token`에 랜덤 uuid를 쓰는 **1회 UPDATE**뿐. 회원 로그인 라우트(`/api/member-auth`)가 이 토큰으로 회원을 찾으므로, **재발급 = 토큰 덮어쓰기 = 이전 링크로 재로그인 불가**(자동 폐기). 복사는 순수 클라(DB 안 씀). 클라 생성 uuid는 라우트의 `UUID_RE`(소문자 v4)와 정합 → 서버 수정 불필요.

## 1. 배치 결정 (착수 전 1개 · 내 권장대로 가도 됨)

**권장 = PT 뷰(`PTView.jsx`) 상단 바.** 이유:

- 회원앱의 실제 값(수업일지 `daily_workout_log` · 인바디 `inbody_log`)은 **등록(PT) 회원에게만 쌓인다.** OT 회원은 아직 계약·수업일지가 없어 링크를 줘도 빈 화면 → 발급은 PT 진입 후가 자연스러움.
- `PTView`는 **보관(inactive)·OT를 자동 제외**(그 뷰는 children/InactiveView로 감) → "누구에게 링크를 주나" 게이트가 배치만으로 충족.
- **삽입점 1곳**(PT 3서브탭 위)이라 회귀면 최소. 리스트 카드(중앙 UI)를 안 건드림.
- B2(프리미엄 게이트)가 어차피 이 발급을 premium 전용으로 잠글 예정 → PT 뷰 한 곳에 게이트 얹기 쉬움.

> 대안(원하면): **회원 리스트 카드**에 링크 액션(모든 상태 회원 대상). 단 카드가 `<button>`이라 중첩버튼 회피 위해 카드 구조를 손봐야 함(회귀면 넓음). MVP는 PT 뷰 권장 — OT 회원 조기 링크가 필요해지면 후속에 카드로 확장.

**이 스펙은 PT 뷰 배치 기준으로 작성.** 리스트 카드로 바꾸려면 말해줘(PART C 대체본 줄게).

## 2. PART A — 공용 클립보드 유틸 `lib/clipboard.js` (신규)

기존 복사 로직이 `VoiceLogTab`·`PtWorkoutTab`에 중복(`navigator.clipboard` → `execCommand` 폴백). 새 컴포넌트가 재사용하도록 **공용 1곳**으로 추출(기존 두 파일은 이번에 안 건드림 — 회귀 0, 후속 dedup은 선택).

```js
// lib/clipboard.js — 클립보드 복사(공용). navigator.clipboard 실패 시 execCommand 폴백.
// (PtWorkoutTab/VoiceLogTab의 copyToClipboard 패턴을 그대로 추출 · 반환 boolean = 복사 성공.)
export async function copyText(text) {
  try {
    await navigator.clipboard.writeText(text);
    return true;
  } catch {
    const ta = document.createElement("textarea");
    ta.value = text;
    ta.style.position = "fixed";
    ta.style.opacity = "0";
    document.body.appendChild(ta);
    ta.select();
    let ok = false;
    try {
      ok = document.execCommand("copy");
    } catch {
      ok = false;
    }
    document.body.removeChild(ta);
    return ok;
  }
}
```

## 3. PART B — `mapMemberRow`에 토큰 2필드 추가 (`app/page.jsx`) · 누락 배관

현재 `mapMemberRow`는 `member_token`·`member_auth_id`를 **안 실어준다**(`select("*")`로 DB엔 오지만 매핑에서 드롭). 링크 UI가 상태(발급됨/연결됨)를 알려면 두 필드를 매핑에 추가. **순수 가산.**

`app/page.jsx`의 `mapMemberRow(r)` 반환 객체에 아래 2줄 추가(예: `trainer_id` 줄 근처):

```js
    trainer_id: r.trainer_id ?? null,   // 내 회원 판별(원장 스코프)
    member_token: r.member_token ?? null,     // 회원앱 링크 토큰(S3 발급 UI)
    member_auth_id: r.member_auth_id ?? null, // 회원 auth 연결(있으면 '연결됨' = 1회+ 로그인)
```

> 트레이너의 `select("*")`는 자기 account 회원 행을 컬럼 제한 없이 읽으므로 두 값이 그대로 온다(회원측 안전뷰 제외와 무관 — 그건 회원 세션 얘기). `onMemberPatch`(이미 존재)가 발급 직후 로컬 낙관 갱신에 재사용됨(§5).

## 4. PART C — `components/views/MemberAppLink.jsx` (신규 · PT 뷰 상단 바)

발급·복사·재발급을 담은 자립 컴포넌트. 기존 디자인 시스템(`Button`·`Badge`·`Toast`·`useToast`) 재사용. `crypto.randomUUID()`로 토큰 생성(https·localhost=secure context라 사용 가능).

```jsx
// components/views/MemberAppLink.jsx — 회원앱 링크 발급·복사·재발급 (S3). PT 뷰 상단 바.
"use client";

import { useState } from "react";
import { Link2, Copy, RefreshCw, Check } from "lucide-react";
import { supabase } from "@/lib/supabaseClient";
import Button from "@/components/ui/Button";
import Badge from "@/components/ui/Badge";
import Toast from "@/components/ui/Toast";
import { useToast } from "@/hooks/useToast";
import { copyText } from "@/lib/clipboard";

export default function MemberAppLink({ member, onMemberPatch }) {
  const token = member?.member_token || null;
  const linked = Boolean(member?.member_auth_id); // 회원이 최소 1회 로그인함
  const [busy, setBusy] = useState(false);
  const [confirmReissue, setConfirmReissue] = useState(false);
  const { toast, showToast } = useToast();

  const linkFor = (t) =>
    (typeof window !== "undefined" ? window.location.origin : "") + "/m/" + t;

  // 발급/재발급 = user_table.member_token UPDATE(.select() 하드닝) → 링크 복사.
  // ★B2(프리미엄 게이트) 때 이 supabase.update 한 곳을 서버 RPC(issue_member_token, plan 확인)로 교체 = 유일 교체점.
  const issue = async (mode) => {
    if (busy) return;
    if (!supabase) {
      showToast("데모 모드 — 실제 발급 불가");
      return;
    }
    setBusy(true);
    const newToken = crypto.randomUUID();
    const { data, error } = await supabase
      .from("user_table")
      .update({ member_token: newToken })
      .eq("id", member.id)
      .select("id, member_token"); // 하드닝: 0행이면 조용한 실패(UPDATE 정책 부재 등)
    if (error || !data || data.length === 0) {
      setBusy(false);
      setConfirmReissue(false);
      showToast("발급 실패 — 정책/0행" + (error ? ": " + error.message : ""));
      return;
    }
    onMemberPatch?.(member.id, { member_token: newToken }); // 로컬 낙관 갱신(배지·버튼 즉시 반영)
    const ok = await copyText(linkFor(newToken));
    showToast(
      ok
        ? mode === "reissue"
          ? "재발급·복사됨 (이전 링크 무효)"
          : "링크 생성·복사됨"
        : "발급됨 (복사 실패 — 아래 링크 길게 눌러 복사)"
    );
    setBusy(false);
    setConfirmReissue(false);
  };

  // 기존 토큰 복사(발급 없이)
  const copyExisting = async () => {
    if (!token) return;
    const ok = await copyText(linkFor(token));
    showToast(ok ? "링크 복사됨" : "복사 실패 — 길게 눌러 복사");
  };

  return (
    <div className="rounded-xl border border-line bg-card px-3 py-2.5 shadow-sm">
      <div className="flex flex-wrap items-center gap-2">
        <Link2 className="h-4 w-4 shrink-0 text-primary-strong" />
        <span className="text-xs font-semibold text-ink">회원앱 링크</span>
        {token && (
          <Badge tone={linked ? "sky" : "primary"} className="ml-0.5">
            {linked ? "연결됨" : "발급됨"}
          </Badge>
        )}
        <div className="ml-auto flex items-center gap-1.5">
          {!token ? (
            <Button variant="primary" size="sm" onClick={() => issue("new")} disabled={busy}>
              <Link2 className="h-3.5 w-3.5" /> {busy ? "생성 중…" : "링크 생성"}
            </Button>
          ) : (
            <>
              <Button variant="solid" size="sm" onClick={copyExisting} disabled={busy}>
                <Copy className="h-3.5 w-3.5" /> 링크 복사
              </Button>
              {confirmReissue ? (
                <Button variant="danger" size="sm" onClick={() => issue("reissue")} disabled={busy}>
                  <Check className="h-3.5 w-3.5" /> {busy ? "재발급 중…" : "정말 재발급"}
                </Button>
              ) : (
                <Button variant="ghost" size="sm" onClick={() => setConfirmReissue(true)} disabled={busy}>
                  <RefreshCw className="h-3.5 w-3.5" /> 재발급
                </Button>
              )}
            </>
          )}
        </div>
      </div>
      {confirmReissue && (
        <p className="mt-1.5 text-[10px] leading-relaxed text-muted">
          재발급하면 이전 링크는 즉시 무효가 됩니다. (이미 로그인한 회원 세션은 리프레시 만료 전까지 유지 — 즉시 끊는 강제만료는 후속.)
        </p>
      )}
      <Toast message={toast} />
    </div>
  );
}
```

**동작 요약:**

| 상태 | 버튼 | 배지 |
|---|---|---|
| `member_token` 없음 | **링크 생성**(발급+복사) | — |
| `member_token` 있음 · `member_auth_id` 없음 | **링크 복사** + **재발급** | `발급됨`(primary) |
| `member_token`·`member_auth_id` 둘 다 | 동일 | `연결됨`(sky) |

- **재발급 = 2단계**(재발급 → "정말 재발급") — `window.confirm` 안 씀(모달·다이얼로그 회피 컨벤션). 오발 방지 + 경고 문구.
- **데모 모드**(`!supabase`) 가드 → 토스트 안내, DB 안 침범.

## 6. 검증 절차 (배포·수동)

1. **PART A~D 배포**(Vercel Ready) 후, **PT 회원**(계약 있는 등록 회원) 선택 → PT 뷰 상단에 '회원앱 링크' 바 노출 확인. (OT/보관 회원 뷰엔 안 뜸 = 의도.)
2. **최초 발급:** 토큰 없는 회원 → [링크 생성] → 토스트 "링크 생성·복사됨" + 배지 '발급됨'. Supabase에서 `select member_token from user_table where id='<회원>'` → uuid 채워짐 확인.
3. **복사·전송:** 붙여넣기(카톡/메모)로 `https://pt-navigator.vercel.app/m/<token>` 확인 → 폰에서 열기 → 휴대 끝4 입력 → **회원 홈(S2) 렌더**. 이후 그 회원 바 배지가 '연결됨'으로(재조회/새 회원선택 시 `member_auth_id` 반영).
4. **재발급:** [재발급] → [정말 재발급] → 새 토큰. **이전 링크로 재로그인 시도 → 401**(FAIL) 확인. 새 링크는 정상.
5. **하드닝(조용한 실패) 감지:** 만약 UPDATE가 0행이면 토스트 "발급 실패 — 정책/0행" 떠야 함(정상 발급이면 안 뜸). 이게 뜨면 `user_table` UPDATE 정책 점검(아래 §7).
6. **데모 모드:** 키 없이 열면 [링크 생성] → "데모 모드" 토스트, DB 미침범.
7. **회귀:** PT 3서브탭(운동일지 저장·차감·카톡복사 / 재등록 / 인바디) 정상. OT 6탭·리스트·오늘/설정/내실적 무영향(가산 컴포넌트 1개 + 매핑 2필드 + PTView 1줄).

**(선택) 정책 리트머스 — SQL 아님, 읽기 확인만.** UPDATE 정책 존재 재확인(발급이 잘 되면 불필요):

```sql
select policyname, cmd, qual, with_check from pg_policies
  where tablename='user_table' and cmd in ('UPDATE','ALL') order by policyname;
-- 기대: account 스코프 update/all 정책 존재(account_id = auth_account_id()). member_token 변경은 account_id 불변이라 with_check 통과.
```

## 8. 이후 (범위 밖 · 연결점)

- **B2 — 프리미엄 게이트:** MemberAppLink의 `issue()`가 유일 발급점 → B2에서 이 UPDATE를 `security definer` RPC(`issue_member_token(member_id)`, `plan='premium'` 아니면 거절)로 교체 + 일반구독이면 바 자체를 🔒 표시. `/api/member-auth`에도 소속계정 plan 확인 추가(해지 시 나간 링크도 그날부터 차단). → **B1(구독 스키마) 선행 필요.**
- **즉시 강제만료(선택):** 재발급 시 이미 로그인한 회원 세션까지 끊으려면 회원 auth 유저 비번 회전/세션 revoke 추가(현재는 리프레시 만료까지 유지 = MVP 허용).
- **토큰 강화(선택):** uuid로 충분(122bit 랜덤 + 끝4 2차 + 라우트 스로틀). 더 길게는 후속.
