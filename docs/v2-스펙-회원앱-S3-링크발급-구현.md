# v2 스펙 — 회원 앱 S3 (트레이너 링크 발급 + PT 종료) · 구현

> 성격: **구현 스펙**(클로드코드 실행용). 전제 = S1(인증·RLS)·S2(회원 홈·안전뷰) 완료·배포·검증. 설계 근거 = `docs/v2-설계-회원앱-MVP-아키텍처.md` §4.
> 스코프: 트레이너가 회원의 **회원앱 링크(`/m/<token>`)를 생성·복사**하고, 필요 시 **'PT 종료'로 접근을 완전 차단**(세션 삭제 + 토큰 폐기)하는 UI. 이거 하면 손으로 SQL 안 만지고 회원앱 v1(인증·열람·발급·종료) **완결**.
> 흐름: 이 스펙 → 트레이너 → 클로드코드(diff) → 트레이너(git diff) → 웹Claude(검토) → 폰 검증. 커밋은 트레이너 직접(파일지정 `git add` · `npm.cmd run lint` 통과 후 · 직후 `git show --stat HEAD`).
> ★**SQL 없음.** `user_table` UPDATE 정책은 이미 account 스코프로 존재(발급용). PT 종료는 회원 auth 유저 삭제(service_role)인데, S1에서 `member_auth_id`를 `references auth.users(id) on delete set null`로 만들어놔서 삭제 시 링크가 자동 해제 → 마이그레이션 불필요.

## 0. 한눈에

```
[트레이너] PT 회원 선택 → PT 뷰 상단 '회원앱 링크' 바
   │
   ├─ (토큰 없음)  [링크 생성] → crypto.randomUUID() → user_table.member_token UPDATE(.select() 하드닝)
   │                            → https://<origin>/m/<token> 클립보드 복사 → 토스트   (클라 직접)
   │
   ├─ (토큰 있음)  [링크 복사] → 기존 토큰으로 URL 복사(발급 없음)                       (클라 직접)
   │              +  [PT 종료] → 2단계 확인 → POST /api/member-revoke                    (서버 · 완전 차단)
   │                            → 회원 auth 유저 삭제(세션 무효) + member_token/auth_id=null
   │
   └─ 배지: member_token만 있으면 '발급됨' / member_auth_id도 있으면 '연결됨'(회원이 1회+ 로그인)
[트레이너] 복사된 링크를 기존 카톡 흐름으로 회원에게 전송 → 회원이 열고 끝4 입력 → 회원 홈(S2)
```

**발급 원리:** `user_table.member_token`에 랜덤 uuid를 쓰는 **1회 UPDATE**뿐(클라 직접). 로그인 라우트(`/api/member-auth`)가 이 토큰으로 회원을 찾아 세션 발급. 복사는 순수 클라. 클라 생성 uuid는 라우트의 `UUID_RE`(소문자 v4)와 정합 → 로그인 서버 수정 없음.

**PT 종료(완전 차단) 원리:** `member_token=null`만으론 **재로그인만** 막고 이미 로그인한 세션은 안 끊긴다(토큰은 로그인 때만 쓰이고, 이후 열람은 세션 JWT + `member_auth_id`로 감). 그래서 종료는 **서버 라우트가 회원 auth 유저 자체를 삭제** → ① 재로그인 불가 ② 기존 세션은 다음 요청부터 `auth_member_id()`가 매핑을 못 찾아 **0행**(사실상 즉시 차단) ③ 합성 이메일이 풀려 나중에 **재개통도 깔끔**. (access token JWT는 무상태라 최대 ~1시간 잔존 가능 — 어떤 방식이든 동일 한계, 실질 차단엔 충분.)

**★데이터는 안 죽는다:** 삭제되는 건 **로그인 열쇠(auth 유저) 1개**뿐. 회원 기록(수업일지·인바디·계약·회원정보)은 전부 **`user_table.id`**에 매달려 있어 무관. 재등록/재개통하면 새 열쇠를 다시 끼워 **옛 기록까지 그대로 열림**.

> ★유출 안전: 링크가 새어도 그 회원 **휴대폰 끝4**를 알아야 열림(라우트 2차 확인). PT 종료는 유출·명시적 "이 회원 끊자" 용도의 킬스위치.
> ★자동 종료는 넣지 않음(의도): 지금 회원앱은 **읽기 전용이라 열람 비용 0** + 남은 0은 보통 재등록 직전이라 열어두는 게 유리. 비용이 드는 **AI 기능은 후속에 "활성 PT(남은>0)일 때만" 게이트**로 잠근다(앱은 열되 돈 드는 것만 잠금). → §10.

## 2. PART A — 공용 클립보드 유틸 `lib/clipboard.js` (신규)

기존 복사 로직이 `VoiceLogTab`·`PtWorkoutTab`에 중복. 새 컴포넌트가 재사용하도록 공용 1곳으로 추출(기존 두 파일은 안 건드림 — 회귀 0).

```js
// lib/clipboard.js — 클립보드 복사(공용). navigator.clipboard 실패 시 execCommand 폴백.
export async function copyText(text) {
  try {
    await navigator.clipboard.writeText(text);
    return true;
  } catch {
    const ta = document.createElement("textarea");
    ta.value = text;
    ta.style.position = "fixed";
    ta.style.opacity = "0";
    document.body.appendChild(ta);
    ta.select();
    let ok = false;
    try {
      ok = document.execCommand("copy");
    } catch {
      ok = false;
    }
    document.body.removeChild(ta);
    return ok;
  }
}
```

## 4. PART C — `components/views/MemberAppLink.jsx` (신규 · PT 뷰 상단 바)

발급·복사는 **클라 직접**(supabase UPDATE), PT 종료는 **서버 라우트 호출**(세션 삭제). 기존 디자인 시스템(`Button`·`Badge`·`Toast`·`useToast`) + `authHeader`(Bearer 토큰) 재사용.

```jsx
// components/views/MemberAppLink.jsx — 회원앱 링크 발급·복사·PT 종료 (S3). PT 뷰 상단 바.
"use client";

import { useState } from "react";
import { Link2, Copy, Ban, Check } from "lucide-react";
import { supabase } from "@/lib/supabaseClient";
import { authHeader } from "@/lib/authHeader";
import Button from "@/components/ui/Button";
import Badge from "@/components/ui/Badge";
import Toast from "@/components/ui/Toast";
import { useToast } from "@/hooks/useToast";
import { copyText } from "@/lib/clipboard";

export default function MemberAppLink({ member, onMemberPatch }) {
  const token = member?.member_token || null;
  const linked = Boolean(member?.member_auth_id); // 회원이 최소 1회 로그인함
  const [busy, setBusy] = useState(false);
  const [confirmEnd, setConfirmEnd] = useState(false);
  const { toast, showToast } = useToast();

  const linkFor = (t) =>
    (typeof window !== "undefined" ? window.location.origin : "") + "/m/" + t;

  // 링크 생성 = user_table.member_token에 새 토큰 UPDATE(.select() 하드닝) → 복사. (클라 직접)
  // ★B2(프리미엄 게이트) 때 이 supabase.update 한 곳을 서버 RPC(issue_member_token, plan 확인)로 교체 = 유일 교체점.
  const issue = async () => {
    if (busy) return;
    if (!supabase) {
      showToast("데모 모드 — 실제 발급 불가");
      return;
    }
    setBusy(true);
    const newToken = crypto.randomUUID();
    const { data, error } = await supabase
      .from("user_table")
      .update({ member_token: newToken })
      .eq("id", member.id)
      .select("id, member_token"); // 하드닝: 0행이면 조용한 실패(UPDATE 정책 부재 등)
    if (error || !data || data.length === 0) {
      setBusy(false);
      showToast("발급 실패 — 정책/0행" + (error ? ": " + error.message : ""));
      return;
    }
    onMemberPatch?.(member.id, { member_token: newToken }); // 로컬 낙관 갱신(배지·버튼 즉시 반영)
    const ok = await copyText(linkFor(newToken));
    showToast(ok ? "링크 생성·복사됨" : "발급됨 (복사 실패 — 링크 복사 다시 눌러줘)");
    setBusy(false);
  };

  // 기존 토큰 복사(발급 없이)
  const copyExisting = async () => {
    if (!token) return;
    const ok = await copyText(linkFor(token));
    showToast(ok ? "링크 복사됨" : "복사 실패 — 길게 눌러 복사");
  };

  // PT 종료 = 서버가 회원 auth 유저 삭제(세션 무효) + member_token/auth_id=null. (완전 차단)
  const endPt = async () => {
    if (busy) return;
    if (!supabase) {
      showToast("데모 모드 — 실제 종료 불가");
      return;
    }
    setBusy(true);
    try {
      const res = await fetch("/api/member-revoke", {
        method: "POST",
        headers: { "Content-Type": "application/json", ...(await authHeader()) },
        body: JSON.stringify({ memberId: member.id }),
      });
      const json = await res.json().catch(() => ({}));
      if (!res.ok || !json.ok) {
        setBusy(false);
        setConfirmEnd(false);
        showToast(json.error || "PT 종료 실패");
        return;
      }
      onMemberPatch?.(member.id, { member_token: null, member_auth_id: null }); // 배지 사라짐·[링크 생성]으로 복귀
      showToast("PT 종료됨 · 회원앱 접근 차단");
    } catch {
      showToast("네트워크 오류 · 다시 시도");
    }
    setBusy(false);
    setConfirmEnd(false);
  };

  return (
    <div className="rounded-xl border border-line bg-card px-3 py-2.5 shadow-sm">
      <div className="flex flex-wrap items-center gap-2">
        <Link2 className="h-4 w-4 shrink-0 text-primary-strong" />
        <span className="text-xs font-semibold text-ink">회원앱 링크</span>
        {token && (
          <Badge tone={linked ? "sky" : "primary"} className="ml-0.5">
            {linked ? "연결됨" : "발급됨"}
          </Badge>
        )}
        <div className="ml-auto flex items-center gap-1.5">
          {!token ? (
            <Button variant="primary" size="sm" onClick={issue} disabled={busy}>
              <Link2 className="h-3.5 w-3.5" /> {busy ? "생성 중…" : "링크 생성"}
            </Button>
          ) : (
            <>
              <Button variant="solid" size="sm" onClick={copyExisting} disabled={busy}>
                <Copy className="h-3.5 w-3.5" /> 링크 복사
              </Button>
              {confirmEnd ? (
                <Button variant="danger" size="sm" onClick={endPt} disabled={busy}>
                  <Check className="h-3.5 w-3.5" /> {busy ? "종료 중…" : "정말 종료"}
                </Button>
              ) : (
                <Button variant="danger" subtle size="sm" onClick={() => setConfirmEnd(true)} disabled={busy}>
                  <Ban className="h-3.5 w-3.5" /> PT 종료
                </Button>
              )}
            </>
          )}
        </div>
      </div>
      {confirmEnd && (
        <p className="mt-1.5 text-[10px] leading-relaxed text-muted">
          PT를 종료하면 이 회원의 앱 접근이 차단됩니다(로그인 세션 무효 + 링크 폐기). 회원 기록(수업일지·인바디)은 그대로 보존돼요. 다시 열려면 [링크 생성]으로 새 링크를 발급하면 됩니다.
        </p>
      )}
      <Toast message={toast} />
    </div>
  );
}
```

**동작 요약:**

| 상태 | 버튼 | 배지 |
|---|---|---|
| `member_token` 없음 | **링크 생성**(발급+복사 · 클라) | — |
| `member_token` 있음 · `member_auth_id` 없음 | **링크 복사** + **PT 종료** | `발급됨`(primary) |
| `member_token`·`member_auth_id` 둘 다 | 동일 | `연결됨`(sky) |

- **PT 종료 = 2단계**(PT 종료 → "정말 종료") — `window.confirm` 안 씀(다이얼로그 회피 컨벤션). 서버 라우트가 세션까지 삭제 → 종료 후 배지 사라지고 버튼이 [링크 생성]으로 복귀.
- **'PT 종료'는 회원앱 접근만 차단** — 회원 상태(`pt_active`)·계약·매출·환불(RefundMember) 무변경. 기록도 보존.
- **데모 모드**(`!supabase`) 가드 → 토스트 안내, DB 안 침범.

## 6. PART E — `app/api/member-revoke/route.js` (신규 · service_role 서버전용)

트레이너가 자기 account 회원의 회원앱 접근을 완전 차단. `create-trainer` 패턴(service 키 + `getUser`로 호출자 검증) 재사용. **인증 필수**(트레이너 JWT) + **account 소속 확인**(타 계정 회원 revoke 차단).

```js
// app/api/member-revoke/route.js — 회원앱 접근 완전 차단('PT 종료'). 서버전용.
//   ① 호출 트레이너 인증(JWT)+활성 확인 ② 대상 회원이 내 account인지 확인
//   ③ 회원 auth 유저 삭제(세션 무효) → FK(on delete set null)로 member_auth_id 자동 해제
//   ④ member_token=null(재로그인 차단). 회원 데이터(user_table/일지/인바디/계약)는 안 건드림.
import { createClient } from "@supabase/supabase-js";

export const runtime = "nodejs";

const UUID_RE = /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i;

export async function POST(req) {
  const url = process.env.NEXT_PUBLIC_SUPABASE_URL;
  const key = process.env.SUPABASE_SERVICE_ROLE_KEY;
  if (!url || !key) return Response.json({ error: "서버 키 미설정" }, { status: 503 });

  const authz = req.headers.get("authorization") || "";
  const token = authz.startsWith("Bearer ") ? authz.slice(7) : null;
  if (!token) return Response.json({ error: "인증 필요" }, { status: 401 });

  const body = await req.json().catch(() => ({}));
  const memberId = String(body.memberId || "").trim();
  if (!UUID_RE.test(memberId)) return Response.json({ error: "잘못된 요청" }, { status: 400 });

  const sb = createClient(url, key, { auth: { autoRefreshToken: false, persistSession: false } });

  // ① 호출자 = 활성 트레이너 검증(service 키라 RLS 우회하여 조회)
  const { data: u, error: ue } = await sb.auth.getUser(token);
  if (ue || !u?.user?.id) return Response.json({ error: "세션 무효" }, { status: 401 });
  const { data: me } = await sb.from("trainer").select("account_id, active").eq("id", u.user.id).maybeSingle();
  if (!me || me.active !== true) {
    return Response.json({ error: "권한 없음(비활성/미등록 트레이너)" }, { status: 403 });
  }

  // ② 대상 회원이 내 account 소속인지 확인(타 계정 회원 revoke 차단)
  const { data: member } = await sb
    .from("user_table")
    .select("id, account_id, member_auth_id")
    .eq("id", memberId)
    .maybeSingle();
  if (!member || member.account_id !== me.account_id) {
    return Response.json({ error: "대상 회원을 찾을 수 없습니다." }, { status: 404 });
  }

  // ③ 회원 auth 유저 삭제 → 세션 무효화 + FK로 member_auth_id 자동 null. (없으면 스킵.)
  if (member.member_auth_id) {
    const { error: de } = await sb.auth.admin.deleteUser(member.member_auth_id);
    if (de && !/not.*found/i.test(de.message || "")) {
      return Response.json({ error: "세션 차단 실패: " + de.message }, { status: 500 });
    }
  }

  // ④ 토큰 폐기 + 링크 상태 초기화(멱등 · FK로 이미 null이어도 명시).
  const { data: upd, error: upe } = await sb
    .from("user_table")
    .update({ member_token: null, member_auth_id: null })
    .eq("id", memberId)
    .select("id");
  if (upe || !upd || upd.length === 0) {
    return Response.json({ error: "링크 폐기 실패: " + (upe?.message || "0행") }, { status: 500 });
  }

  return Response.json({ ok: true });
}
```

> **왜 삭제(deleteUser)인가:** `member_token=null`만으론 살아있는 세션이 안 끊긴다(토큰은 로그인 때만 소비). auth 유저를 지우면 세션이 무효화되고, `auth_member_id()`(= `member_auth_id = auth.uid()` 조회)가 매핑을 못 찾아 **회원 쿼리가 즉시 0행**. FK가 `on delete set null`이라 `member_auth_id`는 자동으로 풀리지만, ④에서 `member_token`과 함께 명시적으로 null 처리(멱등·상태 정리). 합성 이메일이 풀려 재개통 시 `member-auth`가 `createUser`로 깔끔히 재생성.

## 7. 검증 절차 (배포·수동)

1. **PART A~E 배포**(Vercel Ready) 후, **PT 회원** 선택 → PT 뷰 상단 '회원앱 링크' 바 노출. (OT/보관 회원 뷰엔 안 뜸 = 의도.)
2. **최초 발급:** 토큰 없는 회원 → [링크 생성] → 토스트 "링크 생성·복사됨" + 배지 '발급됨'. Supabase `select member_token from user_table where id='<회원>'` → uuid 확인.
3. **복사·로그인:** 복사된 `https://pt-navigator.vercel.app/m/<token>` 폰에서 열기 → 끝4 입력 → **회원 홈(S2) 렌더**. 그 회원 배지가 '연결됨'으로(재조회 시 `member_auth_id` 반영).
4. **PT 종료(완전 차단):** [PT 종료] → [정말 종료] →
   - `member-revoke` 200/ok, 토스트 "PT 종료됨". 배지 사라지고 버튼이 [링크 생성]으로 복귀.
   - Supabase: 그 회원 `member_token`·`member_auth_id` 둘 다 null. Authentication에서 `m-<회원id>@member...` 유저 삭제됨.
   - **로그인해 있던 폰에서 새로고침** → 회원 홈이 빈 상태/"정보를 불러오지 못했어요"로 떨어지고(세션 매핑 끊김), 이전 링크로 재로그인 시도 → **401**.
5. **재개통(기록 보존 확인):** 같은 회원 → [링크 생성] → 새 링크로 끝4 재로그인 → **예전 수업일지·인바디가 그대로** 다시 보임(데이터는 user_table.id에 붙어 무손실).
6. **권한 방어:** (선택) 다른 account 회원 id로 `member-revoke` 직접 호출 → 404. 비로그인/토큰 없음 → 401.
7. **데모 모드:** 키 없이 [PT 종료] → "데모 모드" 토스트, DB 미침범.
8. **회귀:** PT 3서브탭(운동일지 저장·차감·카톡복사 / 재등록 / 인바디) 정상. OT 6탭·리스트·오늘/설정/내실적 무영향.

## 9. 보안·엣지 메모

- **완전 차단의 잔여 틈:** 종료 순간 이미 발급된 access token JWT는 무상태라 만료(≤~1시간)까지 형식상 유효. 단 `auth_member_id()`가 매핑을 못 찾아 **읽어도 0행** → 실질 즉시 차단. 재로그인은 즉시 불가.
- **재개통 시 orphan 없음:** auth 유저를 실제 삭제하므로 합성 이메일이 풀림 → 재발급 로그인에서 `createUser` 정상(이메일 충돌 없음). (token만 null로 두고 유저를 남기면 재개통이 이메일 충돌로 실패 — 그래서 '삭제'가 정답.)
- **revoke 라우트 인증:** 트레이너 JWT + `trainer.active` + 회원 account 일치까지 3중. service 키는 서버 전용(클라/깃 노출 금지 · create-trainer와 동일).
- **teardown 연동:** 테스트 회원 정리 시 이 종료 버튼으로 auth 유저 삭제까지 한 번에 됨(미결 항목 일부 자동 해소).
