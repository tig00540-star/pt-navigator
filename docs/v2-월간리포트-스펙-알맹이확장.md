## 0. 원칙 (불변 — 파스 이유였던 "얇음"을 채우되 결은 유지)

1. **추가 fetch 0.** MyStats가 이미 로드해 props로 넘긴 배열(`contracts` = session_log · `logs` = daily_workout_log · `otRows` = ot_log · `schemes` · `runs` · `uid` · `memberIds` · `members` · `contractNames`)만으로 전부 재집계. **prev 월(전월대비)도 같은 배열에 이미 들어있음**(session_log은 RLS로 본인 계약 전체 기간, 월 필터는 fetch가 아니라 순수함수의 `ym` 인자) → 새 쿼리 불필요.
2. **순수함수 재집계.** 모든 지표는 `lib/memberStatus.js`의 기존 순수함수에 `ym` 주입. 신규 집계 로직을 뷰에 인라인하지 말 것(단일 소재지 유지). 전월값은 같은 함수에 `pym`(전월 ym) 주입.
3. **B 부품 상속.** 새 섹션도 `Card`·`SectionHeader`·`StatTile`·`Chip`·`Badge`·`EmptyState`로만. 새 인라인 카드/배지 만들지 말 것.
4. **인쇄 평면 유지.** 리포트 본문(`.report-sheet`)은 4-b(PDF/이미지)를 위해 **`<details>` 접힘 금지 · 전부 인라인 전개**. MyStats의 drill-down은 `<details>`지만 리포트는 평면(현재도 그러함) — 이 원칙 유지.
5. **월 스코프 한계 명시.** 클로징·재등록 퍼널은 `ot_log`/`reg_result`에 **처리월(날짜)이 없어 월 스코프 불가 → 전체 기간 누적**(§5 인계 확인). 해당 섹션 헤더/힌트에 "누적" 라벨 필수(원장이 월 지표로 오독 방지).

### 2-2. 모듈 스코프 헬퍼 2개 (컴포넌트 밖, `lastMonths` 아래)

```jsx
// 전월 ym('YYYY-MM' → 한 달 전 'YYYY-MM'). 연도 경계·1월 안전(UTC 순수 계산, now 안 읽음).
function prevYm(ym) {
  const [y, m] = ym.split("-").map(Number);
  const d = new Date(Date.UTC(y, m - 2, 1)); // m은 1-based → m-2 = 전월의 0-based month index
  return `${d.getUTCFullYear()}-${String(d.getUTCMonth() + 1).padStart(2, "0")}`;
}

// 전월대비 증감 pill. up=긍정지표(매출·수업·급여는 오를수록 좋음).
// prev===0 처리: 신규 발생(cur>0)이면 "신규", 둘 다 0이면 렌더 안 함(null).
function Delta({ cur, prev }) {
  if (!prev && !cur) return null;
  if (!prev) return <span className="text-[11px] font-semibold text-emerald-600">신규</span>;
  const pct = Math.round(((cur - prev) / prev) * 100);
  if (pct === 0) return <span className="text-[11px] text-muted">± 0%</span>;
  const up = pct > 0;
  const Icon = up ? TrendingUp : TrendingDown;
  const cls = up ? "text-emerald-600" : "text-rose-600";
  return (
    <span className={`inline-flex items-center gap-0.5 text-[11px] font-semibold tabular-nums ${cls}`}>
      <Icon className="h-3 w-3" /> {Math.abs(pct)}%
    </span>
  );
}
```

> 방향 규칙: 이 3개 지표는 전부 "오를수록 좋음"이라 up=emerald 고정. (환불·이탈처럼 반대 지표는 이번 스코프 없음 — 나중에 필요하면 `good` prop 추가.)

### 2-5. A 엣지
- 이번달이 가장 오래된 조회월(months[11])이라도 `prevYm`은 그 전월 문자열을 만들고, 데이터가 없으면 prev=0 → "신규" 또는 null. 안전.
- prev=0·cur=0(신규 입사월 등): Delta null → 줄 자체는 렌더되지만 "전월대비 " 라벨만 남아 어색 → **라벨도 델타 있을 때만**. 각 삽입부를 `{Delta(...) && ...}`로 감싸기 어렵다면, Delta가 null 반환할 때 부모 줄을 숨기려면 아래처럼 값 먼저 계산:
  ```jsx
  {revP.total > 0 && <div className="mt-1.5 …">전월대비 <Delta cur={rev.total} prev={revP.total} /></div>}
  ```
  → **권장: `prev > 0` 가드로 부모 줄 자체를 조건부** (신규월엔 "전월대비" 라벨 안 뜸). (a)(b)(c) 전부 이 패턴.

## 5. 서브커밋 D — (선택) 목표 근접 + 트레이너 실명

### 5-1. 다음 급여 구간까지 (banded 스킴 한정)
급여 Card 안, 예상 branch에 "다음 구간까지" 진행 힌트. 명시적 '목표'가 DB에 없으니 **급여 밴드의 다음 `min`을 목표 프록시**로. banded일 때만.
```jsx
// scheme.bands 정렬 후 현재 basis(band_basis)로 다음 밴드까지 남은 양.
const bands = [...(scheme?.bands || [])].sort((a, b) => (a.min ?? 0) - (b.min ?? 0));
const basisVal = scheme?.band_basis === "session_count" ? sessionCount : rev.total;
const nextBand = bands.find((b) => (b.min ?? 0) > basisVal);
const toNext = nextBand ? (nextBand.min ?? 0) - basisVal : 0;
```
표시(예상 branch 내, 밴드 있을 때):
```jsx
{nextBand && (
  <div className="mt-1 text-[11px] text-muted">
    다음 급여 구간까지 {scheme.band_basis === "session_count" ? `${toNext}회` : won(toNext)}
  </div>
)}
```
> `payForScheme`가 이미 밴드를 해석하지만 "다음 구간" 정보는 안 돌려줌 → 뷰에서 계산(읽기 전용, 순수). 원장 리포트라 밴드 노출 OK. **manual 스킴이면 `bands`=빈 배열 → nextBand=undefined → 자동 숨김.**

### 5-2. 트레이너 실명 (헤더 — 원장 문서 신뢰도)
현재 헤더는 `personName(trainerName)` = 이메일 `tig00540@...` → `tig00540`(원장 문서엔 약함). 인계 §1: `trainer.name`(P0-1에서 '황대수' 실명 시드)로 교체 = **원장에 보이기 전 우선**.
- **접근:** MonthlyReport props에 실명이 없음(현재 `trainerName: email`). MyStats가 자기 `user_table` 행 이름을 1회 조회해 넘기는 게 최소.
- MyStats.jsx: 기존 `au` 획득 직후, `uid`로 본인 이름 조회(이미 `user_table` in-쿼리 있으니 그 결과에 uid 포함시키거나 별도 1행):
