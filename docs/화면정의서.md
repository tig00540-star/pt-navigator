## 0. 이 문서를 읽는 법

### 0.1 범위

배포된 코드에 **실제로 존재하는 화면만** 적었습니다. 설계 중이거나 문서에만 있는 화면은 넣지 않았고,
화면 안에서 데모·하드코딩으로 동작하는 부분은 해당 항목에 `데모` 표시를 붙였습니다.

`docs/MASTERPLAN.md`의 화면 지도(§3)는 v1 스냅샷이라 현재와 다릅니다(5탭 → 10탭, 회원앱·관리자 구조 변경).
충돌 시 **이 문서가 최신**입니다.

### 0.2 화면 ID 규칙

| 접두 | 영역 |
|---|---|
| `T-` | 트레이너 앱 (`/`) |
| `M-` | 회원앱 (`/m/[token]`) |
| `A-` | 관리자 (`/admin`) |
| `G-` | 공통·인증 (게이트, 에러, 가입) |
| `X-` | 모달 · 오버레이 |

### 0.3 표기

- **읽기/쓰기** — 해당 화면이 접근하는 Supabase 테이블. `(R)` 읽기, `(W)` 쓰기, `(D)` 삭제
- `데모` — 하드코딩·가짜 데이터로 동작. 실데이터 연결 안 됨
- `게이트` — 구독·권한 조건이 걸린 요소

### 1.2 권한별 진입 지도

```
비로그인 ─── G-01 로그인 ─── G-02 계정 생성
                │
        [구독 활성 확인]
                │
        ├─ 미활성/만료 → G-04 결제벽 (앱 진입 차단)
        └─ 활성
             │
             ├─ 트레이너/원장 → 트레이너 앱 (/)
             │      └─ 원장이면 헤더에 [관리자] → /admin
             │
             └─ 회원 → /m/[token] (별도 인증, 게이트 우회)
```

**중요:** 회원앱 경로 `/m`은 전역 인증 게이트를 **통과하지 않습니다.** 회원은 트레이너 계정이 없고
토큰 + 휴대폰 뒤 4자리로 자체 인증합니다.

## 2. 공통 요소

### 2.1 트레이너 앱 헤더 (전 화면 고정)

| 위치 | 요소 | 조건 | 동작 |
|---|---|---|---|
| 좌 | 앱 아이콘 + 워드마크 + 트레이너 이름 | 항상 | — |
| 우 | 회원 선택 셀렉트 | 회원 1명 이상 | 선택 시 해당 회원 워크플로우로 전환 |
| 우 | 신규 등록 버튼 | 항상 | → X-01 회원 등록 모달 |
| 우 | 공지 벨 + 안읽음 배지 | 항상 | → X-07 공지 재열람 |
| 우 | 관리자 링크 | 센터 계정 + 원장 | → `/admin` |
| 2단 | 워크플로우 서브탭 바 | 현재 탭이 OT/PT 그룹일 때만 | 그룹 내 탭 전환 |
| 2단 | 설정 서브탭 바 | 설정 탭일 때만 | 설정 5개 서브탭 전환 |
| 본문 상단 | 정보 수정 버튼 | 회원 워크플로우 탭 | → X-02 회원 수정 모달 |

### 2.2 하단 탭바 (BottomNav)

**항상 고정 4개** — 오늘 · 회원 · 내 실적 · 설정.
워크플로우 탭(OT 3개 / PT 3개)은 하단에 나오지 않고 **상단 서브탭 바에만** 나옵니다.

### 2.3 워크플로우 탭 노출 규칙

선택된 회원의 상태에 따라 상단 서브탭 바의 내용이 갈립니다.

| 회원 상태 | 노출 탭 | 그룹 색 |
|---|---|---|
| OT 진행 중 (기본값) | 1차 OT 준비 · 1차 피드백 · 2차 OT 준비 | amber |
| PT 등록 완료 (`pt_active`) | 회원자료 · 자료남기기 · 재등록 준비 | sky |
| 종결 (`inactive`) | 없음 (T-09로 대체) | — |

**교차 전환 보정** — PT 회원인데 OT 탭에 있으면 회원자료로, OT 회원인데 PT 탭에 있으면 1차 OT 준비로
강제 이동합니다. 회원을 바꿔도 화면이 깨지지 않게 하는 안전장치입니다.

### 2.4 상태 표시 공통 규칙

| 상태 | 화면 |
|---|---|
| 로딩 | "불러오는 중…" 전체화면 또는 섹션 단위 스피너 |
| 데이터 없음 | 빈 상태 문구 + 다음 행동 버튼 (예: "첫 회원 등록") |
| 저장 실패 | 토스트. 쓰기 실패는 **조용히 넘어가지 않고 반드시 표시** |
| AI 키 미설정 | 데모 리포트 + "데모" 라벨 명시 |

| 목적 | 1차 OT 입장 직전 3분 사전무장 |

**구성요소**

| 영역 | 내용 |
|---|---|
| 회원 기본정보 카드 | 등록 시 받은 문진 요약 |
| AI 사전무장 블록 | 대화 방향 · 클로징 준비 · 거절 5종 선제 방어 |
| 현장 메모 | 수업 중 즉시 기입용 자유 입력 |

**액션** — AI 브리핑 생성 / 재생성 · 메모 작성

**AI 사양**

| 항목 | 내용 |
|---|---|
| 입력 근거 | 회원 기본정보 + OT 사전 문진 9필드 + 트레이너의 활성 패키지 목록 |
| 출력 | 회원 해석 · 오프닝 대사 · 수업 계획 · 타깃 종목 2개 · 세일즈 비유 · 클로징 대사 · **거절 5종 방어**(가격/생각해볼게요/효과의심/시간부족/타센터) · 추천 프로그램 · 부족한 정보 |
| 캐시 | **캐시됩니다.** 1회차 행에 저장하고 회원 정보가 바뀌면 스테일 감지 |
| 실패 시 | **데모 폴백 없이 블록 자체를 숨깁니다** |

**가격 환각 방지** — AI는 패키지 **번호만** 반환하고, 실제 금액은 앱이 가격표에서 조회해 표시합니다.
AI가 가격을 지어내지 못하게 하는 구조입니다.

**액션** — 브리핑 생성 / 재생성 · 클로징 결과 기록

**AI 사양** — **캐시됩니다.** 결과를 `ot_log` 2회차 행에 저장하고, 1차 관찰이 바뀌면 스테일 배지를

**읽기/쓰기** — `ot_log`(RW) `pt_package`(R)

**주의** — 같은 행에 **브리핑 캐시와 클로징 결과가 공존**합니다. 각 writer는 자기 필드만 갱신해야 하며,
