## 0. 정의 (트레이너 확정)

- **총수업수 = 내가 해야 할 총** = **내 활성 PT 회원(status `pt_active`) 전원**의 **모든 계약(session_log) 세션 합**. 계약당 = `sessions_total + service_sessions`(**서비스 세션 포함** — 실제로 가르치는 수업). 신규+재등록+인계 이월 계약 전부 포함.
- **잔여수업수 = 총 − 완료** = 그 회원들 각 계약 잔여의 합(`remainingSessions().total` 합).
- **완료 = 진행·노쇼로 차감된 수업**(비-voided `daily_workout_log`). = 총 − 잔여.
- **제외:** `inactive`(PT 끝난) · `ot`(아직 PT 아님) 회원. 남의 회원(원장이라도 `trainer_id ≠ 나`)도 제외 — 급여/실적 스코프와 동일.

> 데이터는 **MyStats가 이미 로드** 중: `contracts`(session_log 전건·RLS 본인), `logs`(daily_workout_log 전건), `members`(목록), `uid`. **추가 쿼리 0.**

### (b) 파생 추가 — 기존 파생부 끝(현재 `refundRows` L97 근처) 뒤에

```jsx
// #1 — 내 활성 PT 회원(pt_active) 전체의 총/잔여 수업 합. inactive·OT·남의 회원 제외.
const ptMemberIds = new Set(
  members.filter((m) => m.trainer_id === uid && viewFor(m) === "pt").map((m) => m.id)
);
const ptContracts = contracts.filter((c) => ptMemberIds.has(c.user_id));
const sessTotalAll = ptContracts.reduce(
  (s, c) => s + (c.sessions_total ?? 0) + (c.service_sessions ?? 0), 0
);
const remAll = ptContracts.reduce((a, c) => {
  const r = remainingSessions(c, logs);
  a.paid += r.paid; a.service += r.service; a.total += r.total; return a;
}, { paid: 0, service: 0, total: 0 });
const doneAll = sessTotalAll - remAll.total; // 진행+노쇼(차감분). 총 상한·음수 없음.
```

## 3. 검증 (폰 = Vercel 배포본 하드리프레시)

1. `npm.cmd run lint` 통과(신규 import unused 없나).
2. **내 실적** 탭 → 'PT 수업 현황' 카드에 `잔여 N회 · 완료 X/총 Y (유료·서비스)` + 진행 바.
3. **손검산:** PT 회원 2~3명의 (계약 세션합)·(비-voided 로그수)를 눈으로 더한 값과 카드 숫자 일치.
4. **차감 반영:** 운동일지에서 저장+차감/노쇼 1회 후 내 실적 재진입 → 잔여 −1·완료 +1(MyStats는 마운트 시 로드라 탭 재진입/새로고침 기준).
5. **재등록:** PT 회원에 계약 추가 → 총·잔여 그만큼 증가.
6. **인계 회원:** 이월 계약이 총·잔여에 포함되나.
7. **제외 확인:** OT 회원·inactive(환불/오프보딩) 회원은 총/잔여에 안 잡힘.
8. **데모 모드**(키 없이)도 렌더(members 데모 기준).

## 4. 리스크 / 회귀

- `memberStatus.js` 무변경 — `remainingSessions`·`viewFor` 재사용만. 기존 MyStats 집계(매출·급여·클로징) 무영향(파생·카드 추가일 뿐).
- 데이터 스코프: `contracts`는 RLS로 본인 계약, `ptMemberIds`는 `trainer_id===uid`+pt_active로 이중 필터 → 남의 회원 안 샘.
- 엣지: 과소진(로그>계약)이어도 `remainingSessions.total` 0 바닥 → `doneAll` 총 상한(음수·초과 없음). 진행 바도 0~100 클램프.
- 최신성: MyStats는 **마운트 1회 로드**라 카드도 그 스냅샷 기준(다른 탭에서 차감해도 재진입/새로고침 후 반영). 기존 매출·수업 지표와 동일 동작이라 일관.
