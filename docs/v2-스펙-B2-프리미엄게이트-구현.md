## 0. 무엇을 만드나 (한 줄)

**회원관리(회원앱)를 `plan='premium'` 전용으로 잠근다(층2).** basic 계정은 회원 링크 발급도, 회원 로그인도 서버에서 거절. **선행 B1 완료됨**(account.plan·subscription_status·current_period_end·`auth_account_id()`).

**게이트 두 지점(둘 다 서버 = UI 우회 차단):**
1. **발급** — `MemberAppLink.issue()`의 클라 UPDATE → **`issue_member_token()` RPC**(premium 아니면 거절)로 교체.
2. **회원 로그인** — `/api/member-auth`가 회원 소속 계정 plan 확인 → premium 아니면 401/403. (premium 해지 시 이미 나간 링크도 그날부터 안 열림.)

> **베타 영향 없음:** 베타 계정은 전원 `plan='premium'`이라 B2는 지금 눈에 안 보임. B2는 **basic 티어로 유료 판매할 때** 회원앱을 premium 전용으로 파는 스위치.

## 1. 확정 결정

| # | 항목 | 결정 |
|---|---|---|
| 1 | 프리미엄 범위 | **회원관리 = 회원앱**(링크 발급 + 회원 로그인). |
| 2 | 발급 게이트 | 클라 UPDATE → **SECURITY DEFINER RPC** `issue_member_token()`. plan 확인 서버측. |
| 3 | 로그인 게이트 | `member-auth`에서 계정 plan+active+만료 확인. 실패 시 거절. |
| 4 | 다운그레이드 반영 | **로그인 차단**으로 반영(재로그인 불가). 이미 로그인된 세션은 JWT 만료(≤~1h)까지 잔존 — S3 revoke와 동일 한계, 실질 충분. (즉시 컷 원하면 §6 후속.) |
| 5 | UI | premium 아니면 링크 바를 **🔒 업그레이드** 상태로(우회해도 서버가 거절 = 이중). |

## 2. SQL — 마이그레이션 (Supabase 수동)

**`docs/migrations/2026-07-16-b2-premium-gate.sql`**

```sql
-- =============================================================================
-- B2 — 프리미엄 게이트: auth_account_plan() + issue_member_token()
-- 전제: B1(account.plan/subscription_status/current_period_end · auth_account_id) 완료. 멱등.
-- =============================================================================

-- 1) 계정 등급 헬퍼 — auth_account_id()와 동일 틀(active+구독활성+만료 통과일 때만 plan, 아니면 NULL).
create or replace function auth_account_plan()
returns text language sql stable security definer set search_path = public as $$
  select a.plan
  from trainer t
  join account a on a.id = t.account_id
  where t.id = auth.uid()
    and t.active
    and a.subscription_status = 'active'
    and (a.current_period_end is null or a.current_period_end > now())
$$;
grant execute on function auth_account_plan() to authenticated;

-- 2) 회원 링크 발급 RPC — premium 아니면 거절(UI 우회 차단). 대상 회원이 내 account 소속인지도 확인.
create or replace function issue_member_token(p_member_id uuid)
returns uuid language plpgsql security definer set search_path = public as $$
declare
  v_account uuid := auth_account_id();
  v_token   uuid;
begin
  if v_account is null then
    raise exception 'unauthorized' using errcode = '42501';
  end if;
  if auth_account_plan() is distinct from 'premium' then
    raise exception 'premium_required' using errcode = '42501';
  end if;
  if not exists (select 1 from user_table where id = p_member_id and account_id = v_account) then
    raise exception 'member_not_found' using errcode = 'P0002';
  end if;
  v_token := gen_random_uuid();                       -- 소문자 v4 → member-auth UUID_RE 정합
  update user_table set member_token = v_token where id = p_member_id;
  return v_token;
end;
$$;
grant execute on function issue_member_token(uuid) to authenticated;

-- 검증:
--   select auth_account_plan();            -- 본인 세션 없으면 NULL(대시보드) — 앱/REST로 확인.
--   (앱) premium 계정에서 [링크 생성] → 토큰 반환. basic로 내리면 premium_required.
-- 롤백:
--   drop function if exists issue_member_token(uuid);
--   drop function if exists auth_account_plan();
```

> `raise exception 'premium_required'` → supabase-js에서 `error.message === 'premium_required'`로 옴 → 클라가 그걸로 업그레이드 안내 분기.
> `user_table` 기존 발급 UPDATE 정책은 그대로 둬도 무방(이제 발급은 RPC로 가므로 미사용). 굳이 제거 안 함(회귀 위험 회피).

## 3. `app/api/member-auth/route.js` — 로그인에 plan 게이트

**(1) 회원 조회 select에 `account_id` 추가**(1번 블록):
```js
  const { data: member } = await svc
    .from("user_table")
    .select("id, name, phone_number, member_auth_id, account_id")   // ← account_id 추가
    .eq("member_token", token)
    .maybeSingle();
  if (!member) return FAIL();
```

**(2) 끝4 대조(2번) 직후, auth 유저 생성(3번) 앞에 plan 게이트 삽입:**
```js
  // 2.5) 소속 계정이 premium·활성·미만료인지 (층2 게이트). 아니면 회원앱 로그인 거절.
  const { data: acct } = await svc
    .from("account")
    .select("plan, subscription_status, current_period_end")
    .eq("id", member.account_id)
    .maybeSingle();
  const premiumActive =
    acct &&
    acct.plan === "premium" &&
    acct.subscription_status === "active" &&
    (!acct.current_period_end || new Date(acct.current_period_end) > new Date());
  if (!premiumActive) {
    return Response.json(
      { error: "회원앱 이용이 일시 중단되었어요. 담당 트레이너에게 문의해 주세요." },
      { status: 403 }
    );
  }
```

- 효과: 트레이너 계정이 basic이거나 구독 만료/해지면 회원이 **로그인 불가**(재방문 세션도 만료되면 재로그인 막힘).
- 나머지 로직(auth 유저 확보·세션 발급)은 **불변**.

## 4. `components/views/MemberAppLink.jsx` — RPC 발급 + 🔒 UI

**(1) `issue()`를 RPC 호출로 교체** (클라 UPDATE 제거 · S3 스펙 §4에 "B2 유일 교체점"으로 표시된 곳):
```jsx
  const issue = async () => {
    if (busy) return;
    if (!supabase) { showToast("데모 모드 — 실제 발급 불가"); return; }
    setBusy(true);
    const { data: newToken, error } = await supabase.rpc("issue_member_token", {
      p_member_id: member.id,
    });
    if (error || !newToken) {
      setBusy(false);
      if (error?.message === "premium_required") {
        showToast("회원앱은 프리미엄 전용이에요 — 업그레이드가 필요해요.");
      } else {
        showToast("발급 실패" + (error ? ": " + error.message : ""));
      }
      return;
    }
    onMemberPatch?.(member.id, { member_token: newToken }); // 낙관 갱신
    const ok = await copyText(linkFor(newToken));
    showToast(ok ? "링크 생성·복사됨" : "발급됨 (복사 실패 — 링크 복사 다시 눌러줘)");
    setBusy(false);
  };
```

**(2) 🔒 사전 표시 (선택·권장):** `isPremium` prop을 받아, `false`면 발급 버튼 대신 잠금 안내:
```jsx
export default function MemberAppLink({ member, onMemberPatch, isPremium }) {
  // ...
  // 렌더 상단(발급 버튼 자리)에서 isPremium === false면 잠금 바:
  //   <Badge tone="zinc">🔒 프리미엄</Badge> + <span className="text-[11px] text-muted">회원앱은 프리미엄 전용이에요</span>
  //   (기존 [링크 생성]/[링크 복사]/[PT 종료]는 숨김)
```
- `isPremium`이 `undefined`(안 넘겨줌)면 **기존대로 동작**(발급 시도 → 서버가 거절하면 토스트). 즉 prop plumbing은 **선택**이고, 서버 RPC가 실제 게이트다.
- prop을 넘기려면(권장): 트레이너 앱이 계정 plan을 1회 조회해 내려줌 → §5.

## 5. (선택) 트레이너 앱에서 plan 내려주기 — 사전 🔒용

이미 B1 `AuthGate`가 `my_account_status()`를 부른다(거기 `plan`·`access` 있음). 사전 🔒을 원하면:

- **간단안:** `app/page.jsx`(또는 PT 뷰 로드부)에서 `supabase.rpc("my_account_status")` 1회 → `plan` state → `PTView`로 → `MemberAppLink isPremium={plan === "premium"}`.
- 프롭 연결이 번거로우면 **이 단계는 건너뛰고** §4(1)의 서버 거절+토스트로만 가도 됨(게이트는 이미 서버에서 강제). 사전 🔒은 UX 개선일 뿐.

> 판단: prop 연결이 1~2군데면 넣고, 여러 단계 관통이면 생략(서버 게이트로 충분). 클로드코드가 실제 prop 흐름 보고 결정.

## 7. 주의·엣지

- **RPC가 진짜 게이트:** UI 🔒은 UX. 서버 `issue_member_token`/`member-auth`가 basic·만료를 강제하므로 콘솔에서 직접 rpc/REST 때려도 거절.
- **다운그레이드 즉시성:** 로그인 게이트라, 이미 로그인된 회원 세션은 JWT 만료(≤~1h)까지 열람 가능(그 후 재로그인 막힘). 즉시 컷이 필요하면 **후속**: `auth_member_id()`를 회원 소속 계정 plan으로 게이트(B1-B처럼 광범위 변경 → 단계 배포). 지금은 불필요.
- **gen_random_uuid 소문자:** member-auth `UUID_RE`(`/i`)·`toLowerCase()`와 정합. 문제 없음.
- **member_not_found:** 타 계정 회원 id로 발급 시도 시 RPC가 거절(account 소속 확인).
- **B1 헬퍼 재사용:** `auth_account_id()`·`auth_account_plan()` 모두 active+구독+만료를 본다 → basic뿐 아니라 **미결제·만료 계정도 발급/로그인 거절**(일관).
