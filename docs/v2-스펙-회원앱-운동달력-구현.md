## 0. 무엇을 만드나 (한 줄)

회원 홈에 **월 달력**을 붙여, 각 날짜에 그날의 활동을 점(dot)으로 표시: **PT**(수업일지 올라온 날)·**개인운동**(스케줄 personal)·**유산소**(유산소 기록). **같은 날 여러 개면 다 표시.** SQL·새 쿼리 없음 — 이미 로드된 `logs`·`cardio`·`schedule` 파생.

---

## 1. 데이터 소스 · 마커 규칙

세 데이터셋(이미 홈에서 로드됨)을 **로컬 날짜(YYYY-MM-DD)로 버킷팅**:

| 마커 | 소스 | 날짜 키 |
|---|---|---|
| **PT** | `logs`(member_workout_log) · **또는** `schedule` kind='pt' | 로그: `session_at ?? created_at`(timestamptz→로컬 날짜) · 스케줄: `on_date` |
| **개인운동** | `schedule` kind='personal' | `on_date` |
| **유산소** | `cardio` | `performed_on` |

- **PT는 수업일지(트레이너 작성)가 authoritative** + 회원이 스케줄에서 'PT'로 체크한 날도 합집합(둘 중 하나만 있어도 PT).
- `performed_on`·`on_date`는 이미 'YYYY-MM-DD' 문자열(date 컬럼) → **그대로 키**. 로그의 timestamptz만 `new Date`로 로컬 변환.


### 2.2 파생 — 날짜별 활동 Set

```js
// 로컬 'YYYY-MM-DD'
function ymd(d) {
  const m = String(d.getMonth() + 1).padStart(2, "0");
  const day = String(d.getDate()).padStart(2, "0");
  return `${d.getFullYear()}-${m}-${day}`;
}

// logs/cardio/schedule → { 'YYYY-MM-DD': Set('pt'|'personal'|'cardio') }
function buildActivityMap(logs, cardio, schedule) {
  const map = {};
  const add = (key, type) => { (map[key] ||= new Set()).add(type); };
  (logs || []).forEach((l) => { const iso = l.session_at ?? l.created_at; if (iso) add(ymd(new Date(iso)), "pt"); });
  (schedule || []).forEach((s) => { if (s.on_date) add(s.on_date, s.kind === "pt" ? "pt" : "personal"); });
  (cardio || []).forEach((c) => { if (c.performed_on) add(c.performed_on, "cardio"); });
  return map;
}
```

### 2.3 `MemberActivityCalendar` 컴포넌트

**Props:** `{ logs, cardio, schedule }`.

**상태·파생:**
```js
const [cursor, setCursor] = useState(() => { const d = new Date(); return { y: d.getFullYear(), m: d.getMonth() }; });
const [selected, setSelected] = useState(null); // 선택한 날짜 키(상세용) · null이면 상세 없음
const activityMap = useMemo(() => buildActivityMap(logs, cardio, schedule), [logs, cardio, schedule]);
```

**월 그리드 계산:**
```js
const first = new Date(cursor.y, cursor.m, 1);
const startWeekday = first.getDay();                 // 0=일
```

**렌더:**
- **헤더:** `◀  {cursor.y}년 {cursor.m+1}월  ▶`(ChevronLeft/Right · 월 이동).
- **요일 행:** 일 월 화 수 목 금 토(text-muted · 작게).
- **날짜 그리드**(`grid grid-cols-7 gap-1`): 각 셀
  - 빈칸이면 빈 div.
  - 날짜면: 정사각 셀, 날짜 숫자(오늘이면 강조 — 예 `ring-1 ring-primary` 또는 primary 텍스트), 아래 **마커 점들**: `MARK_ORDER.filter(t => set.has(t)).map(t => <span className={`h-1.5 w-1.5 rounded-full ${ACTIVITY_MARKS[t].dot}`} />)`.
  - 셀 `onClick={() => setSelected(key)}`(활동 있으면 클릭 가능 · 커서 포인터). 활동 있는 날은 배경 살짝(`bg-elevate`).
- **범례:** `ACTIVITY_MARKS`를 점+라벨로 한 줄(PT · 개인운동 · 유산소).
- **선택일 상세(그리드 아래):** `selected`면 그날 활동 목록 표시:
  - PT: 그 날짜 `logs` 중 매칭 → "PT 수업"(있으면). (ai_summary 요약 한 줄은 선택 · 길면 생략.)
  - 개인운동: 그 날짜 `schedule` personal의 note.
  - 유산소: 그 날짜 `cardio`의 종류·분.
  - 없으면 "기록 없음"(또는 상세 영역 안 띄움).

**톤:** 회원앱 토큰(`bg-card`·`border-line`·큰 글씨). 셀은 정사각(`aspect-square`)·모바일 탭 크기. 새 색은 §2.1 맵만(동적 조립 금지). lucide `CalendarDays`·`ChevronLeft`·`ChevronRight`.

## 4. 주의·엣지

- **SQL 없음 · 순수 파생** — `logs`·`cardio`·`schedule`는 이미 로드됨. 달력은 렌더 계산만(추가 쿼리 0).
- **날짜 정합:** `performed_on`·`on_date`는 date 문자열(로컬) 그대로 키. 로그 timestamptz만 `new Date`로 로컬 변환 → KST 브라우저 전제(회원앱 기존 fmtDay와 동일).
- **로드 한도:** 현재 `logs`는 전체, `cardio`/`schedule`는 최근 60건 로드. 달력에서 지난달로 많이 넘어가면 60건 밖은 점이 안 뜰 수 있음 → MVP는 최근 위주로 충분. 필요 시 후속에 월별 조회로 승급(지금은 범위 밖).
- **색 토큰:** `bg-primary` 미정의면 `bg-rose-500`. 세 색 static map만(퍼지 안전).
- **트레이너앱:** 이번 범위 밖(원하면 후속에 트레이너용 회원 활동 달력도 같은 파생으로 추가 가능).
