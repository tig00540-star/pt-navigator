## 0. 이번 세션 성과 (전부 배포됨 · origin/main)

### 무게 진단 (코드 변화 없음)
- `/` 트레이너 First Load ≈ **260KB gzip**(모던폰). 절반이 고정 벤더(react 70 + supabase 60). 감당 OK.
- 발견: 라우트 분할은 자동(/·/admin 별도), `/` 안엔 코드분할 0이었음 → 아래 PTView 지연로딩으로 착수.
- supabase realtime 미사용인데 번들에 딸려옴(60KB). 정 가볍게 필요할 때 postgrest-js+auth-js 분리 카드(비트리비얼).

### PTView 지연로딩 (무게 위생)
- `components/views/MemberViewShell.jsx` — PTView를 `next/dynamic`(ssr:false, ViewLoading 폴백)으로. PT 전용 서브트리(운동일지·재등록·인바디·환불 ~60KB 소스)가 PT 회원 선택 시에만 로드. InactiveView는 정적 유지.
- 후속(선택): OT 무거운 탭(SecondOTTab·FirstOTAssist·ObservationTab)도 같은 패턴. 큰 파일 분리는 **기회주의적으로만**(테스트 프레임워크 없어 순수 리팩터 리스크).

### 원(₩→원) 표기 통일
- `lib/format.js` `won` = `(n)=> (n ?? 0).toLocaleString("ko-KR") + "원"` → "200,000원". `app/admin/page.jsx` 중복 정의 제거하고 lib import로 일원화. 모든 매출이 won() 경유라 앱 전체 반영.

### D-3 — 2차 브리핑에 "내 과거 클로징 케이스" 근거 (dark·게이트 5건)
- **스펙:** `docs/v2-D3-spec-2차케이스근거.md`. 마이그레이션 없음(closing_detail·closing_profile 기존 재료 읽기만).
- **lib:** `closingSuccessCount`(게이트용, user_table!inner 조인 트레이너 스코프) · `closingCasesForTrainer`(익명 프로파일+3박자) · `closingCaseGate`(5+ on: 5~9 잠정/10+ 확신).
- **서버** `app/api/ot-brief/route.js` — secondPrompt에 `closingCases` 있을 때만 케이스 입력블록 + `case_feedback` 스키마(diagnosis·proven_lead·avoid_repeat·example·your_read) additive. 없으면 바이트 동일(회귀 안전). maxTokens 6144(케이스 시). FIELD_TERMS 누출방어 추가.
- **클라** `components/tabs/SecondOTTab.jsx` — 트레이너 스코프 케이스 로드→게이트→body 첨부, `case_feedback` 카드 렌더. 개발 오버라이드 `?d3=1`(+데모 케이스). 캐시 렌더 뒤로 로드(즉시성).
- **톤 검증됨:** `?d3=1` 데모로 proven_lead가 "대사 이식" 아닌 "리딩", 진단이 표면 아닌 진짜 장애물 짚음 확인.
- **상태 = 게이트 OFF**(성공 클로징 0~2건). 실사용자엔 안 뜸. **5건 되면 자동 발동.**
- **watch:** 실데이터에서 케이스 구체(횟수·금액) 창작 재발 시 프롬프트에 "케이스에 없는 수치 창작 금지" 한 줄.

### 페이롤 — 확장 가능 급여 시스템 (A~E 완성)
- **철학:** 자동계산=보조, **최종급여는 원장이 확정하는 편집 필드(payroll_run.final_total)=진실.** 모델 하드코딩 안 함 — 규칙이면 scheme 타입 추가, 아니면 manual.
- **스펙:** `docs/v2-페이롤-스펙-확장가능급여.md`.
- **A(마이그·docs/migrations 기록본):**
  - `2026-07-12-pay-scheme.sql` — pay_scheme(account_id·trainer_id(null=계정기본/set=override)·type('banded'|'manual')·band_basis('revenue'|'session_count')·bands jsonb). 계정/원장 RLS. **account_id DEFAULT auth_account_id()**. 기존 pay_policy→계정기본 banded seed.
  - `2026-07-12-payroll-run.sql` — payroll_run(account·trainer·ym·computed_total·final_total·scheme_snapshot·locked_at). SELECT 원장 전체/트레이너 본인, 쓰기 원장. account_id DEFAULT.
- **B(lib):** `resolveScheme`(트레이너 override→계정기본→null) · `sessionCountByTrainer` · `payForScheme`(banded: basis값으로 밴드 선택 → payout_type[pct_of_price/flat_per_session/fixed] + incentive · manual→computed=null). **payForMonth 제거(콜사이트 0).**
- **C1/C2:** `components/AdminPayrollSettings.jsx` — 원장용 스킴 편집. type·band_basis·bands CRUD + 라이브 미리보기(payForScheme) + 트레이너별 override(스코프 선택·저장·삭제=계정기본 따름). 동적 헤더.
- **D:** `components/PayrollConfirm.jsx` + admin/page.jsx — 트레이너별 "예상 급여(payForScheme)→최종액 확정(payroll_run upsert)". admin pay_policy·payForMonth 제거.
- **E:** `components/views/MyStats.jsx` — payForScheme 전환. 헤드라인 3분기: 확정(final)/예상(미확정·computed)/수동 대기(manual 미확정).
- **커버리지:** 매출구간%·수업수밴드×회당정액·단순회당고정 = 자동. 시간대별·기타 = manual(원장 확정). 예: 수업 160개×[100→2만/150→3만]=480만 자동.
- **레거시:** pay_policy 테이블은 DB에 잔존(코드 미참조).

## 1. 즉시 다음 액션 후보 (다음 채팅)

1. **파일럿 계속 굴리기** — D-3는 성공 클로징 **5건**이면 자동 켜짐. 케이스 쌓는 게 최우선(코드 아님). 5건 근처면 D-3 후속(1차 대칭·블라인드스팟·결정권자 키트) 착수.
2. **페이롤 `time_slot` 자동** — 수업 시각별 단가(아침 2만/저녁 3만류). `daily_workout_log.session_at`(KST 시)로 슬롯 매칭·합산. pay_scheme.type='time_slot' + slots jsonb + payForScheme 분기. 데이터 있어 소규모. (그 전엔 manual로 커버됨.)
3. **6기능 wishlist** — 추천 순서 **6(페이롤 UI·완료)→2(할일/완료)→1(앱내 공지)→4(월간 리포트)→3(FC 도메인)→5(회원 앱)**. 값싼 2·1부터. FC(3)는 도메인 신설 선행, 회원앱(5)은 회원 로그인=별도 챕터.
4. **무게 후속(선택·짬)** — OT 무거운 탭 dynamic. 큰 파일 분리는 기회주의적으로만.

## 2. 파일럿 관찰 포인트

- **페이롤:** 원장이 스킴(밴드/manual·트레이너별) 세팅 자연스러운지 / 예상→확정 흐름 체감 / 확정액이 트레이너 MyStats에 "확정 급여"로 뜨는지 / manual 센터에서 최종액 입력 마찰.
- **D-3(5건 후):** proven_lead가 리딩(대사이식 X)인지 / 진단이 진짜 장애물 짚는지 / 케이스 구체 창작 없나.
- **무게:** PT 회원 진입 시 "불러오는 중…" 후 PTView 정상 / 원 표기 전 화면.

## 3. 참고 — 현재 지점

- **급여:** pay_scheme(계정기본+override) → resolveScheme → payForScheme(banded/manual) → admin 확정(payroll_run.final_total) → MyStats 확정 우선. pay_policy·payForMonth 없음.
- **AI 지점(총정리본):** /api/ot-brief 전 phase sonnet·thinking off. secondPrompt = 1차 관찰 + (게이트 시)케이스근거. 1차=firstPrompt(+내 패키지) · 재등록·급한불 무영향.
- **보안 미결(총정리 §5 유지):** ot_log·daily_workout_log·inbody_log 트레이너 RLS 부재(계정 스코프만) · 오프보딩(active) 미반영 · API 라우트 인증/레이트리밋.
