## 7. 🟢 낮은 우선순위 (기록만)
- **[감사] 회원앱 끝4 브루트포스** — `member-auth/route.js:9-16` 스로틀이 **인메모리 Map**이라 서버리스에서 인스턴스 수만큼 뚫림. 비밀공간 1만개(`phoneLast4`). `member_token` UUID가 추측 불가라 HIGH가 아니라 MEDIUM이지만, **링크를 카톡으로 공유하는 설계**라 실질 위험. `requireTrainer`의 20req/min 스로틀도 동일 한계.
- **[감사] 멀티테넌트 격리가 100% RLS 의존** — 클라 쿼리 어디에도 `account_id` 필터가 없다(DB DEFAULT + RLS에 위임). 가장 날카로운 곳: `ot-brief/route.js:620-632`가 `center_machine`을 unfiltered로 읽어 프롬프트에 넣음 → RLS 정책이 마이그레이션 중 깨지면 **타 센터 장비 목록이 유출**. `.eq("account_id", ...)` 추가는 공짜. (대조: `member-revoke/route.js:34-40`은 service-role이라 RLS가 없어서 명시적으로 account 검증을 함 — 올바른 본능.)
- **[감사] `DEMO_MEMBER`에 `id`가 없음** (`app/page.jsx:46-62, 552`) — `confirmPtActive`가 `member.id`를 무가드 사용(`:583,588,604`) → `.eq("user_id", undefined)`. `PTView`의 `key={member.id}`도 `undefined`가 되어 **회원 전환 시 리마운트가 안 걸림**(서브탭 폼 상태가 회원 간 누수). 신규 계정이거나 `loadMembers` 실패 시 도달.
- **[감사] `perSession`이 `price` null 미가드** → `NaN원` 렌더. `FirstOTAssist.jsx:184`, `SecondOTTab.jsx:526`. **세일즈 클로징 화면에 뜨는 값**이라 인상이 나쁨.
- **[감사] `member-auth/route.js:95`** — `member_auth_id` update에 `.select()` 없음. service-role이라 교훈1은 아니지만, 0행 매칭 시 그대로 진행됨. `member-revoke/route.js:52-58`은 올바르게 처리 중이라 정합성 차원.
- **[감사] 인덱스 key 이슈** — `SetsEditor.jsx:101,122`가 삭제 가능한 리스트에 index key. 입력값은 controlled라 정상이나, `ExerciseNameInput`의 `open` 상태가 위치 기반이라 **삭제 시 엉뚱한 행의 드롭다운이 열림**. `ObservationTab` movements도 동일.
- **[감사] `create-trainer/route.js`가 임시 비밀번호를 응답 바디로 반환** — owner 게이트가 있고 플로우상 필요하지만, 평문이 브라우저를 경유.
- **[감사] `AnnouncementGate` 조회 2곳 catch 없음** — 실패 시 공지가 조용히 안 뜬다. **필수 확인(mandatory-ack) 공지도 포함**이라 제품 정합성 문제.
