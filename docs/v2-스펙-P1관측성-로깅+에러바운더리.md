## 0. 왜
- 코드베이스 전체 `console.error` **4건뿐 · 전부 AI 라우트**(`ot-brief:714`·`voice-log:143,180`·`machine-cues:57`). `create-trainer`/`member-auth`/`member-revoke`는 **0줄**. **[확인]**
- 방금 배포한 묶음 A가 **새 403 경로("구독이 필요합니다")를 만들었는데 로그가 없다.** 베타 트레이너가 "AI 안 돼요" 하면 **세션만료 / 오프보딩 / 구독미활성 / 스로틀**을 구분 불가 — 그리고 이게 **가장 먼저 터질 확률이 높은 항목.**
- 에러 바운더리(`error.jsx`/`global-error.jsx`)는 배치 1-A가 만들었으나 `error`를 **받아놓고 안 씀.** 프로덕션은 메시지가 마스킹되고 `digest`만 남는데 그걸 안 찍고·안 보여줘서 **서버 로그와 대조할 열쇠가 없다.**

## 1. 로깅 규칙 (전 파일 공통 · 반드시 준수)
- **🔒 비밀 미로깅(최우선):** **JWT 토큰 원문·회원 `member_token`·휴대 끝4(`last4`)·`authorization` 헤더를 절대 로깅하지 않는다.** Vercel 함수 로그는 팀 구성원이 열람 가능. 식별은 **`uid`(트레이너 auth id) / `member_id`(user_table id)**만 사용(둘 다 UUID·비밀 아님).
- **레벨 구분:** 예상된 거부(401/403/429/형식오류) = `console.warn` · 서버 실패(생성/삭제/연결/설정 오류) = `console.error`. → `console.error`는 "진짜 고장"만 남아 알림 가치 유지.
- **태그 접두:** 파일별 `[requireTrainer]` · `[create-trainer]` · `[member-auth]` · `[member-revoke]`로 grep 가능하게. 사유를 한눈에 구분되게 문구에 명시(예: `403 구독 미활성`).
- **상태코드·응답 body·로직 불변.** 로그 줄만 각 `return` **직전**에 삽입(분기를 블록 `{}`으로 감쌈).
