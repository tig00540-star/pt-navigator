# 백로그 — 알림 / 리마인더 자동화 (보류 · 급하지 않음)

> 트레이너가 '오늘 챙길 것'을 수동으로 열어봐야 아는 것 → 능동 알림으로.
> 조건 계산(재접근·재등록·이탈위험 등)은 이미 lib/memberStatus에 있음. 없는 건 '전달 채널 + 스케줄'뿐.

## 무엇을 알림 (역할별)
- 트레이너: 재접근 예정일 도래 · 재등록 재접근 · 재등록 타이밍(유료<10) · 미확정 클로징 · 연체 예약 · 이탈 위험(잔여>0·14일 무수업)
- 원장: 미확정 클로징 · 월말 급여 · 매월1일 예상매출 · 신규/환불 이상치
- 회원(프리미엄): 오늘 운동 미기록 · 인바디 측정 주기 · PT 예약 전날

## 단계 (작은 것 → 큰 것)
1. 인앱 뱃지 — 헤더에 "챙길 것 N" 숫자(어느 화면서든). 눌러 이동 or 목록 펼침.
   · 외부 0, 지금 바로 가능. 한계: 앱 열어야 보임. (뱃지 동작 a=탭이동 / b=드롭다운 미정)
2. 웹 푸시(PWA) — 앱 안 열어도 폰에 띡. 필요: 서비스워커 + VAPID 키 + Cron(매일 조건 평가). iOS는 홈화면 추가 PWA만.
3. 카카오 알림톡 / 문자 — 도달 최강(회원 리마인더). 외부 계정·심사·비용.

## 자동화(스케줄)
Vercel Cron 또는 Supabase pg_cron/Edge Function으로 하루 1회 조건 평가 → 뱃지 채움/푸시 발송.

## 진행 상황
- [완료] 이탈 위험 조기경보 카드('오늘' 탭, 잔여>0·14일 무수업) — 알림센터 1번 조각.
- [보류] 인앱 뱃지 · 웹 푸시 · 카톡.
