# v2 스펙 — 센터 머신·덤벨 등록 → OT/재등록 루틴 반영 · 구현 (기능 ②)

> 성격: **구현 스펙**(클로드코드 실행용). 근거 = `v2-기능요청-가능성판정.md` ②.
> 확정(트레이너): 관리 UI = **설정 안**(대표/solo 편집·트레이너 열람) · 장비 **종류(kind) 필드 추가** · AI = **센터 전체 장비 → "있는 것으로만"**.
> 두 파트로 분리 배포: **②-A(머신 관리 = SQL+UI · 먼저 커밋·검증) → ②-B(AI 반영 = ot-brief 라우트만).**
> 흐름: 이 스펙 → 트레이너 → 클로드코드(diff) → 트레이너(git diff) → 웹Claude(검토) → 폰/Supabase 검증. 커밋은 트레이너 직접.

---

## 0. 한눈에

```
②-A  [대표/solo] 설정 → '보유 장비' → 머신·덤벨 등록(이름·종류[머신/프리웨이트/맨몸]·브랜드)
        └ center_machine(account 스코프 · owner 편집 · 트레이너 열람)
②-B  [1·2차 OT·재등록 AI 생성] → ot-brief 라우트가 그 계정 center_machine을 조회
        └ 프롬프트에 [보유 장비] 블록 주입 + "운동 구성은 이 장비 범위에서만" 지시
        → AI가 없는 장비 안 만들고, 센터에 있는 머신·덤벨로 루틴 구성
```

**성능:** center_machine은 장비 이름 몇 줄짜리 작은 테이블. ②-B는 AI 호출당 가벼운 조회 1번 + 프롬프트 몇 줄 추가뿐 → 폰/앱 무게 0, 체감 지연 없음. **출력 스키마 무변경** → 기존 캐시(first_assist·brief·reg_brief) 그대로 렌더, 새로 생성분만 장비 반영.
**스케일:** 모든 조회가 **account 스코프**(내 센터 것만) + `account_id` 인덱스라, 전 센터 합산이 수천~수만 행이어도 각 센터는 자기 장비 수십 개만 읽음 → 총량 무관. AI 프롬프트도 상한 120으로 캡 → 프롬프트 안 부풂.

## 1. center_machine 현재 상태 (확인됨)

- 컬럼: `id·created_at·brand·name·account_id`. RLS: **SELECT**(`account_id=auth_account_id()`)·**INSERT**(account 스코프)만 존재 → **UPDATE·DELETE 정책 없음**(편집·삭제하려면 추가 필요).
- `account_id`에 DEFAULT 없음 → 클라 insert 시 명시 필요(불편). `library_item`처럼 DEFAULT 걸어 간소화.
- `auth_is_owner()` 헬퍼 존재(solo 트레이너 = 자기 solo account의 owner라 true).

# ②-A · 머신 관리 (SQL + UI)

## 2. PART A1 — SQL 마이그레이션 · 기록본 `docs/migrations/2026-07-16-center-machine-manage.sql`

```sql
-- =============================================================================
-- 센터 장비 관리(②) — kind 컬럼 + account_id 기본값 + owner 쓰기 정책(update/delete 추가)
-- 실행: Supabase SQL Editor(수동) · 이 파일 = git 기록본. 멱등.
-- ★ 데이터 무변경(정책·컬럼만). 기존 center_machine 행 유지.
-- =============================================================================

-- 1) 장비 종류 컬럼(머신/프리웨이트/맨몸 · null=미분류)
alter table center_machine add column if not exists kind text;

-- 2) account_id 기본값 = 내 계정(클라 insert 시 미포함 가능 · library_item 패턴)
alter table center_machine alter column account_id set default auth_account_id();

-- 3) account_id 인덱스 — '내 센터 것만' 조회를 전체 테이블 크기와 무관하게 빠르게(스케일 대비).
create index if not exists center_machine_account_idx on center_machine (account_id);

-- 4) 쓰기 = 대표(owner)만. 기존 INSERT(account 스코프)를 owner로 좁히고 UPDATE/DELETE 추가.
--    SELECT(auth_read_center_machine · account 스코프)는 그대로 = 트레이너 열람 유지.
--    (인덱스 3) 위에서 추가 · account 스코프 조회라 전 센터 총합이 수만 행이어도 각 센터는 자기 것만 읽음.)
drop policy if exists "auth_write_center_machine" on center_machine;
create policy "owner_insert_center_machine" on center_machine for insert to authenticated
  with check (account_id = auth_account_id() and auth_is_owner());
create policy "owner_update_center_machine" on center_machine for update to authenticated
  using      (account_id = auth_account_id() and auth_is_owner())
  with check (account_id = auth_account_id() and auth_is_owner());
create policy "owner_delete_center_machine" on center_machine for delete to authenticated
  using      (account_id = auth_account_id() and auth_is_owner());

-- =============================================================================
-- 검증
--  select policyname, cmd from pg_policies where tablename='center_machine' order by cmd;
--    기대: SELECT(auth_read_center_machine) + INSERT/UPDATE/DELETE(owner_*_center_machine)
--  select column_name from information_schema.columns
--    where table_name='center_machine' and column_name='kind';  -- kind 존재
-- ROLLBACK:
--  alter table center_machine drop column if exists kind;
--  alter table center_machine alter column account_id drop default;
--  drop index if exists center_machine_account_idx;
--  drop policy if exists "owner_insert_center_machine" on center_machine;
--  drop policy if exists "owner_update_center_machine" on center_machine;
--  drop policy if exists "owner_delete_center_machine" on center_machine;
--  create policy "auth_write_center_machine" on center_machine for insert to authenticated
--    with check (account_id = auth_account_id());
-- =============================================================================
```

## 3. PART A2 — `components/views/CenterMachineSettings.jsx` (신규)

`TrainerLibrary` CRUD 패턴 미러. `useAccount().isOwner`로 편집/열람 분기. `account_id`는 DB DEFAULT라 insert 미포함. `.select()` 하드닝·데모 가드·Toast·2단계 삭제.

```jsx
"use client";
/* 보유 장비 — 머신·덤벨·프리웨이트 CRUD(설정탭). center_machine(account 스코프).
   대표(owner)·solo만 편집, 트레이너는 열람. account_id=DB DEFAULT(auth_account_id)라 insert 미포함.
   AI 루틴(ot-brief)이 이 목록을 '있는 장비'로 참조. */
import { useEffect, useState } from "react";
import { Dumbbell, Plus, Trash2, Pencil, X } from "lucide-react";
import { supabase } from "@/lib/supabaseClient";
import { useAccount } from "@/lib/useAccount";
import Eyebrow from "@/components/ui/Eyebrow";
import Button from "@/components/ui/Button";
import Toast from "@/components/ui/Toast";
import { useToast } from "@/hooks/useToast";

const KINDS = [
  { v: "machine", label: "머신" },
  { v: "free_weight", label: "프리웨이트" },
  { v: "bodyweight", label: "맨몸/도구" },
];
const KIND_LABEL = { machine: "머신", free_weight: "프리웨이트", bodyweight: "맨몸/도구" };

export default function CenterMachineSettings() {
  const { isOwner } = useAccount();
  const [rows, setRows] = useState([]);
  const [loading, setLoading] = useState(true);
  const [saving, setSaving] = useState(false);
  const [editingId, setEditingId] = useState(null);
  const [confirmId, setConfirmId] = useState(null);
  const [name, setName] = useState("");
  const [brand, setBrand] = useState("");
  const [kind, setKind] = useState("machine");
  const { toast, showToast } = useToast();

  useEffect(() => {
    let cancelled = false;
    (async () => {
      if (!supabase) { if (!cancelled) setLoading(false); return; }
      const { data } = await supabase.from("center_machine").select("*")
        .order("kind", { ascending: true }).order("name", { ascending: true });
      if (cancelled) return;
      setRows(data || []);
      setLoading(false);
    })();
    return () => { cancelled = true; };
  }, []);

  const resetForm = () => { setName(""); setBrand(""); setKind("machine"); setEditingId(null); };

  const save = async () => {
    if (saving) return;
    if (!name.trim()) return showToast("장비 이름을 입력하세요");
    const payload = { name: name.trim(), brand: brand.trim() || null, kind };
    setSaving(true);
    if (!supabase) {
      if (editingId) setRows((p) => p.map((r) => (r.id === editingId ? { ...r, ...payload } : r)));
      else setRows((p) => [...p, { ...payload, id: `demo-${Date.now()}` }]);
      resetForm(); showToast("저장됨(데모)"); setSaving(false); return;
    }
    if (editingId) {
      const { data, error } = await supabase.from("center_machine").update(payload).eq("id", editingId).select();
      if (error || !data || data.length === 0) { showToast("저장 실패 — 권한(대표만)·정책 확인"); setSaving(false); return; }
      setRows((p) => p.map((r) => (r.id === data[0].id ? data[0] : r)));
      showToast("수정됨");
    } else {
      const { data, error } = await supabase.from("center_machine").insert(payload).select();
      if (error || !data || data.length === 0) { showToast("저장 실패 — 권한(대표만)·정책 확인"); setSaving(false); return; }
      setRows((p) => [...p, data[0]]);
      showToast("추가됨");
    }
    resetForm(); setSaving(false);
  };

  const startEdit = (r) => { setEditingId(r.id); setName(r.name ?? ""); setBrand(r.brand ?? ""); setKind(r.kind ?? "machine"); setConfirmId(null); };

  const remove = async (id) => {
    if (confirmId !== id) { setConfirmId(id); return; }
    if (!supabase) { setRows((p) => p.filter((r) => r.id !== id)); setConfirmId(null); showToast("삭제됨(데모)"); return; }
    const { data, error } = await supabase.from("center_machine").delete().eq("id", id).select();
    if (error || !data || data.length === 0) { showToast("삭제 실패 — 권한/정책 확인"); setConfirmId(null); return; }
    setRows((p) => p.filter((r) => r.id !== id));
    if (editingId === id) resetForm();
    setConfirmId(null); showToast("삭제됨");
  };

  const groups = {};
  for (const r of rows) { const k = KIND_LABEL[r.kind] || "미분류"; (groups[k] ||= []).push(r); }
  const cats = Object.keys(groups);
  const inputCls = "w-full rounded-lg border border-line bg-elevate px-3 py-2 text-sm text-ink placeholder-muted outline-none focus:border-primary disabled:opacity-50";

  return (
    <div className="space-y-4">
      <Eyebrow icon={Dumbbell}>보유 장비</Eyebrow>

      {isOwner ? (
        <section className="rounded-2xl border border-line bg-card p-5 shadow-sm">
          <div className="space-y-3">
            <label className="block">
              <span className="mb-1 block text-[11px] font-medium text-muted">장비 이름 *</span>
              <input type="text" value={name} onChange={(e) => setName(e.target.value)} disabled={saving} placeholder="레그프레스 · 덤벨 세트 …" className={inputCls} />
            </label>
            <div className="grid grid-cols-2 gap-3">
              <label className="block">
                <span className="mb-1 block text-[11px] font-medium text-muted">종류</span>
                <select value={kind} onChange={(e) => setKind(e.target.value)} disabled={saving} className={inputCls}>
                  {KINDS.map((k) => <option key={k.v} value={k.v}>{k.label}</option>)}
                </select>
              </label>
              <label className="block">
                <span className="mb-1 block text-[11px] font-medium text-muted">브랜드 <span className="text-muted">(선택)</span></span>
                <input type="text" value={brand} onChange={(e) => setBrand(e.target.value)} disabled={saving} placeholder="테크노짐 …" className={inputCls} />
              </label>
            </div>
            <div className="flex gap-2">
              <Button variant="primary" size="md" onClick={save} disabled={saving} className="flex-1 gap-2">
                {editingId ? <Pencil className="h-4 w-4" strokeWidth={2.5} /> : <Plus className="h-4 w-4" strokeWidth={2.5} />}
                {saving ? "저장 중…" : editingId ? "수정 저장" : "장비 추가"}
              </Button>
              {editingId && (
                <Button variant="ghost" size="md" onClick={resetForm} disabled={saving}><X className="h-4 w-4" /> 취소</Button>
              )}
            </div>
            <p className="text-[10px] leading-relaxed text-muted">여기 등록한 장비를 AI가 1·2차 OT·재등록 루틴에 '있는 장비'로 반영합니다. 덤벨·바벨은 종류를 '프리웨이트'로.</p>
          </div>
        </section>
      ) : (
        <p className="text-[11px] text-muted">장비는 대표가 관리합니다. (열람 전용)</p>
      )}

      <section className="rounded-2xl border border-line bg-card p-5 shadow-sm">
        <Eyebrow icon={Dumbbell}>등록된 장비</Eyebrow>
        {loading ? (
          <p className="mt-2 text-sm text-muted">불러오는 중…</p>
        ) : rows.length === 0 ? (
          <p className="mt-2 text-sm text-muted">아직 등록된 장비가 없어요.{isOwner ? " 위에서 보유 머신·덤벨을 등록하면 AI 루틴에 반영됩니다." : ""}</p>
        ) : (
          <div className="mt-3 space-y-3">
            {cats.map((cat) => (
              <div key={cat}>
                <div className="flex items-center gap-1.5 py-1">
                  <span className="text-[11px] font-semibold uppercase tracking-wider text-muted">{cat}</span>
                  <span className="rounded-full bg-elevate px-1.5 py-0.5 text-[10px] font-semibold text-sub">{groups[cat].length}</span>
                </div>
                <ul className="mt-1.5 flex flex-wrap gap-2">
                  {groups[cat].map((r) => (
                    <li key={r.id} className="flex items-center gap-1.5 rounded-lg border border-line bg-elevate px-2.5 py-1.5">
                      <span className="text-sm text-ink">{[r.brand, r.name].filter(Boolean).join(" ")}</span>
                      {isOwner && (confirmId === r.id ? (
                        <span className="flex items-center gap-1">
                          <button onClick={() => remove(r.id)} className="rounded border border-rose-500/40 bg-rose-500/10 px-1.5 py-0.5 text-[10px] font-bold text-rose-700">삭제?</button>
                          <button onClick={() => setConfirmId(null)} className="text-[10px] text-muted">취소</button>
                        </span>
                      ) : (
                        <span className="flex items-center gap-1">
                          <button onClick={() => startEdit(r)} className="text-muted hover:text-primary-strong" aria-label="수정"><Pencil className="h-3.5 w-3.5" /></button>
                          <button onClick={() => setConfirmId(r.id)} className="text-muted hover:text-rose-600" aria-label="삭제"><Trash2 className="h-3.5 w-3.5" /></button>
                        </span>
                      ))}
                    </li>
                  ))}
                </ul>
              </div>
            ))}
          </div>
        )}
      </section>
      <Toast message={toast} />
    </div>
  );
}
```

## 6. PART B1 — 라우트에서 센터 장비 조회 + 프롬프트 주입

3개 콜사이트(first/second/reregister)가 이미 `authHeader()`로 트레이너 JWT를 보냄 → **클라 무변경.** 라우트가 그 토큰으로 center_machine을 account 스코프 조회(RLS)해 프롬프트에 주입.

**(a) 상단 import 추가:**
```js
import { createClient } from "@supabase/supabase-js";
```

**(b) 헬퍼 2개 추가(파일 상단 유틸 근처):**
```js
// 트레이너 JWT로 그 계정 보유 장비 조회(RLS account 스코프). 실패·데모면 [].
async function fetchCenterMachines(request) {
  const url = process.env.NEXT_PUBLIC_SUPABASE_URL;
  const anon = process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY;
  const authz = request.headers.get("authorization") || "";
  const token = authz.startsWith("Bearer ") ? authz.slice(7) : null;
  if (!url || !anon || !token) return [];
  const sb = createClient(url, anon, {
    global: { headers: { Authorization: `Bearer ${token}` } },
    auth: { persistSession: false, autoRefreshToken: false },
  });
  const { data } = await sb.from("center_machine").select("name, kind, brand");
  return Array.isArray(data) ? data : [];
}

// 프롬프트용 장비 블록. 종류별 그룹. 비면 '맨몸 위주'.
function equipmentBlock(machines) {
  // 상한 120 — 정상 센터는 수십 개라 무관하나, 비정상적으로 많아도 프롬프트가 안 부풀게 안전장치.
  const list = (Array.isArray(machines) ? machines : []).slice(0, 120);
  if (!list.length)
    return "[보유 장비 — 운동 구성은 이 범위에서만] 등록된 장비 없음 → 맨몸·기본 동작 위주. 특정 머신·기구 이름 창작 금지.";
  const label = (k) => (k === "free_weight" ? "프리웨이트" : k === "bodyweight" ? "맨몸/도구" : "머신");
  const groups = {};
  for (const m of list) {
    if (!m || !m.name) continue;
    const g = label(m.kind);
    (groups[g] ||= []).push([m.brand, m.name].filter(Boolean).join(" "));
  }
  const lines = Object.entries(groups).map(([k, arr]) => `- ${k}: ${arr.join(", ")}`).join("\n");
  return `[보유 장비 — 운동 구성은 이 범위에서만] (★목록에 없는 머신·기구 이름 창작 금지. 맨몸 동작은 항상 허용.)\n${lines}`;
}
```

```js
    : acutePrompt(member, acuteContext);
```

**(d) 프롬프트 빌더 3개 시그니처·본문 수정** — `firstPrompt`·`secondPrompt`·`reregisterPrompt`에 `centerMachines` 파라미터 추가하고, **[회원 기본정보] 블록 아래(운동 구성 지시 근처)에 `${equipmentBlock(centerMachines)}` 삽입** + 운동 구성 지시(session_plan/session_flow 등)에 한 줄 추가:
```
- [기구 제약] 동작에 기구가 필요하면 위 [보유 장비] 목록에서만 고르고, 없으면 맨몸·기본 동작으로. 목록에 없는 머신 이름을 지어내지 말 것.
```
예: `firstPrompt`의 `[회원 기본정보] … machines=${machines}` 줄 다음에 `\n${equipmentBlock(centerMachines)}\n` 삽입, `② session_plan` 지시에 위 [기구 제약] 한 줄 추가. second/reregister도 동일 결(각 운동 구성 섹션 근처).

> **스키마·캐시 영향 없음** — 출력 JSON 스키마 무변경, 입력 컨텍스트만 추가. 기존 캐시 그대로 렌더, 새 생성분만 장비 반영. `acute`(급한불)는 의료 우선이라 제외.

## 7. 검증

**②-A:**
1. PART A1 SQL 실행 → 검증쿼리(정책 4종·kind 컬럼) 확인.
2. 대표(owner) 계정 → 설정 → '보유 장비' → 레그프레스(머신)·덤벨 세트(프리웨이트) 추가 → 목록 종류별 표시. 수정·삭제 동작.
3. 일반 트레이너(role=trainer) 계정 → 같은 화면 **열람 전용**(편집 폼 없음), 삭제/수정 버튼 없음. (solo → isOwner=true라 편집 가능.)
4. 권한 방어: 트레이너가 직접 insert 시도 → 정책상 실패("권한(대표만)" 토스트).
5. 신규 회원 등록폼 '보유머신'에 새 장비가 뜨고, 덤벨(브랜드 없음)이 앞 공백 없이 표시.

**②-B:**
6. 장비 몇 개 등록된 상태에서 1차 OT AI '다시 생성' → session_plan/target_exercise 동작이 **등록된 장비·맨몸 범위 내**로 나오고, 센터에 없는 머신을 지어내지 않는지. 2차·재등록도 동일.
7. 장비 0개 계정 → "맨몸 위주"로 무난히 생성(에러 없음).
8. 회귀: 급한불(acute) 정상(장비 미주입) · 기존 캐시 회원 렌더 정상.
