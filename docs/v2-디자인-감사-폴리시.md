# v2 디자인 감사 — 라이브 폴리시 스펙 (A) + 디자인시스템 진입점 (B)

> 역할·흐름: 웹Claude(이 스펙) → 트레이너 → 클로드코드(diff) → 트레이너(git diff) → 웹Claude(검토) → 폰.
> 커밋은 트레이너 직접(파일지정 `git add` · lint 통과 후). 환경: PowerShell · lint=`npm.cmd run lint` · 폰=Vercel 하드리프레시.
> 감사 방법: Claude in Chrome으로 배포본(pt-navigator.vercel.app) **원장 로그인** 상태 라이브 캡처.
> **커버(이번 라운드):** 스케줄 · 할일 · 회원 · 내 실적 · /admin(공지 포함) — 데스크톱 폭.
> **미커버(다음 라운드):** OT/PT AI 탭(1·2차 OT·재등록·인바디·운동일지) · **정밀 모바일 폭**(도구 resize가 캡처에 반영 안 돼 폰 실기기 확인 필요). 아래 모바일 항목은 구조 기반 추론 + 폰 육안 검증 대상.

---

## 0. 총평

이미 **반쯤 디자인 시스템이 있다.** 라이트 테마, `rounded-2xl` 카드(회색 배경 위 흰 카드+옅은 그림자), 트레이너=초록(lime/emerald)·원장=보라(fuchsia) accent, 섹션별 의미색(재접근=amber, 미처리=중립, 급여=emerald). 내 실적 화면의 타이포 위계는 이미 깔끔하다. 완성도를 올리는 지렛대는 두 가지다: **(1) 눈에 띄는 즉효 폴리시 몇 개(P0), (2) 여러 파일에 인라인 중복된 패턴을 `components/ui` 프리미티브로 추출(P2=B).** P1(색·타이포·버튼 역할 통일)은 그 사이를 잇는다.

우선순위 = **P0 즉효 → P1 일관성 → P2 근본(B)**. P0·P1은 작은 diff라 회귀 위험 낮고 바로 티난다.

---

## 1. P0 — 즉효 (눈에 띄고 값싼 것부터)

1. **트레이너를 이메일로 표시 중 → 이름으로.** 스케줄 셀·필터 범례가 트레이너를 `tig00540@naver.com`로 노출(여러 셀 반복 = 시각 노이즈). 근본 원인: `trainer.name`이 시드 때 이메일로 채워짐(⑦-b seed `u.email`). 처리: (a) trainer.name에 실제 이름 채우기(원장 UI 또는 SQL 1회), (b) 표시단에서 `name`이 이메일 형태면 `@` 앞부분만 보여주는 폴백. → 스케줄이 즉시 깔끔.
2. **이름 폴백 "회원"(할일 · 미처리 예약).** 목록에 없는 회원(숨김/환불/타 트레이너 담당)이 카드에 `회원`으로 떠 미완성처럼 보임. 처리: 예약행에 회원명 조인해 실제 이름, 없으면 `이름 미상`을 muted 톤으로(현재는 기본 텍스트라 진짜 회원처럼 보임). 또는 스코프에서 제외.
3. **빈값 대시 남발(회원 카드).** `아루 OT -세 / - / - · 목표 미설정`처럼 빈 필드가 `-`로 여러 개 → 미완성 인상. 처리: 없는 필드는 **숨김**(라인 자체 생략), 나이 미상이면 `-세` 대신 나이 배지 생략, 빈 목표는 `목표 미설정`을 muted 칩으로. "없음"을 대시로 채우지 말고 자연스럽게 비우기.
4. **스케줄 모바일 = 7열 그리드 압박(최우선 · 현장=폰).** 주간 7열이 폰 폭(~390px)에서 열당 ~50px라 사실상 못 씀. 이미 있는 `오늘`(일 뷰) 토글을 활용: **좁은 폭에서 기본을 일 뷰로**, 또는 주간을 **가로 스크롤 + 시간열 sticky**. (폰 실기기로 현재 상태 먼저 확인 후 택1.)
5. **금액 자간 과다.** `204,000원`이 과하게 벌어져 보임(넓은 letter-spacing/폭). `tabular-nums` + 과한 tracking 제거로 숫자 정렬은 유지하되 자연스럽게.

## 2. P1 — 일관성 (색·타이포·버튼 역할)

6. **시맨틱 색 역할을 토큰화.** 지금 섹션마다 색이 약간 애드혹(amber/rose/slate/emerald/primary 혼용). 의미 고정: 주의=amber · 위험/삭제=rose · 정보/중립=slate · 성공/급여=emerald · 기본강조=primary. 이 역할을 토큰 맵(`C`/globals.css)에 명시해 **모든 기능이 같은 의미색**을 쓰게.
7. **버튼 변주 정리.** 게시=큰 보라 그라데이션, 추가=solid, 등록=그라데이션, 보조=보더 — 지금 제각각. `primary(그라데이션)/solid/ghost/danger` × `sm/md` 변주로 통일(트레이너=초록 계열, 원장=보라 계열은 accent만 스왑).
8. **타이포 스케일 정의.** 헤드라인 크기·굵기 편차 정리: `display`(급여 등 큰 숫자, mono tabular) · `title`(섹션 제목) · `label`(uppercase tracking eyebrow) · `body` · `num`. 스케일 표를 정해 컴포넌트가 그걸 참조.
9. **헤더 회원 셀렉트에 라벨 없음(트레이너).** 상단 우측 `김민준 ▾`이 "선택된 회원"인데 라벨이 없어 무엇인지 즉시 안 읽힘(계정/트레이너와 혼동 소지). `회원: 이해금 ▾`처럼 라벨 붙이거나 위치·아이콘으로 구분.
10. **섹션 헤더 통일.** /admin은 어떤 섹션은 `Eyebrow`(아이콘+대문자 라벨)를 쓰고 공지는 자체 `Megaphone` 헤더 → 공통 `SectionHeader`로 통일(아이콘+제목+선택 카운트/힌트).
11. **대비(접근성 AA) 점검.** 회색 배경 위 `text-muted`(부제·`-세`·비활성 탭 라벨) 대비가 낮아 보이는 지점 확인 → 미달분 한 톤 진하게. 현장 조명(밝은 헬스장)에서 폰 가독성에 직접 영향.

## 3. P2 — 근본: 디자인시스템 추출 (B · 다음 단계)

이번 감사에서 **여러 파일에 인라인 중복**이 반복 확인됨(ReapproachToday · TodoManual · AnnouncementGate · AdminAnnouncements · MyStats · 회원 카드 등이 같은 카드/배지/섹션헤더/빈상태 패턴을 각자 재작성). 이걸 `components/ui` 프리미티브로 뽑는 게 B:

- **Card** — `rounded-2xl border bg-card shadow-sm` 표준 컨테이너(+톤 변주).
- **SectionHeader** — 아이콘 + 제목 + 선택 카운트 배지 + 힌트 한 줄.
- **Badge** — 상태(OT/PT) · 강도(필수확인/일반) 등 pill.
- **Chip** — MBTI/통증/목표 같은 메타 태그.
- **StatTile** — 내 실적의 라벨+큰 숫자 타일(+보조 라인).
- **ListRow** — 회원명 + 컨텍스트 + `ChevronRight` 탭 행(재접근·미처리예약·공지 다 같은 꼴).
- **EmptyState** — "할 일 없음"류 빈 화면 한 줄.
- **Modal** — 게이트/삭제확인 오버레이(백드롭·카드·닫기 규칙).
- **Button** — §2-7 변주.

→ 추출하면 앱 전체 일관성이 오르고, **앞으로 4 리포트·3 FC·5 회원앱이 폴리시를 기본 상속**한다(가장 값싼 타이밍 = 지금, 새 기능 쌓기 전). 추출한 프리미티브를 이후 **C(claude.ai/design)** 에 카드로 올리면 살아있는 단일 소스가 된다.

## 4. 진행 순서 · 다음 액션

1. **P0부터 클로드코드 서브커밋**(1번 트레이너 이름 → 2 이름폴백 → 3 빈값 → 5 금액자간 → 4 스케줄 모바일). 작은 것·회귀 낮은 것부터, 항목별 diff 검토.
2. **P1**은 토큰/타이포/버튼을 먼저 정의(스펙 1개) → 클로드코드가 반영.
3. **P2(B)** — ui 프리미티브 추출 스펙을 따로 써서 진행(이번 X, 다음 라운드).
4. **미커버 감사 보완** — OT/PT AI 탭 + **폰 실기기 모바일** 캡처 몇 장(스케줄·2차 OT·회원)로 2라운드. 도구 모바일 캡처가 불안정하니 네가 폰 스크린샷 주면 그걸로 정밀 감사.
5. 전체 로드맵: **A(P0→P1) → B(ui 추출) → 남은 기능 4·3·5 → C(디자인 단일 소스).** 보안은 실회원 정보 입력 시작 직전 게이트.

## 5. 참고 — 감사 근거(캡처 관찰)

- **스케줄:** 주간 7열 그리드, 셀=회원명+트레이너(이메일 표기), 완료 체크 아이콘. 데스크톱 OK, 모바일 압박 우려.
- **할일:** 오늘 재접근(amber 카드) + 미처리 예약(중립, count 6, "N일 지남") + (하단)메모. 이름 폴백 "회원" 노출.
- **회원:** 아바타 이니셜 + OT/PT 배지 + 나이/직업 + MBTI·통증·목표 칩. 빈값 대시 다수.
- **내 실적:** 예상급여(emerald 카드, 큰 숫자) + 매출/클로징률 2타일 + 이번달 수업 + 매출내역 + PT가격설정. 타이포 위계 양호(참고 모델).
- **/admin:** 보라 테마, LIVE 배지, 트레이너 추가, **공지(C)**—작성폼(대상·필수확인·핀)+게시목록(핀·필수확인 칩 정상), 실데이터 요약 등. 공지 UI 배포·동작 확인됨.
