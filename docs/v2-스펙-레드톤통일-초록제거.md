## 1. 색 매핑 원칙 (초록 → ?)

| 기존 초록의 의미 | 새 색 |
|---|---|
| 브랜드·CTA·긍정·성공·상승·목표진행 | **레드**(`primary` 토큰 또는 `red-500/600`) |
| 위험·삭제·환불·하락·미등록 | **rose**(유지) |
| PT 회원 구분 | **sky**(PT 그룹 탭색과 일치) |
| OT 회원 구분 | **amber**(OT 그룹 탭색과 일치) |

- 텍스트: `text-emerald-*` → 긍정/강조면 `text-primary-strong`, 중립이면 `text-sub`/`text-muted`.
- 틴트 배경·보더: `bg-emerald-500/*`·`border-emerald-500/*` → `bg-primary-soft`·`border-primary/*`(또는 `bg-primary/5`·`border-primary/20`).
- 점·바·게이지: `bg-emerald-*`/`from-emerald-* to-emerald-*` → `bg-primary`/`from-red-500 to-red-600`.

## 2-B. CTA 그라데이션 초록 → 레드 일괄 (커밋 B · 기계적)

CTA 버튼이 대부분 리터럴 초록 그라데이션이다. **지금 즉시 레드로** 바꿔 초록을 눈에서 없앤다(이후 Button 프리미티브 이관 때 같은 레드라 색 변화 없음).

전 파일에서:
- `from-lime-400 to-emerald-500` → `from-red-500 to-red-600`
- `from-emerald-400 to-emerald-600` → `from-red-500 to-red-600`
- **위 버튼들의 짝 텍스트** `text-zinc-950` → `text-white`(레드 위 가독 · 로고 흰 모노그램 에코). ※ 그 버튼 요소에 한해서. `text-zinc-950`이 그라데이션 버튼이 아닌 다른 곳에도 있으면 거기는 두기.

대상(초록 그라데이션 있는 파일): `app/page.jsx`·`components/views/TodoManual.jsx`·`ScheduleBoard.jsx`·`TrainerLibrary.jsx`·`TrainerGoalSetter.jsx`·`TrainerProfileSettings.jsx`·`PtPricingSettings.jsx`·`PtWorkoutTab.jsx`·`PtReRegTab.jsx`·`PtInbodyTab.jsx`·`PasswordChange.jsx`·`ObservationTab.jsx`·`SecondOTTab.jsx`·`FirstOTTab.jsx`·`FirstOTAssist.jsx`·`VoiceLogTab.jsx`·`AuthGate.jsx`·`AnnouncementGate.jsx`·`AdminPayrollSettings.jsx`·`app/admin/page.jsx` 등. `Select-String -Path (파일들) -Pattern "emerald|lime"`로 잔여 확인.

> **게이지/프로그레스바도 이 그라데이션을 쓰면 함께 레드가 된다**(목표 달성 게이지 등) — 의도됨(진행·달성=레드). 프로그레스바는 버튼이 아니므로 `text-*` 교체 대상 아님(색 그라데이션만).
