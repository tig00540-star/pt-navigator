# pt-navigator — 실행 순서 로드맵 (지금부터)

> 마스터 백로그(`v2-앞으로-할일-마스터정리.md`) + 신규 기능 4건(`v2-기능요청-가능성판정.md`)을 **하나의 실행 순서**로 통합. 각 항목 = 무엇 / 크기 / 왜 이 순서 / 선행.
> 순서는 권장안(내 판단)이며 얼마든지 바꿔도 됨. 착수는 **한 번에 하나씩**, 스펙 잡기 전에 그 항목 전용 질문을 먼저.

---

## Phase 0 — 위생 (지금 · 30분 · 다음 SQL에 묶기)

- 테스트/시뮬 데이터 teardown(샘플 회원 토큰·phone·인바디·auth 유저 · 'PT 종료' 버튼으로 일부 자동) + 시드 `SIM-데이터` 정리.
- `2026-07-15-member-intake.sql` Supabase 실행 확인(미실행이면 신규 회원 등록 실패).
- B설계문서 git add · 루트 스크래치 12개 청소 · Anthropic 크레딧 Auto-reload · Confirm email ON.

## Phase 1 — 빠른 고가치 (제품 체감 먼저)

**1. ② 센터 머신·덤벨 등록 → 1·2차 OT·재등록 루틴 반영** [작음 · 반배선]
- 대표용 `center_machine` CRUD UI + AI에 '센터 전체 인벤토리' 주고 "있는 장비로만 루틴" 지시(프롬프트).
- 왜 먼저: 작고 즉효 · `center_machine`·AI 머신 주입 이미 있음 · 대표에게 바로 체감.

**2. ③ 수업일지 구조화 정리 (+ 회원앱 종목별 그래프)** [중]
- 음성일지 AI 출력을 `{종목·무게·횟수·세트}`로 정규화 저장 → 종목별 날짜순 집계 → 회원앱 "무게 증가 그래프".
- 왜 여기: 재등록 유도 강력 · 음성일지 AI가 이미 수치 추출 중 · 회원앱 심화·AI 게이트와 결.

---

## Phase 2 — 돈 받는 스위치 (수익화)

**3. B1 구독 스키마 + 층1 수동 잠금** [중 · 단계 배포 주의]
- `account` 딱지 2개 + `auth_account_id()` subscription AND + 기존 active 백필 + `my_account_status()` + 결제벽/온보딩 분기.

**4. B2 프리미엄 게이트(층2)** [소 · 선행 B1]
- 회원관리(S3 발급)·member-auth에 premium 체크 + UI 🔒. S3 발급이 유일 교체점이라 자연스러움.

> (B3 카카오·B4 결제 웹훅은 이후 · B4는 B1 선행.)

---

## Phase 3 — 토대 + 배정 (트레이너 성향 데이터)

**5. ④ 클로징 통계 대시보드** [중]
- 트레이너 클로징률 + 접근(통증/외형/가치)별 강점. 원천 데이터(`closing_result`/`approach`) 이미 축적 중.
- 왜 여기: **#4 배정 추천 · #1 이직 포터블 자산의 공통 토대** — 먼저 세우면 둘이 그 위에 얹힘.

**6. #4 OT 스케줄링 자동화 + 관리자 배정** [대형 에픽 · 조각별 배포]
- 조각 순서: ⓐ트레이너 가능시간 설정(`trainer_availability`) → ⓑ회원 슬롯 선택(회원앱 확장·동시성 가드) → ⓒ관리자 OT 파이프라인 뷰(신청/스케줄됨/미정) → ⓓ관리자 등록·배정(트레이너 수동추가 MemberForm 유지) → ⓔ배정 추천(⑤ 위에).
- 뼈대(`appointment`·ScheduleBoard·회원앱·owner 전체보기) 이미 있음. = 보류했던 FC 앞단을 '관리자 role'로 경량화.

---

## Phase 4 — UX/AI 심화

**7. OT 여정 전/후 재편** [중~대] — 3층 셸(지금 한마디/핵심 3카드/접힘) + 2차 실시간 분기 선택기. "한눈에 안 들어옴" 해소. 1단계(렌더+분량상한, 캐시 호환)부터.
**8. 회원앱 자가입력(비포애프터 사진) → AI 기능 + AI 게이트(남은>0)** [중~상] — 돈 드는 AI만 활성 PT일 때 열림(열람은 유지).
**9. 자기진화 루프** [중~대] — 트레이너 과거 결과·피드백 few-shot 주입(맞춤 세일즈 선생님). 절반은 뼈대 존재.

---

## Phase 5 — 이직 방지 + 런칭 준비

**10. #1 센터 선택 / 이직 자료 이전** [중대형 · 데이터 모델 분리]
- 정체성(auth·포터블 자산) ↔ 멤버십(account) 분리 + 센터 합류(코드/승인). B3 카카오/온보딩과 묶으면 효율.
- ★선행 결정: 포터블 범위 = **개인 성장자산만**(권장) vs 회원 데이터까지.

**11. 남은 데모 → 실데이터** — 재등록 CRM 대본 · admin QC(TRAINERS 하드코딩) · 카피봇 · 회원 성향요약 Hero.
**12. 보안 하드닝 + 런칭 체크리스트** — trainer_id 스코프·오프보딩 RLS·service_role 점검·라우트 인증·프롬프트 인젝션 / 개인정보 동의·의료카피·데모라벨.

> 수시 백로그: 초대메일(SMTP) · 회원정보 수정 UI · 스케줄 반복/변경/취소 · pay_policy 편집 확인 · 테스트 프레임워크.
