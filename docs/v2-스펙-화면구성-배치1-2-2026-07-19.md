# v2 수정 스펙 — 화면구성 배치 1·2 (바닥 충돌/가림 + 폼 뭉개짐)

> 근거: `docs/v2-화면구성-감사-2026-07-19.md`. **배치 1·2만** — 대수 불만("회원정보 뭉개짐"·"1차 OT 화면 이상")의 직격. 3~7은 폰 확인 후 판단(안 해도 되면 안 함).
> 규율: **로직·RLS·payload 불변.** 표시/배치/additive만. **라인번호 아님 — 코드 내용으로 매칭.** 배치별 1커밋 = revert 단위. 새 import는 파일 상단에.
> 리뷰 반영(클로드코드 지적 4건):
> - **①** 뿌리④ 해법은 `grid-cols-1` 못박기 ❌ → **`grid-cols-1 sm:grid-cols-2`**(폰 1열·태블릿+ 2열). **레포 내 참고 구현**: `app/page.jsx` MemberListTab `grid gap-3 sm:grid-cols-2` · `ObservationTab.jsx` ③/㉠ `grid gap-3 … sm:grid-cols-2` · `app/admin/page.jsx` 동일. Modal sheet는 `max-w-lg`(데스크톱 512)라 2열이 안 좁음 — 이 패턴이 정확히 맞음.
> - **②** 배치 4(글씨↑)와 5(overflow)는 **순서 의존** — 4를 먼저 하면 overflow가 악화. 4·5는 묶거나 5 먼저. **이 스펙(1·2)엔 4·5 없음** — 기록만.
> - **③** 배치 3(색 통일)은 취향·purge 위험이라 **마지막.** 이 스펙에 없음.
> - **④** 로그아웃은 탭바와 **"충돌"이 아니라 "가림"**(AuthGate가 `bottom-[calc(5rem+…)]` 오프셋이 있어 탭바와 안 겹침). 실제 탭바와 **충돌하는 건 FirstOTTab 메모바 하나.** 아래 명칭 정정 반영.

## 배치 1 — 바닥 정리 (커밋 2개)

### 커밋 1a · 로그아웃 플로팅 "가림" 제거 → 설정(내 정보)로 이관

**증상:** `AuthGate`의 로그아웃 버튼이 `fixed`로 **로그인된 전 화면 우하단에 상시 노출** → 스케줄 그리드·할일 카드·태도태그 칩 위를 덮음(가림). (탭바와 충돌은 아님 — 5rem 오프셋 있음.)

**수정 1 — `components/AuthGate.jsx`: 플로팅 블록 삭제**

찾을 것(렌더 반환부의 이 블록 전체):
```jsx
        {supabase && session && (
          <div className="fixed right-4 z-40 bottom-[calc(5rem+env(safe-area-inset-bottom))]">
            <button
              onClick={signOut}
              className="rounded-md border border-line bg-card px-2.5 py-1 text-[11px] font-medium text-sub shadow-sm hover:text-ink"
            >
              로그아웃
            </button>
          </div>
        )}
```
→ **삭제**(해당 `{supabase && session && (…)}` JSX만 제거, 감싸는 `<> … {inner}</>`는 유지).
⚠️ **`signOut` 함수 정의는 그대로 둘 것** — Paywall(`<Paywall … onSignOut={signOut} />`)이 여전히 사용. 지우지 말 것.

**수정 2 — `components/views/SettingsView.jsx`: '내 정보' 하단에 계정/로그아웃 추가**

상단 import에 추가:
```jsx
import { LogOut } from "lucide-react";
import { supabase } from "@/lib/supabaseClient";
import Eyebrow from "@/components/ui/Eyebrow";
```
`sub === "me"` 블록의 `<PasswordChange />` **바로 아래**에 추가:
```jsx
          <section className="rounded-2xl border border-line bg-card p-5 shadow-sm">
            <Eyebrow icon={LogOut}>계정</Eyebrow>
            <button
              onClick={() => supabase?.auth.signOut()}
              className="mt-3 w-full rounded-lg border border-line bg-elevate px-4 py-2.5 text-sm font-medium text-sub transition hover:border-primary hover:text-primary-strong active:scale-95"
            >
              로그아웃
            </button>
          </section>
```
- `supabase?.auth.signOut()` — 데모(supabase null) 가드 겸용. signOut 시 AuthGate의 `onAuthStateChange`가 session=null → 로그인 폼으로 자동 전환(기존 흐름 재사용, 로직 불변).
- 위치는 '내 정보'(첫 서브탭) 최하단이라 발견성 충분.

### 커밋 1b · FirstOTTab 현장메모 고정바 ↔ 하단탭바 "충돌" 해소

**증상:** `FirstOTTab`의 메모 입력바가 `fixed inset-x-0 bottom-0 z-30`인데 전역 `BottomNav`도 `fixed … bottom-0 z-40` → **둘 다 bottom-0**이라 메모바가 탭바에 겹쳐 눌림(하단탭바 도입 때 이 옛 고정바를 안 올린 회귀). = 7:18 "뭉개짐"의 직접 원인.

**해법(권장 · 최소·단일파일): 고정 해제 → 인플로우 카드로.** 바닥 고정 요소를 탭바 하나로 정리(픽셀 계산 불필요, 회귀 안전).

**`components/tabs/FirstOTTab.jsx`**

찾을 것:
```jsx
        <div className="fixed inset-x-0 bottom-0 z-30 border-t border-line/80 bg-card/85 backdrop-blur-xl">
          <div className="mx-auto flex max-w-5xl items-center gap-2 px-4 py-3 sm:px-6">
            <input
              value={note}
              onChange={(e) => setNote(e.target.value)}
              onKeyDown={(e) => e.key === "Enter" && addNote()}
              placeholder="현장 메모 빠르게 남기기…"
              className="flex-1 rounded-xl border border-line bg-elevate px-4 py-2.5 text-sm text-ink placeholder-muted outline-none transition focus:border-primary"
            />
            <button
              onClick={addNote}
              className="flex h-11 w-11 shrink-0 items-center justify-center rounded-xl bg-gradient-to-br from-red-500 to-red-600 text-white shadow-lg shadow-red-500/30 transition active:scale-95"
              aria-label="메모 저장"
            >
              <Send className="h-4 w-4" strokeWidth={2.5} />
            </button>
          </div>
        </div>
```
바꿀 것(바깥 `fixed` 래퍼 → 인플로우 카드, 내부 input/button은 그대로):
```jsx
        <div className="mt-6 flex items-center gap-2 rounded-2xl border border-line bg-card p-3 shadow-sm">
          <input
            value={note}
            onChange={(e) => setNote(e.target.value)}
            onKeyDown={(e) => e.key === "Enter" && addNote()}
            placeholder="현장 메모 빠르게 남기기…"
            className="flex-1 rounded-xl border border-line bg-elevate px-4 py-2.5 text-sm text-ink placeholder-muted outline-none transition focus:border-primary"
          />
          <button
            onClick={addNote}
            className="flex h-11 w-11 shrink-0 items-center justify-center rounded-xl bg-gradient-to-br from-red-500 to-red-600 text-white shadow-lg shadow-red-500/30 transition active:scale-95"
            aria-label="메모 저장"
          >
            <Send className="h-4 w-4" strokeWidth={2.5} />
          </button>
        </div>
```
- 루트 `pb-28`(탭바 여백)은 유지 — 메모가 인플로우라 탭바 뒤로 안 숨음. page.jsx 변경 불필요.
- 회원 identity 카드가 AI 아래인 **순서 문제(뿌리⑥)는 배치 6** — 이 커밋에서 건드리지 않음(스코프 유지).
- **대안(원하면):** "항상 하단 고정" UX를 유지하려면 `fixed … bottom-0`을 `fixed … bottom-[calc(3.25rem+env(safe-area-inset-bottom))]`로 올려 탭바 위에 얹고 루트 `pb-28`→`pb-40`. 단 픽셀 의존이라 **권장은 위 인플로우.**

## 배치 2 — 폼 뭉개짐(긴 한글 라벨 2열 → 폰 1열) · 커밋 2개

공통 규칙: **`grid grid-cols-2 gap-3` → `grid grid-cols-1 gap-3 sm:grid-cols-2`** (폰 1열, sm+ 2열). `col-span-2`(이름 필드 등)는 1열에서도 전폭이라 **그대로 두면 됨**(변경 불필요).

### 커밋 2a · 회원 등록/수정·계약 폼 ("회원정보 뭉개짐" 직격)

**1 — `app/page.jsx` · `MemberForm`** (2곳)
- 17필드 그리드: `{fields.map(` 를 감싼 `<div className="grid grid-cols-2 gap-3">` → `<div className="grid grid-cols-1 gap-3 sm:grid-cols-2">`
- 이월계약 그리드(구분: `mt-3` 있음): `<div className="mt-3 grid grid-cols-2 gap-3">` → `<div className="mt-3 grid grid-cols-1 gap-3 sm:grid-cols-2">`

**2 — `components/views/MemberEditForm.jsx`** (1곳)
- `{FIELDS.map(` 를 감싼 `<div className="grid grid-cols-2 gap-3">` → `<div className="grid grid-cols-1 gap-3 sm:grid-cols-2">`
- 참고: 시트(`Modal variant="sheet"`)는 `max-h-[90vh] overflow-y-auto` — **저장버튼 접힘 걱정은 이미 해소됨**(추가 조치 불필요).

**3 — `components/views/ContractAmountFields.jsx`** (1곳, PT 등록 확정 모달)
- 최상위 `<div className="grid grid-cols-2 gap-3">` → `<div className="grid grid-cols-1 gap-3 sm:grid-cols-2">`
- 참고: 이 모달은 `max-w-sm`(384)로 sm+에서도 좁아 2열이 다소 빠듯하지만, **불만은 폰(1열로 해소)** 이라 충분. 데스크톱까지 넉넉히 하려면 이 파일만 `grid-cols-1`(고정)로 둬도 무방 — 선택.

### 커밋 2b · 설정 폼 (같은 결함 · 우선순위 낮음)

**4 — `components/views/PtPricingSettings.jsx`** (2곳 모두)
- 파일 내 `<div className="grid grid-cols-2 gap-3">` **2곳**(가격 4필드 · 노출/표시순서) 전부 → `<div className="grid grid-cols-1 gap-3 sm:grid-cols-2">`

**5 — `components/views/TrainerProfileSettings.jsx`** (단일 항목을 2열로 감싼 잔재 → 언랩)

찾을 것:

## 검증 (폰 · 배포본 하드리프레시)

1. **1a:** 어느 탭에서도 우하단 로그아웃 플로팅 없음. 설정 → 내 정보 최하단 '계정 · 로그아웃'으로 정상 로그아웃(→ 로그인 폼). 데모모드 크래시 없음.
2. **1b:** 1차 OT 준비 하단에서 현장메모 입력이 **하단탭바에 안 겹치고** 카드로 정상 표시. 메모 추가 동작 유지.
3. **2a:** 회원 등록/수정 모달에서 입력칸이 **한 줄에 하나씩**(라벨 줄바꿈 계단현상 사라짐), placeholder 안 잘림, 저장버튼 보임. PT 등록 확정 모달 금액칸도 1열.
4. **2b:** 설정 가격/프로필 폼 1열 정렬, MBTI 칸 오른쪽 텅 빔 사라짐.
5. **빌드/린트:** `rm -rf .next && npm run build` green · `npm run lint` 0. Tailwind purge: 새 클래스 전부 정적 리터럴(동적 조립 0 유지 · `sm:max-w-[12rem]`는 정적).
