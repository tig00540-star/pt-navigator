## 0. 개요 · 비용/속도 가드레일 (설계 대전제)

M1의 회원 쓰기 패턴에 **스토리지**를 더한 것. **비용·속도 문제는 "업로드 전 클라 압축" 하나로 거의 해결.**

- **① 클라 압축·리사이즈**(최장변 ~1600px · JPEG ~0.82) → 장당 원본 3~8MB → **~200~500KB**. 스토리지·대역폭 10~20배↓, 업로드도 빠름. (아이폰 HEIC도 이 과정서 JPEG로.)
- **② 비공개 버킷 + RLS** — 회원 본인 폴더만 쓰기/읽기, 트레이너는 자기 account 회원만 읽기. 열람은 **서명 URL(signed URL)**.
- **③ DB엔 경로만** — 이미지 자체는 스토리지(CDN 서빙), DB(`member_photo`)엔 `storage_path` 텍스트만 → **앱 쿼리 속도 그대로.**
- **④ 갤러리 lazy load** — `<img loading="lazy">`.

> 결과: 압축 시 수천 명까지 Supabase Pro(스토리지 100GB 포함) 안에서 사실상 무비용, 앱 무영향.

## 1. 데이터·스토리지 계약

### 1.1 스토리지 버킷 `member-photos` (비공개)

- 경로 규약: **`{user_id}/{uuid}.jpg`** — 첫 폴더 = 회원 id(RLS 스코프 키).
- 비공개(`public=false`) → 열람은 서명 URL로만.

### 1.2 테이블 `member_photo` (회원 소유 · M1 패턴)

```
member_photo (
  id           uuid pk default gen_random_uuid(),
  user_id      uuid not null references user_table(id) on delete cascade,
  storage_path text not null,             -- 'member-photos' 버킷 내 경로({user_id}/{uuid}.jpg)
  label        text,                       -- 'before' | 'progress' | 'after' (자가 분류 · 자유)
  taken_on     date not null default current_date,
  note         text,
  created_at   timestamptz default now()
)
```

- 민감 이미지지만 **비공개 버킷 + RLS + 서명URL**로 보호. DB 행엔 경로·메타만.
- 프리미엄: 회원은 premium일 때만 로그인(B2) → 로그인된 회원 = premium. 별도 게이트 불필요.

### 1.3 마이그레이션

**`docs/migrations/2026-07-16-member-photo.sql`**

```sql
-- =============================================================================
-- 회원 자가입력 M2 — 비포애프터 사진: 비공개 버킷 + storage RLS + member_photo 테이블 + RLS
-- 실행: Supabase SQL Editor(수동) · git 기록본. 멱등.
-- 전제: auth_member_id() · auth_account_id(). storage.objects는 Supabase 기본 RLS on.
-- =============================================================================

-- 1) 비공개 버킷
insert into storage.buckets (id, name, public)
  values ('member-photos', 'member-photos', false)
  on conflict (id) do nothing;

-- 2) 스토리지 객체 RLS (경로 첫 폴더 = user_id)
--   회원: 본인 폴더 업로드/열람/삭제
drop policy if exists "member_photo_obj_insert" on storage.objects;
create policy "member_photo_obj_insert" on storage.objects for insert to authenticated
  with check (bucket_id = 'member-photos' and (storage.foldername(name))[1] = auth_member_id()::text);
drop policy if exists "member_photo_obj_select" on storage.objects;
create policy "member_photo_obj_select" on storage.objects for select to authenticated
  using (bucket_id = 'member-photos' and (storage.foldername(name))[1] = auth_member_id()::text);
drop policy if exists "member_photo_obj_delete" on storage.objects;
create policy "member_photo_obj_delete" on storage.objects for delete to authenticated
  using (bucket_id = 'member-photos' and (storage.foldername(name))[1] = auth_member_id()::text);
--   트레이너: 폴더(user_id)가 자기 account 회원이면 열람(읽기만)
drop policy if exists "trainer_photo_obj_select" on storage.objects;
create policy "trainer_photo_obj_select" on storage.objects for select to authenticated
  using (bucket_id = 'member-photos' and exists (
    select 1 from user_table u
    where u.id::text = (storage.foldername(name))[1] and u.account_id = auth_account_id()
  ));

-- 3) member_photo 테이블 + RLS (M1 cardio_log와 동일 틀)
create table if not exists member_photo (
  id           uuid primary key default gen_random_uuid(),
  user_id      uuid not null references user_table(id) on delete cascade,
  storage_path text not null,
  label        text,
  taken_on     date not null default current_date,
  note         text,
  created_at   timestamptz default now()
);
create index if not exists member_photo_user_idx on member_photo (user_id, taken_on desc);
alter table member_photo enable row level security;

drop policy if exists "member_photo_select" on member_photo;
create policy "member_photo_select" on member_photo for select to authenticated
  using (user_id = auth_member_id());
drop policy if exists "member_photo_insert" on member_photo;
create policy "member_photo_insert" on member_photo for insert to authenticated
  with check (user_id = auth_member_id());
drop policy if exists "member_photo_delete" on member_photo;
create policy "member_photo_delete" on member_photo for delete to authenticated
  using (user_id = auth_member_id());
-- 트레이너: 자기 account 회원 것 조회(읽기만)
drop policy if exists "trainer_photo_select" on member_photo;
create policy "trainer_photo_select" on member_photo for select to authenticated
  using (exists (
    select 1 from user_table u where u.id = member_photo.user_id and u.account_id = auth_account_id()
  ));

-- 검증:
--   select id, public from storage.buckets where id='member-photos';  -- public=false 1행
--   select policyname from pg_policies where tablename in ('objects','member_photo') and policyname like '%photo%';
-- 롤백: drop table if exists member_photo cascade;
--        delete from storage.objects where bucket_id='member-photos'; delete from storage.buckets where id='member-photos';
--        (+ 위 storage 정책 4개 drop)
```

## 2. 회원앱 UI — `app/m/[token]/page.jsx` (사진 섹션 추가)

**위치:** "유산소 기록"(M1) 아래에 **"비포애프터 사진"** 섹션.

### 2.1 클라 압축 유틸 (신규 · 파일 상단 or `lib/image.js`)

```js
// 이미지 압축·리사이즈 — 최장변 maxDim, JPEG quality. HEIC은 브라우저가 못 읽으면 실패(에러 처리).
export async function compressImage(file, maxDim = 1600, quality = 0.82) {
  const url = URL.createObjectURL(file);
  try {
    const img = await new Promise((res, rej) => {
      const i = new Image();
      i.onload = () => res(i);
      i.onerror = () => rej(new Error("이미지를 읽을 수 없어요"));
      i.src = url;
    });
    const scale = Math.min(1, maxDim / Math.max(img.width, img.height));
    const w = Math.round(img.width * scale), h = Math.round(img.height * scale);
    const canvas = document.createElement("canvas");
    canvas.width = w; canvas.height = h;
    canvas.getContext("2d").drawImage(img, 0, 0, w, h);
    const blob = await new Promise((res) => canvas.toBlob(res, "image/jpeg", quality));
    if (!blob) throw new Error("변환 실패");
    return blob;
  } finally {
    URL.revokeObjectURL(url);
  }
}
```

### 2.2 `PhotoSection` (신규 함수 컴포넌트)

**Props:** `{ me, photos, onReload }` (photos = member_photo 행 배열).

**동작:**
- **업로드:** `<input type="file" accept="image/*">`(모바일=카메라/앨범) + 라벨 선택(비포/진행/애프터 · 기본 진행) + 날짜(기본 오늘) + [올리기].
  1. `compressImage(file)` → blob.
  2. `path = `${me.id}/${crypto.randomUUID()}.jpg``.
  3. `memberSupabase.storage.from("member-photos").upload(path, blob, { contentType: "image/jpeg" })`.
  4. 성공 시 `memberSupabase.from("member_photo").insert({ user_id: me.id, storage_path: path, label, taken_on, note }).select()`(하드닝). 실패 시 방금 올린 파일 `storage.remove([path])`로 롤백.
  5. `onReload()`.
  - 업로드 중 스피너/진행 표시(파일 큰 첫 압축이 잠깐 걸릴 수 있음).
- **갤러리(서명 URL):** photos 행에 대해 `memberSupabase.storage.from("member-photos").createSignedUrls(paths, 3600)`로 서명 URL 일괄 생성 → `<img loading="lazy" src={signedUrl}>` 그리드(2~3열). 라벨·날짜 배지.
  - 서명 URL은 만료(1h)라 **섹션 로드/재조회 때마다 재생성**(state에 캐시).
- **삭제:** 각 사진 휴지통 → `member_photo.delete().eq("id",id).select()`(하드닝) + `storage.remove([storage_path])`. → `onReload()`.
- **빈 상태:** "아직 사진이 없어요. 비포 사진부터 남겨보세요."
- **데모/키 없음** 가드. **me 로드 후에만** 업로드 활성.
- **(선택) 비포↔최신 비교:** label='before' 1장과 최신 1장을 나란히 보여주는 상단 비교 카드(있으면 강력 · 없으면 그리드만).

**톤:** 회원앱 토큰·큰 글씨. lucide(`Camera`/`ImagePlus`/`Trash2`). 새 색/동적클래스 금지.

## 5. 주의·엣지

- **HEIC(아이폰):** 파일 인풋으로 올릴 때 iOS Safari가 대개 JPEG로 넘겨줘 canvas 압축이 먹음. 드물게 원본 HEIC이 안 읽히면 `compressImage`가 에러 → "다른 사진으로 시도" 안내. 베타서 잦으면 `heic2any` 도입(지금은 무의존).
- **EXIF 회전:** 최신 브라우저는 img/canvas에 방향 자동 적용(iOS OK). 사진이 옆으로 눕으면 그때 orientation 처리 추가.
- **서명 URL 만료:** 1h. 갤러리 재로드 시 재생성(state 캐시). 링크가 새어도 만료되면 무효 → 비공개 안전.
- **삭제 정합:** DB행 + 스토리지 파일 **둘 다** 지움. 스토리지 remove 실패해도 행은 지워졌으면 갤러리엔 안 뜸(고아 파일은 후속 정리 스크립트 여지 · 드묾).
- **업로드 실패 롤백:** 스토리지 업로드는 됐는데 DB insert 실패 시 방금 파일 `storage.remove`로 되돌림(고아 방지).
- **용량:** 압축 필수(§0). 원본 그대로 올리는 경로 만들지 말 것.
- **PT 종료/재개통:** member_auth 유저만 삭제라 `member_photo`·스토리지 파일 **보존**(user_table.id 기준) → 재개통 시 그대로 열림.
