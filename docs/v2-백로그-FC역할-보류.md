# v2 백로그 — FC(피트니스 컨설턴트) 역할 · 보류 기록

> 상태: **보류(defer).** 2026-07 결정. 트레이너(대수)가 PT 포커스 배경이라 FC 역할 직관이 약함 → "상상으로 설계하면 헛돈다" 판단. **실제 FC(상담실장)와 함께 설계 가능할 때 착수.**
> 이 문서 = 그때 바로 꺼내쓸 재료. 결론 먼저, 근거는 아래.

---

## 0. 한 줄 결정
풀 FC 역할은 **보류.** 단, 가장 값나가는 조각인 **트레이너 배정 매칭(AI)** 은 FC 없이도 트레이너/원장이 쓰는 가벼운 기능으로 **먼저 뽑을 수 있음**(선택). 그 외 상담 큐·상담 브리핑·FC 실적 뷰는 실제 FC 사용자 확보 후.

---

## 1. FC란 (퍼널에서의 위치)
FC = 회원권·PT를 파는 **상담실장.** 퍼널의 **맨 앞**을 담당:
```
방문 → 신규 상담 → 등록(헬스/PT) → 결제 → 트레이너 배정 → [OT → PT → 재등록: 트레이너 담당]
```
현재 앱은 **OT부터** 시작 → FC가 맡는 앞단(방문→등록→배정)이 통째로 비어 있음. 센터 매출의 큰 덩어리가 이 구간에서 나오지만 앱엔 미노출.

---

## 2. AI 레버리지 포인트 (매출 상승 관점)

1. **상담 브리핑 (FC판 OT 브리핑).** 트레이너용 클로징 브리핑을 상담에 대칭 적용 — 방문자 기본정보+목표 → 상담 arc + 거절대처 + 회원권/PT 추천 논리. `app/api/ot-brief` 패턴 재사용. "정답 대본 아닌 스파링" 철학 유지.

2. **★ 트레이너 배정 매칭 — 핵심 레버.** Phase 0의 **트레이너 프로필**(강한 방향·세일즈 스타일)을 배정에 활용. 예: "방문자 = 통증개선 목표 + 소극적 성향 → 통증방향 강하고 soft 세일즈 잘하는 트레이너 배정" → OT→PT 전환율↑. **잘못된 배정으로 새는 전환을 AI가 막는 것 = 매출 상승 원천.** FC ↔ 트레이너 프로필이 여기서 맞물림.
   - **조기 추출 가능:** 이 슬라이스는 FC 역할 없이도 트레이너/원장이 OT 배정 시 쓰는 기능으로 구현 가능. 나중에 FC 붙으면 그대로 편입. (전제: Phase 0 트레이너 프로필 선행.)

3. **퍼널 가시성(원장).** FC가 앞단을 채우면 원장 대시보드가 끝에서 끝까지: 방문 N → 상담전환 % → OT배정 → PT전환 → 재등록. 지금은 OT부터라 "방문→상담" 누수가 안 보임(실제 매출 누수가 거기일 수 있음).

---

## 3. 앱 모델링(착수 시)
- **역할:** `trainer.role`에 `fc` 추가(owner/trainer 옆). 기존 Auth/account/role RLS 재사용 — 새 뼈대 아님.
- **FC 전용 라우트**(#7 원칙 = 역할별 별도 라우트 + lazy): 신규 상담 큐 · 상담기록+AI브리핑 · 등록/결제 입력 · 트레이너 배정(AI 매칭) · FC 본인 실적(등록건수·매출·전환율).
- **등록:** 기존 `session_log` 계약 생성 로직을 FC가 만들고 `trainer_id` 배정하는 식.
- **상담 기록:** 신규 경량 테이블(consult_log) 또는 user_table 생성 시점 확장 — 착수 시 결정.

---

## 4. 보류 이유 & 재개 조건
- **보류 이유:** 설계자(트레이너)가 FC 역할 직관 부족 → 잘못 설계 위험. 앱 무게중심(PT·회원)과 설계자 강점이 회원앱 쪽 → 투자 우선순위 낮음.
- **재개 조건:** (a) 실제 FC/상담실장과 함께 설계, 또는 (b) 센터 파일럿에서 FC 워크플로 관찰 가능해질 때.
- **그 전까지:** 2번 "배정 매칭" 슬라이스만 필요 시 조기 추출(선택). 나머지는 동결.

---

## 5. 로드맵 영향
FC 보류 → Phase 2가 solo(개인트레이너) + 회원앱 준비로 채워지고 **회원앱이 앞으로 당겨짐.** 전체 순서:
Phase 0(목표매출+트레이너프로필) → Phase 1(클로징통계·admin탭분리·라이브러리) → Phase 2(solo + 회원앱 준비, "역할=별도 라우트" 원칙 정착) → Phase 3(보안 하드닝) → Phase 4(회원앱+사진).
