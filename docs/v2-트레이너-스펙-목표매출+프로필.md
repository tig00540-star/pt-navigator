# v2 트레이너 스펙 — 월별 목표매출 + 프로필 (Phase 0)

> 역할·흐름: 웹Claude(이 스펙) → 트레이너 → 클로드코드(diff) → 트레이너(git diff) → 웹Claude(검토) → 폰.
> 커밋은 트레이너 직접(파일지정 `git add` · lint 통과 후 · `git show --stat HEAD`). SQL=Supabase 대시보드 실행 후 git엔 `docs/migrations` 기록본. 환경: PowerShell · lint=`npm.cmd run lint` · 폰=Vercel 하드리프레시.

## 0. 목표 & 원칙
두 기능, 둘 다 트레이너 계열 데이터 · 둘 다 보안 게이트 밖(트레이너 본인 데이터, 기존 계정 RLS 재사용).
1. **월별 목표매출** — 트레이너가 자기 이달 목표 설정 → **월간 리포트/내실적의 "목표 달성률"을 진짜로 완성**(4-a에서 다음 급여구간을 임시 프록시로 썼던 자리를 실제 목표로 교체).
2. **트레이너 프로필** — 성향·강한 방향·세일즈 스타일·mbti·소개. 지금은 **스키마 + 편집 UI만**(AI 프롬프트 연동·배정 매칭은 후속). 데이터를 먼저 쌓는다.

핵심 설계 결정:
- **프로필은 `trainer` 테이블 컬럼 추가 금지 → 별도 `trainer_profile` 테이블.** trainer에 본인 UPDATE를 열면 `role`/`active` 자기수정(권한상승) 위험. 격리 테이블로 role/active 잠금 유지.
- **구조화 재사용:** `strong_approaches`=CLOSING_APPROACH_OPTS 값 · `sales_style`=SALES_INTENSITY_OPTS 값(labels.js 기존). 자유서술 최소화 → 나중에 측정값 대조·집계 가능.
- 컴포넌트는 **PtPricingSettings 패턴**(자기완결 fetch·`.select()` 하드닝·데모 가드·Toast) 그대로.

## 1. 서브커밋 구성
- **A. 목표매출** — A1 SQL(`trainer_goal`) · A2 `TrainerGoalSetter` + MyStats 배선(fetch·달성률·리포트 전달) · A3 MonthlyReport 달성률.
- **B. 프로필** — B1 SQL(`trainer_profile`) · B2 `TrainerProfileSettings` + MyStats 렌더.

A 먼저(리포트 닫힘) → B. SQL은 각 서브커밋 착수 시 Supabase 실행 후 기록본 커밋.

## 2. 서브커밋 A — 월별 목표매출

### A1. SQL 마이그레이션 — `docs/migrations/2026-07-12-trainer-goal.sql`
```sql
-- 2026-07-12 trainer_goal — 트레이너 월별 개인 목표매출 (달성률 원천)
-- 스펙: docs/v2-트레이너-스펙-목표매출+프로필.md. Supabase 대시보드 실행 후 이 파일=git 기록본.
-- pay_scheme/payroll_run 계정 RLS 패턴. 단 쓰기 주체 = 트레이너 본인(자기 목표 자기 설정).

create table if not exists trainer_goal (
  id             uuid primary key default gen_random_uuid(),
  account_id     uuid not null default auth_account_id(),
  trainer_id     uuid not null default auth.uid(),
  ym             text not null,                 -- 'YYYY-MM' (KST)
  target_revenue integer null,                  -- 목표 순매출(원). null=미설정
  updated_at     timestamptz not null default now(),
  unique (trainer_id, ym)
);

alter table trainer_goal enable row level security;

-- SELECT: 같은 계정(원장 전체 + 트레이너 본인). 쓰기: 트레이너 본인만.
create policy trainer_goal_sel on trainer_goal for select
  using (account_id = auth_account_id() and (auth_is_owner() or trainer_id = auth.uid()));
create policy trainer_goal_ins on trainer_goal for insert
  with check (account_id = auth_account_id() and trainer_id = auth.uid());
create policy trainer_goal_upd on trainer_goal for update
  using (account_id = auth_account_id() and trainer_id = auth.uid())
  with check (account_id = auth_account_id() and trainer_id = auth.uid());
-- delete 정책 없음(목표는 값 수정으로 관리). 필요 시 추가.

-- 검증: select trainer_id, ym, target_revenue from trainer_goal;
-- 롤백: drop table if exists trainer_goal;
```
> `unique(trainer_id, ym)` = 트레이너·월당 1행. 설정은 upsert(onConflict). `target_revenue` integer(≤21억, payroll_run과 동일형). account_id/trainer_id는 DEFAULT라 앱이 생략해도 with_check 통과.

### A2. `components/views/TrainerGoalSetter.jsx` (신규 · 컴팩트)
목표 입력 + 이달 달성률 표시. MyStats가 `goals` 소유(리포트에도 넘겨야 함) → 이 컴포넌트는 편집 UX만, 저장 후 `onSaved(row)`로 상위에 반영.

```jsx
"use client";
import { useState } from "react";
import { Target } from "lucide-react";
import { supabase } from "@/lib/supabaseClient";
import { won } from "@/lib/format";
import Toast from "@/components/ui/Toast";
import { useToast } from "@/hooks/useToast";

// props: uid(트레이너 id) · ym('YYYY-MM') · target(현재 목표 or null) · revTotal(이달 순매출) · onSaved(row)
export default function TrainerGoalSetter({ uid, ym, target, revTotal, onSaved }) {
  const [value, setValue] = useState(target == null ? "" : String(target));
  const [saving, setSaving] = useState(false);
  const { toast, showToast } = useToast();
  const pct = target ? Math.round((revTotal / target) * 100) : null;

  const save = async () => {
    if (saving) return;
    if (value === "" || isNaN(Number(value)) || Number(value) < 0) return showToast("목표 금액을 입력하세요");
    const payload = { trainer_id: uid, ym, target_revenue: Number(value), updated_at: new Date().toISOString() };
    setSaving(true);
    if (!supabase) { onSaved?.({ ...payload }); showToast("저장됨(데모)"); setSaving(false); return; }
    // account_id는 DEFAULT auth_account_id() — 생략(with_check 통과). onConflict=trainer_id,ym → upsert.
    const { data, error } = await supabase.from("trainer_goal")
      .upsert(payload, { onConflict: "trainer_id,ym" }).select();
    if (error || !data || data.length === 0) { showToast("저장 실패 — 다시 시도하세요"); setSaving(false); return; }
    onSaved?.(data[0]);
    showToast("이달 목표 저장됨");
    setSaving(false);
  };

  const inputCls = "w-full rounded-lg border border-line bg-elevate px-3 py-2 text-sm text-ink placeholder-muted outline-none focus:border-primary disabled:opacity-50";
  return (
    <section className="rounded-2xl border border-line bg-card p-5 shadow-sm">
      <div className="flex items-center gap-2 text-[11px] font-semibold uppercase tracking-wider text-muted">
        <Target className="h-3.5 w-3.5" /> 이달 목표매출 · {ym}
      </div>
      {target != null && (
        <div className="mt-3">
          <div className="flex items-baseline justify-between text-sm">
            <span className="text-sub">달성률</span>
            <span className="tabular-nums font-bold text-ink">{pct}% <span className="text-[11px] font-normal text-muted">({won(revTotal)} / {won(target)})</span></span>
          </div>
          <div className="mt-1.5 h-2 overflow-hidden rounded-full bg-elevate">
            <div className="h-full rounded-full bg-gradient-to-r from-emerald-400 to-emerald-600" style={{ width: `${Math.min(100, pct ?? 0)}%` }} />
          </div>
        </div>
      )}
      <div className="mt-3 flex gap-2">
        <input type="number" inputMode="numeric" value={value} onChange={(e) => setValue(e.target.value)} disabled={saving} placeholder="목표 순매출(원)" className={inputCls} />
        <button onClick={save} disabled={saving}
          className="shrink-0 rounded-lg bg-gradient-to-br from-emerald-400 to-emerald-600 px-4 py-2 text-sm font-bold text-zinc-950 transition active:scale-95 disabled:opacity-50">
          {saving ? "저장 중…" : "저장"}
        </button>
      </div>
      <Toast message={toast} />
    </section>
  );
}
```
> 100% 초과 달성 시 progress는 100%로 clamp(숫자 pct는 실제값 표시). 달성률 색은 emerald(급여·성공 톤 일치).

### A 엣지·검증
- 목표 미설정 월(과거월 등): target=null → 달성률 UI 안 뜸, 리포트는 nextBand 폴백. 신규 도입이라 과거월은 대부분 미설정 = 자연스러움.
- target=0 저장 방지(< 0만 막고 0은 허용? 0이면 나눗셈 위험) → **저장 시 0도 막기**: `Number(value) <= 0`이면 showToast. (위 A2 코드의 `< 0`을 `<= 0`으로.)
- upsert `.select()` 0행 = 조용한 실패 하드닝(교훈1) 적용됨.
- lint: 서브커밋 A만 올릴 때 B용 import 넣지 말 것.

## 3. 서브커밋 B — 트레이너 프로필

### B1. SQL — `docs/migrations/2026-07-12-trainer-profile.sql`
```sql
-- 2026-07-12 trainer_profile — 트레이너 프로필(성향·강한방향·세일즈스타일·mbti·소개)
-- ★ 별도 테이블 이유: trainer에 컬럼 추가 + 본인 UPDATE 정책 = role/active 자기수정(권한상승) 위험.
--   프로필을 격리 테이블로 두어 trainer.role/active는 SQL/원장만 건드리게 유지.
-- strong_approaches = CLOSING_APPROACH_OPTS(pain/appearance/value/health/other) → closingApproachStats(측정)와 대조.
-- sales_style = SALES_INTENSITY_OPTS(soft/standard/strong).

create table if not exists trainer_profile (
  trainer_id        uuid primary key references trainer(id) default auth.uid(),
  account_id        uuid not null default auth_account_id(),
  strong_approaches text[] null,
  sales_style       text null,
  mbti              text null,
  bio               text null,     -- 성향·성격·한줄소개(자유서술 · 후속 AI 톤 재료)
  updated_at        timestamptz not null default now()
);

alter table trainer_profile enable row level security;

-- SELECT: 계정 내 열람(원장 배정 참고 + 트레이너 본인). 쓰기: 본인만.
create policy trainer_profile_sel on trainer_profile for select
  using (account_id = auth_account_id());
create policy trainer_profile_ins on trainer_profile for insert
  with check (account_id = auth_account_id() and trainer_id = auth.uid());
create policy trainer_profile_upd on trainer_profile for update
  using (account_id = auth_account_id() and trainer_id = auth.uid())
  with check (account_id = auth_account_id() and trainer_id = auth.uid());

-- 검증: select trainer_id, strong_approaches, sales_style, mbti from trainer_profile;
-- 롤백: drop table if exists trainer_profile;
```

### B2. `components/views/TrainerProfileSettings.jsx` (신규 · 자기완결 = PtPricingSettings 패턴)
- **fetch**: 마운트 1회, `supabase.auth.getUser()` → uid → `from("trainer_profile").select("*").eq("trainer_id", uid).maybeSingle()`. 값 폼에 로드.
- **폼 필드**:
  - `strong_approaches` — CLOSING_APPROACH_OPTS 다중선택(칩 토글). 선택된 값 배열.
  - `sales_style` — SALES_INTENSITY_OPTS 단일선택(라디오/세그먼트).
  - `mbti` — text input(짧게).
  - `bio` — textarea(성향·성격·한줄소개).
- **저장**: 데모 가드 → `upsert({ trainer_id: uid, strong_approaches, sales_style: sales_style||null, mbti: mbti.trim()||null, bio: bio.trim()||null, updated_at }, { onConflict: "trainer_id" }).select()` → `.length>0` 하드닝 → Toast. account_id 생략(DEFAULT).
- **UI**: `Eyebrow icon={UserCircle} 내 프로필` + 카드. labels import: `import { CLOSING_APPROACH_OPTS, SALES_INTENSITY_OPTS, labelOf } from "@/lib/labels";`. 칩 토글은 기존 Chip 스타일(선택=primary-soft, 미선택=elevate) 참고. lucide 아이콘은 `UserCircle`(또는 IdCard) 사용.
- **문구**: 상단 힌트 한 줄 "여기 입력은 이후 AI 브리핑이 내 스타일에 맞추는 재료가 돼요(배정·개인화)." — 지금은 저장만 함을 과장 없이.

> 구현 편의: 다중선택 칩 토글 패턴 —
> ```jsx
> const toggle = (v) => setApproaches((p) => p.includes(v) ? p.filter((x) => x !== v) : [...p, v]);
> {CLOSING_APPROACH_OPTS.map((o) => (
>   <button key={o.value} type="button" onClick={() => toggle(o.value)}
>     className={`rounded-full px-3 py-1 text-xs font-medium transition ${approaches.includes(o.value) ? "bg-primary-soft text-primary-strong" : "bg-elevate text-sub hover:text-ink"}`}>
>     {o.label}</button>
> ))}
> ```

### B3. MyStats.jsx 렌더
- import: `import TrainerProfileSettings from "@/components/views/TrainerProfileSettings";`
- 설정 영역(PtPricingSettings·PasswordChange 근처)에 `<TrainerProfileSettings />` 렌더. **자기완결이라 MyStats 상태·통계와 무관**(AI 연동은 후속).

### B 엣지·검증
- 프로필 없던 트레이너: maybeSingle=null → 빈 폼. 첫 저장=insert(upsert). 이후 update.
- `strong_approaches` 빈 배열 저장 OK(text[]). null과 빈배열 혼용 주의 — 빈 선택이면 `[]` 저장.
- `sales_style` 미선택 허용(null).
- AI 연동(ot-brief 프롬프트에 프로필 주입)·배정 매칭 = **이번 스코프 밖**(별도 스프린트). 이번엔 데이터+편집만.
- lint: B용 import는 B 서브커밋에서만.

## 4. 검증 체크리스트(커밋 전)
1. **SQL 먼저 Supabase 실행 → 검증쿼리로 테이블·정책 확인 후** 코드 커밋(코드가 없는 테이블 치면 조용히 실패).
2. `npm.cmd run lint` green · 미사용 import 0.
3. 폰: 목표 저장 → 달성률 progress·% 즉시 반영, 월 셀렉터로 이달만 목표 뜨는지(과거월 미설정=폴백).
4. 목표 0/음수/빈값 저장 차단. upsert 재저장 시 값 갱신(중복행 안 생김 · unique).
5. 프로필 저장 후 새로고침 재진입 시 값 유지(upsert·재로드).
6. 권한: 다른 트레이너 목표/프로필이 안 보이는지(RLS sel — 목표는 본인만, 프로필은 계정내 열람 의도).

## 5. 참고 — labels.js 재사용(신규 0)
`CLOSING_APPROACH_OPTS`(pain/appearance/value/health/other) · `SALES_INTENSITY_OPTS`(soft/standard/strong) · `labelOf`. 이미 존재 — import만.
